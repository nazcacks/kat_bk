1. 표기 범례

TN-05 화면 ID (화면설계서 인벤토리 연결) Step-up 재인증 필요 작업 포함 화면(3.2) 관리자 회사 관리자(또는 운영 책임자) 전용 파트너 차단 외부 세무대리인 기본 접근 불가(4.4) AI AI 보조 기능(휴먼인더루프) └ 회색 들여쓰기: 화면 내 탭/하위 기능(별도 메뉴 아님)
  • 메뉴는 Role의 메뉴권한으로 렌더링하고, 화면 내 버튼은 행위권한으로 별도 통제한다(4.3). 권한 없는 메뉴는 비활성 표시가 아니라 미노출이 기본이다.
  • 외부 세무대리인은 permittedScopeTags에 포함된 메뉴만 노출되며(화이트리스트 방식), 인사/급여·민감정보 화면은 태그와 무관하게 차단된다(4.4).
  • 운영자가 테넌트 컨텍스트로 진입하면 업무 화면 메뉴 구조를 동일하게 사용하되 접근모드(5장)에 따라 행위가 제한되고 전 화면에 접근모드 배너가 표시된다.

4. 메뉴 운영 규칙

  • 메뉴 미노출 원칙: 권한 없는 메뉴는 비활성이 아닌 미노출. 단, 요금제 미포함 기능(다중 회계기준·XBRL·외부 변환 입력 등 feature.* 플래그, 11.6)은 업그레이드 안내 배지로 노출 가능(영업 정책).
  • 기능 플래그 연동: feature.multiStandardLedger OFF 시 TN-20의 병행 기준 탭과 TN-10의 기준 선택 드롭다운 숨김, feature.xbrlDisclosure OFF 시 TN-11 미노출.
  • 구독 상태 연동(7.1): SUSPENDED=조회 메뉴만 활성(입력·신고 메뉴 비활성+사유 툴팁), GRACE=조회+내보내기(TN-19)만, TERMINATED=보관기간 내 조회/다운로드만.
  • 접근모드 연동(5장): 운영자 VIEW 진입 시 조회 메뉴만, SUPPORT_SESSION은 설정 메뉴까지(회계 실무 메뉴 차단), BREAK_GLASS는 승인 범위 내 메뉴만 한시 활성.
  • 마감 상태 연동(11.9): 메뉴 자체는 유지하되 마감 기간 데이터의 입력 액션이 화면 단위로 차단된다(CLOSING_LOCKED).
