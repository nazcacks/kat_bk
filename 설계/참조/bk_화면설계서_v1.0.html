1. 문서 개요

목적상세설계서 v2.0의 기능·엔티티·API·검증규칙을 화면 단위로 구체화한다. 화면 ID, 레이아웃(와이어프레임), 필드, 액션·검증, 연계 API, 권한·상태 표시를 정의한다.
대상 채널운영 콘솔 관리회사 운영자(OperatorUser) 전용 / 업무 화면 이용회사 사용자(TenantUser) 전용 — 세션·도메인 분리(상세 3.4)
화면 ID 체계CO-xx 공통(로그인·알림), OP-xx 운영 콘솔, TN-xx 테넌트 업무 화면
상세화 수준핵심 14개 화면은 와이어프레임+필드+액션·검증까지 상세 설계, 그 외 화면은 4장 인벤토리에 범위·관련 설계 장·대표 API를 정의(차기 버전에서 상세화)
표기 원칙모든 화면 요소는 상세설계서 v2.0의 장 번호로 근거를 표기한다(예: “마감 잠금(16.1)”). 엔티티·API·오류코드는 v2.0 12·13장 정의를 그대로 사용한다.

항목표준

키보드 주도 UXTab/Enter 셀·행 이동, Ctrl+S 저장, F2 셀 편집, Ctrl+D 행 복사. 마우스 의존 최소화.

실시간 검증 레이어차대평균·필수값을 전송 전 풋터에서 연산. 경고 2단계: Amber 경고(저장 가능) / Red 차단(저장 불가). 서버 검증 이중 유지.

2.3 민감정보 · 마스킹 표시 (17장, 13.3)

  • 기본 목록·출력은 항상 마스킹: 주민번호 9201**-*******, 계좌 110-***-****56, 급여 ***,***.
  • 평문 조회는 👁 아이콘 → Step-up 인증 모달(재인증+사유 입력) → 제한 시간 표시 후 자동 재마스킹. PersonalDataAccessLog 기록 고지 문구를 모달에 명시.
  • 외부 파트너 계정은 평문 조회 아이콘 자체를 미노출(4.4 — API 403과 이중 차단). 예외 조회는 승인 절차 화면(TN-03) 경유.

2.5 처리 신뢰성 UX (13.3)

  • 멱등 처리: 생성·전송·반영 버튼은 클릭 즉시 비활성화+진행 표시, Idempotency-Key 자동 발급. 재클릭 시 동일 응답 안내. 충돌 시 409 IDEMPOTENCY_CONFLICT 안내.
  • 분산 락 충돌: LOCK_NOT_ACQUIRED 수신 시 “다른 사용자가 진행 중입니다(작업자·시작 시각)” 토스트 + 재시도 버튼(2.1.2).
  • 비동기 작업: 대용량 조회·리포트는 202 접수 → 진행률 표시 → 완료 알림(18장) → 결과 다운로드(AsyncReportJob, 2.3).
  • 오류 표준: 13.3 오류코드를 사용자 메시지로 매핑(8장 부록 표). 코드·traceId를 ‘상세 보기’로 노출(문의 시 전달용).
