BK 회계/세무 서비스 구현 화면설계서 v1.0 — Phase 1

작성일: 2026-06-16 · 버전: v1.0 · 범위: Phase 1(코어 플랫폼 & 기장 기본) 구현 대상 화면 — 특히 전표 상세 입력 화면 전종과 전표 보조부 입력 화면 전종을 구현 수준으로 상세화

기술 스택: 백엔드 NestJS / 프론트 React 18 / DB PostgreSQL 16(RLS·파티셔닝) · 본 문서의 모든 그리드는 가상 스크롤·키보드 주도·실시간 검증(16.2)을 전제로 한다.

1. 문서 개요 · Phase 1 범위

목적Phase 1 구현 착수를 위해 화면별 레이아웃(와이어프레임)·입력 필드·액션/검증·연계 API·권한·상태를 구현 가능한 수준으로 정의한다. 특히 전표 입력과 전표 보조부 입력은 전표 유형별·보조부 유형별 화면을 모두 표현한다.
Phase 1 정의멀티테넌시(SHARED+RLS)·인증/2FA·운영 콘솔(테넌트/구독/표준)·권한 위임·관리자 접근 로그·공통 표준 배포·전표 코어·차원 엔진 기초·회계기간/마감 워크플로우(11.9)·증빙 수집/적격 판정(11.12)·장부/시산표/재무제표 기초(구현계획 Phase 1).
전표 범위일반(대체)·입금·출금·매입·매출 전표의 상세 입력 화면, 복사/반복/템플릿/역분개, 검색·수정·삭제. 라인 차원 입력(9.5), 멱등 채번(tenant+연도+유형), 부가세 자동라인, 외화 환산, 행위자 표식(4.5).
보조부 범위거래처별(매출처/매입처 원장)·어음(받을/지급)·예금(은행계정)·카드·가지급금/가수금·부가세(대급금/예수금)·현금출납장·재고 수불(기초)·고정자산대장(기초). 보조부는 전표 라인의 관리항목으로 입력되며 보조원장으로 집계된다.
표기 원칙화면 ID는 본 문서 전용으로 JV-xx(전표)·AUX-xx(보조부)를 사용하고, 상위 화면설계서의 TN-05(전표 입력)·TN-07(장부)·TN-08(증빙)·TN-09(마감)와 연결한다.
본 문서는 업무 화면(TenantUser) 기준이다. OPERATOR_LED 회사의 기장 담당자는 OP-12 기장 워크벤치에서 BOOKKEEPING 컨텍스트로 동일 화면을 사용한다(상세 5.2.1). TENANT_LED(자가 기장)가 Phase 1의 기본 시나리오다.

2. 공통 UI 표준 (요약)

2.1 화면 셸 · 그리드 표준

  • 셸: 상단 GNB(회사명·기장 모드 배지·알림·사용자) + 좌측 LNB(전표/장부·결산/마감·증빙·설정) + 콘텐츠 영역. 메뉴는 Role 메뉴권한, 버튼은 행위권한으로 렌더링(4.3).
  • 대량 그리드(16.2): 가상 스크롤(windowing), 키보드 주도(Tab/Enter 이동, F2 편집, Ctrl+S 저장, Ctrl+D 행 복사, F3 계정/거래처 검색), 유도규칙 자동기입(코스트센터→사업장), 풋터 실시간 검증.
  • 2단계 검증: Amber 경고(저장 가능) / Red 차단(저장 불가). 서버 검증 이중 유지.
  • 마스킹(17장): 계좌·카드·주민번호 기본 마스킹, 👁 평문 조회는 Step-up 인증 + 사유 + 자동 재마스킹.
  • 보조부 호출: 전표 라인에서 관리항목이 설정된 계정(거래처/어음/카드/예금/수량 등)에 진입하면 F4 또는 자동으로 보조부 입력 팝업이 뜬다(5장).

5.1보조부 개념 · 전표 연동 구조

전표 보조부(보조원장)는 특정 계정의 거래 내역을 관리항목(거래처·어음·카드·예금·수량·세무 등) 단위로 분해·집계하는 종속 장부다. 본 서비스에서 보조부는 별도 입력 화면이 아니라 전표 라인 입력 중 관리항목 계정에 진입할 때 호출되는 입력 팝업(F4)과 독립 조회용 보조원장 뷰 두 형태로 제공된다. 입력은 항상 전표에서 발생하며(단일 입력원), 보조부는 그 결과를 관리항목 키로 누적한다.

구조설명 (근거)
입력원모든 보조부 기표는 전표 라인에서 발생. 라인 저장 시 journal_line_aux(1:N)에 보조부 유형·참조키·금액·수량·일자 기록(11.1·12.x).
관리항목 트리거계정과목의 관리항목 설정(거래처필수/어음/카드/예금/수량/세무)에 따라 라인 진입 시 해당 보조부 팝업이 자동/F4 호출. 필수 미입력 시 Red 차단.
집계 시점보조원장 잔액·증감은 POSTED(기표) 전표만 집계(11.2). 임시저장·결재대기 전표는 미반영(별도 표시).
독립 뷰거래처/어음/예금/카드/가지급금/부가세/현금/재고/고정자산 보조원장은 TN-07 장부 조회에서 조건 검색·드릴다운(원전표) 가능.
마감 연동마감기간 보조부 신규 기표는 전표와 동일하게 CLOSING_LOCKED로 차단(16.1).
차원(사업장·코스트센터)보조부의 1차 키는 관리항목이고 차원은 전표 라인 속성이라 F4 입력 팝업에는 차원을 두지 않는다(라인에서 입력, 보조부 비중복). 독립 보조원장 뷰에서는 보조부 성격에 따라 차원을 필터/컬럼으로 선별 제공한다(아래 각 화면).
