기준 문서: bk_화면설계서_v4.0.html(3장 메뉴 구조 · 4장 화면 인벤토리) · bk_서비스_상세설계서_v_4.0.md

작성일: 2026-06-15 · 버전: v4.0 · 채널: 운영 콘솔(관리회사 OperatorUser) / 업무 화면(이용회사 TenantUser) — 도메인·세션 분리(상세 3.4)

v4.0 변경: 실무 위탁기장 산출물 역분석 16개 보강 항목(상세 0.1·11.13~11.21·18.3) 반영 — 업무 화면에 월별 보고패키지 TN-22·계정별 명세서/tie-out TN-23·기간배분/리스 스케줄 TN-24·외화 평가 TN-25·관계사 거래 TN-26·Cost-plus/차이분석 TN-27·예실 대비 TN-28·그룹 연결보고 Export TN-29·제조원가/재고수불 TN-30·미결항목/질의 TN-31 신설, 「외화·관계사·관리회계」/「원가·재고」 메뉴 그룹 신설, 부가세(TN-12)에 매입원장 세무상태, 운영 콘솔 표준 카탈로그(OP-05)·기장 워크벤치(OP-12)에 보고패키지 표준·조립 추가

1. 표기 범례

TN-05 화면 ID (화면설계서 인벤토리 연결) Step-up 재인증 필요 작업 포함 화면(3.2) 관리자 회사 관리자(또는 운영 책임자) 전용 파트너 차단 외부 세무대리인 기본 접근 불가(4.4) AI AI 보조 기능(휴먼인더루프) 기장 기장 모드(v3.0) 관련 메뉴 — 모드별 노출/가변(4장) v4.0 실무 산출물 보강 메뉴(상세 11.13~11.21·18.3) └ 회색 들여쓰기: 화면 내 탭/하위 기능(별도 메뉴 아님)
  • 메뉴는 Role의 메뉴권한으로 렌더링하고, 화면 내 버튼은 행위권한으로 별도 통제한다(4.3). 권한 없는 메뉴는 비활성 표시가 아니라 미노출이 기본이다.
  • 외부 세무대리인은 permittedScopeTags에 포함된 메뉴만 노출되며(화이트리스트 방식), 인사/급여·민감정보 화면은 태그와 무관하게 차단된다(4.4).
  • 운영자가 테넌트 컨텍스트로 진입하면 업무 화면 메뉴 구조를 동일하게 사용하되 접근모드(5장 — `VIEW`/`SUPPORT_SESSION`/`BOOKKEEPING`(v3.0)/`BREAK_GLASS`)에 따라 행위가 제한되고 전 화면에 접근모드 배너가 표시된다.
  • 기장 모드 배지(v3.0): 업무 화면 GNB에 회사 기장 모드를 상시 표시 — 자가 기장(TENANT_LED) / 위탁 기장(OPERATOR_LED) / 병행(HYBRID). 모드별 메뉴 가변 규칙은 4장.
  • 업종·기능 플래그(v4.0): v4.0 메뉴는 기능 플래그·요금제·업종 설정에 따라 노출된다 — 외화·관계사·예실·그룹보고는 feature.* 플래그, 제조원가/재고(TN-30)는 industryType=MANUFACTURING|DISTRIBUTION_WITH_INVENTORY + manufacturingCostEnabled 회사에만 노출(상세 11.21).

챗봇(TN-18)은 메뉴 접근과 무관하게 응답 데이터가 사용자 권한 범위로 서버에서 강제 필터링되므로(11.8), 매트릭스의 △는 “메뉴 노출”을 의미하고 데이터 범위는 별도 통제다. 운영 콘솔 메뉴는 운영자 직무 Role(테넌트 운영/표준 관리/보안 책임자/감사자 + 기장 Role `BK_PREPARER`/`BK_REVIEWER`/`BK_MANAGER`(v3.0, 상세 4.5))로 분리한다 — 기장 Role은 OP-12와 배정 회사의 업무 화면(BOOKKEEPING 컨텍스트)만 접근하며 거버넌스 승인자 겸직이 금지된다. 본 매트릭스는 TENANT_LED 기준이고 기장 모드별 가변은 4장을 따른다. v4.0 메뉴는 기능 플래그·업종 설정으로 노출이 추가 제한된다.

4. 메뉴 운영 규칙

  • 메뉴 미노출 원칙: 권한 없는 메뉴는 비활성이 아닌 미노출. 단, 요금제 미포함 기능(다중 회계기준·XBRL·외부 변환 입력·외화·관계사·예실·그룹보고·제조원가 등 feature.* 플래그, 11.6·11.16~11.21)은 업그레이드 안내 배지로 노출 가능(영업 정책).
  • 기능 플래그 연동: feature.multiStandardLedger OFF 시 TN-20의 병행 기준 탭과 TN-10의 기준 선택 드롭다운 숨김, feature.xbrlDisclosure OFF 시 TN-11 미노출. v4.0: feature.fxRevaluation→TN-25, feature.intercompany→TN-26, feature.costPlus→TN-27, feature.budgeting→TN-28, feature.groupReporting→TN-29 노출 제어.
  • 업종 설정 연동(v4.0): industryType=MANUFACTURING 또는 DISTRIBUTION_WITH_INVENTORY + manufacturingCostEnabled 회사에만 「7. 원가/재고」(TN-30)와 보고패키지(TN-22)의 제조원가명세(CM) 컴포넌트가 노출된다(상세 11.21).
  • 구독 상태 연동(7.1): SUSPENDED=조회 메뉴만 활성(입력·신고 메뉴 비활성+사유 툴팁), GRACE=조회+내보내기(TN-19)만, TERMINATED=보관기간 내 조회/다운로드만.
  • 접근모드 연동(5장): 운영자 VIEW 진입 시 조회 메뉴만, SUPPORT_SESSION은 설정 메뉴까지(회계 실무 메뉴 차단), BREAK_GLASS는 승인 범위 내 메뉴만 한시 활성.
  • 마감 상태 연동(11.9): 메뉴 자체는 유지하되 마감 기간 데이터의 입력 액션이 화면 단위로 차단된다(CLOSING_LOCKED). v4.0: 계정별 미tie-out(TN-23)·미해결 미결항목(TN-31)·원가계산 미완(TN-30)은 마감 체크리스트(TN-09)·보고패키지 검증(TN-22)의 차단/경고 게이트로 작동한다.
  • 기장 모드 연동(v3.0, 상세 21장):
    • TENANT_LED(자가 기장): 본 구조도 기본 상태. TN-21 기장 협업함·TN-31 미결항목 협업 미노출(자체 사용 시 TN-31은 내부 미결관리로 사용 가능).
    • OPERATOR_LED(위탁 기장): 이용회사 측 전표 입력(TN-05) 비활성(→ TN-21 안내), 마감(TN-09)·결산(TN-10)·신고(TN-12/13)·보고패키지(TN-22)는 확인(Ack)·전송 승인 전용 화면으로 가변. 입력·실행·패키지 조립은 관리회사 기장 담당자가 OP-12 경유 BOOKKEEPING 컨텍스트에서 수행.
    • HYBRID(병행): 분담 규칙(BookkeepingScopeRule, 상세 21.6) owner 영역만 입력 메뉴 활성. 마감·결산·신고·보고패키지 메뉴는 주기장 측에만 실행 버튼 노출(비주기장은 확인 전용).
    • 모드 전환(상세 21.3) 효력일에 배치가 메뉴 구성을 일괄 전환하며, 전환 이력은 BookkeepingConfigHistory로 보존된다.
