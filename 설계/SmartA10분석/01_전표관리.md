# 1. 일반전표입력 — 원문 전체

**메뉴 위치**: [회계관리] 전표관리 ▶ 일반전표입력 · [가이드 영상보기](https://www.wehago.com/westudio/watch/QaAOkd)

## ▣ 개요
이 메뉴는 "부가가치세신고와 관련된 매입매출 거래 이외의 모든 거래를 입력"하는 기능입니다. 세금계산서 수수거래 등 부가가치세 신고 관련 거래는 [매입매출전표입력]에 입력해야 신고 서식에 반영됩니다. 본 메뉴의 입력 데이터는 분개장, 총계정원장 등의 장부 및 재무제표에 반영됩니다.

## ▣ 메뉴 구성

### 1) 기간설정 및 조건검색
**① 기간/월별**
- **[기간]**: 특정 월이 아닌 기간을 설정하여 전표 조회/입력
- **[월별]**: 특정 월의 전표 조회/입력

**② 현금잔액** — 조회일(기간설정 시 조회기간 종료일) 기준 현금잔액이 표시되며, 클릭 시 [현금출납장] 메뉴로 이동합니다.

**③ 조회**
- [∨] 버튼 클릭 시 하단으로 '조건검색'의 조건 화면이 노출되어 상세 검색 가능
- 조건검색 적용 시 분홍색 [∧] 버튼으로 표기됨

**▶ TIP_조건검색의 [동일전표번호(상대전표)] 보기 옵션** — 특정 조건에 부합하는 분개 전체(동일전표번호로 입력된)를 조회하는 경우 체크합니다. *예) 조건검색 [계정과목(811.복리후생비)] 조회 결과*
- **동일전표번호(상대전표)보기_미체크**: 해당 조건만 조회
- **동일전표번호(상대전표)보기_체크**: 동일 전표번호의 전체 분개 조회

### 2) 전표입력
**※ 로그인한 아이디에 따라 화면구성이 상이할 수 있으며, 기본화면구성의 [계정과목(간편입력)] 기준으로 작성되었습니다.**

**① 기간(년/월/일), 월별(월/일)** — 증빙(거래전표, 영수증, 통장내역 등)에 기재된 거래일자를 입력합니다.
- **요일별 색상구분**: 검정색-평일 / 파란색-토요일 / 빨간색-일요일, 공휴일
- **[N]**: 당일 신규 입력한 전표에 표기
- **[U]**: 당일 수정된 전표에 표기

**② 번호** — 일자별/분개별 전표번호가 자동채번되며, 각 일자별 00001~99999까지 입력 가능합니다. 더보기(기능모음)의 [화면구성] > [전표번호보기] 사용여부에 따라 화면 표기여부를 설정할 수 있습니다.

**③ 구분** — 전표의 유형을 선택합니다:
- 현금전표: 1.출금, 2.입금
- 대체전표: 3.차변, 4.대변
- 결산전표: 5.결차, 6.결대

> **주의사항**
> - 결산분개가 아니지만 (5.결차/6.결대)로 입력한 경우 재무제표에 반영되지 않을 수 있음
> - 대체전표(3.차변, 4.대변)입력시 차액 발생 시 상대분개의 구분이 자동으로 반영되도록 하려면 더보기(기능모음)의 분개설정 > [차.대변 변환]을 [1.여]로 설정

**④ Code/계정과목** — 거래 내용에 해당하는 계정과목을 입력하는 란입니다.
- Code 란에서 계정과목명을 두 글자 이상 입력하거나 [F2] 코드도움 또는 [C]아이콘을 통해 검색 입력 가능
- [환경설정]메뉴 공통 탭의 계정과목코드체계 > 세목사용 설정에 따라 표시 자리수 변경:
  - [0.세목미사용] 선택 시: 3자리 (예: 811.복리후생비)
  - [1.세목사용] 선택 시: 5자리 (예: 81100.복리후생비)

**⑤ Code/거래처** — [F2] 코드도움 또는 [C]아이콘을 통해 거래처 검색 및 입력이 가능합니다.
- **㉠ 거래처코드보이기 [ON]**: [자동완성 기능사용여부] 설정 가능, 거래처 Code란 입력시 [거래처등록]메뉴의 거래처가 자동검색됨
- **㉡ 거래처코드보이기 [OFF]**: [자동완성] 비활성화, 화면의 거래처코드란이 제거됨 (※ [환경설정] 공통탭의 거래처코드 화면표시 설정에 따름)
- **㉢ 거래처코드 미입력**: Code 없이 상호만 입력하는 경우 [거래처]란을 클릭하여 직접 입력. 거래처명만 입력한 경우 [거래처원장]에서 거래처별 조회 불가하며 [미등록거래처]로 조회됨
- **㉣ 분개설정_거래처코드**: 전표입력시 엔터키로 상단의 거래처코드와 거래처명을 복사하려면 더보기(기능모음)의 분개설정 > 거래처코드 [1.여]로 설정

**▶ TIP_ 빠른거래처 등록 방법**
1. 거래처 Code란에서 [+] 클릭 시 거래처코드란에 [000000]로 표시
2. 신규 등록할 거래처명을 입력 후 엔터(Enter)
3. [거래처등록] 창이 팝업되면 [수정(Ctrl+D)]을 클릭하여 거래처상세내용을 입력 후 [등록(tab)]

**⑥ 차변(출금)/대변(입금)** — 키보드 숫자키 옆의 [+]로 천원 단위(000) 금액을 빠르게 입력할 수 있습니다.

**⑦ Code/적요** — [적요]란에서 거래내역을 간단히 입력할 수 있으며, [F2]코드도움 또는 [C]아이콘을 통해 적요를 입력할 수 있습니다. 가이드영역의 [적요항목(편집)]란에서 적요를 수정/추가할 수 있습니다.

**⑧ 관리** — 가이드 영역에서 설정한 관리항목, 업무용승용차관리 및 자금관리 등의 내역을 라벨로 표기하며, 입력 값에 따라 두개 이상이 동시 표기될 수 있습니다.
- **관리**: [관리항목](사원, 부서, 현장, 프로젝트, 자금, 제품) 또는 카드계좌가 설정된 경우 표기
- **차량**: [업무용 승용차 관리]내역이 설정된 경우 표기
- **자금**: [당좌예금], [차입금], [지급/받을어음], [미불계획관리 및 신용카드] 내역이 설정된 경우 표기
- ※ 관리항목(사원, 부서, 현장, 프로젝트 등)은 두 글자 이상의 코드 또는 명칭으로 조회하여 등록합니다.
- ※ 관리항목은 [환경설정] 공통탭의 관리코드 사용 여부 및 자리수 관리에서 사용여부를 설정하며, [관리/차량/자금]의 경우 [계정과목 및 적요등록]의 계정과목 코드별 관리항목 사용여부를 [O]로 설정합니다.

### 3) 가이드 영역
- **① 안내메시지 영역**: 마우스 커서가 위치한 항목에 대한 기본 가이드 제공
- **② 공통 단축키 안내**: [환경설정] 공통탭의 단축키 설정 > [1.사용함] 후 이용 / 메뉴 단축키는 툴바 기능버튼 hover 시 표기, 더보기(기능모음)에서 기능별 단축키 확인
- **③ 전표번호/연동정보/정렬순서**: 커서 위치 전표의 전표번호 표기, 전표복사/이동·타 메뉴 연동 전표의 연동정보 표기, 현재 정렬 값 표기
- **④ 선택전표소계**: 커서 위치 전표번호의 차변/대변 합계 표기
- **⑤ 일정**: 위하고 [일정관리] 서비스 정보 확인, 일정 클릭 후 [더보기 ▶] 클릭시 [일정관리] 실행
- **⑥ 작성자**: 커서 위치 전표의 작성자 정보·최종입력시간 확인
- **⑦ 메모**: 커서 위치 전표 상세내용 메모, 조건검색[∨]에서 메모검색 가능

### 4) 기능키(툴바)
- **① 차액전표보기(Shift+F10)**: 대차 차액 발생 전표 조회
- **② 행고정/행고정해제**: [행고정] 클릭 시 검색된 전표 첫행부터 커서 위치 행까지 화면 고정, 해제는 [행고정 해제]
- **③ 자동전표검증**: 현재 회계기간 기준, 자동전표처리의 [신용카드],[현금영수증],[통장]에서 [일반전표입력]으로 전송한 전표 중 [전표확정]이 아닌 경우 검증
- **④ 데이터 정렬**: 정렬 기준(입력순, 날짜순, 거래처코드순, 계정코드순, 계정과목명순) 선택
- **▶ TIP_체크펜**: 추후 확인·검토 데이터는 '체크펜' 이용. 전표 클릭 후 마우스 오른쪽(R)클릭 > 체크펜 설정/취소

### 5) 더보기(기능모음)_편집
- **① 전표이동 (Ctrl+M)**: 액션바의 [전표이동]과 동일, 선택 전표를 지정 일자로 이동
  - 제한: 자금항목(어음등), 일마감, 계좌관리, 증빙, 자동전표 데이터는 [전표이동] 제한 / 이동 시 업무용승용차관리항목·카드계좌여부는 이동되지 않음
- **② 전표복사 (F4)**: 선택 전표를 지정 일자로 복사
  - 제한: 일마감된 기간으로는 복사 제한 / 복사 시 자금항목(어음등)·업무용승용차관리항목·카드계좌여부는 복사되지 않음
- **③ 어음등록 (Shift+F9)**: 지급어음 등록
  - **수령일**: 어음책 수령일 / **어음종류**: 1.어음 2.당좌 3.가계 4.전자 / **어음으로대체**: 어음종류가 (2.당좌,3.가계)인 경우 활성화, 미사용시 (1.어음,4.전자)로 대체 가능 / **금융기관**: 수령 금융기관 / **시작어음번호** / **매수**: 입력 매수만큼 연속 번호 등록 / **전자어음 일괄등록**: 수령일·금융기관 동일 다수 전자어음 일괄등록 / **편집**: 기존 수령일·어음종류·금융기관 입력 후 [편집]으로 수정/삭제
- **④ 중복전표검색 (Shift+F7)**: 조건(일자, 유형, 계정, 거래처, 금액) 중 체크 항목 선택하여 중복전표 검색

### 6) 더보기(기능모음)_엑셀 — 엑셀올리기 (Ctrl+U)
사용자 작성 엑셀파일을 [일반전표입력]으로 업로드. 지정 양식 없음, 업로드시 양식 설정으로 엑셀제목과 데이터 제목 매칭. **※ 과거데이터 제공 기수에서는 미지원.**
- **① 파일찾기**: 업로드할 엑셀파일 [열기]
- **② 엑셀업로드(양식 재설정)** — 작업순서 ㉠ > ㉡ > ㉢
  - **㉠ 엑셀내역**: 업로드 엑셀이 표기, 엑셀제목 라인 클릭
  - **㉡ 엑셀제목설정**: 선택한 제목라인을 [데이터 제목]과 연결하는 기능키
  - **㉢ 엑셀제목**: [엑셀제목]과 [데이터 제목] 항목 연결, 키보드 숫자키로 매칭
  - **필수 매칭**: 월/일/구분/계정과목코드/계정과목명/차변(출금)/대변(입금)
  - **조건부 필수**: 부서명·사원명·현장명·프로젝트명 입력 시 관련 코드도 필수 매칭
  - 예) 엑셀제목설정 (월) = 데이터제목 (1.월)인 경우 엑셀제목설정 [월]란에서 숫자키 [1] 입력
- **③ 미리보기**: 양식 설정 완료 후 업로드 데이터 미리보기
- **④ 변환옵션**
  - **1. 계정과목** ([환경설정] 공통탭 계정과목코드 체계에 따라 자리수 입력)
    - *계정코드 기준*: [계정과목 및 적요등록]의 코드·명으로 업로드 (예: 등록 '101.현금', 엑셀 '101.현찰' → [101.현금])
    - *계정과목명이 다를 경우 기초데이터 반영*: 엑셀 계정명으로 업로드 (예: → [101.현찰]로 변경 후 업로드)
  - **2. 거래처** (코드 미입력 시 거래처명만 반영)

# 2. 매입매출전표입력 — 원문 전체

**메뉴위치**: 회계 전표관리 ▶ 매입매출전표입력

## ▣ 개요
부가가치세신고와 관련한 매입매출 거래를 입력하는 메뉴로, 매입매출 거래를 입력하는 상단부와 분개를 입력하는 하단부로 구성. 상단부 데이터는 부가가치세 신고자료(부가가치세신고서, 세금계산서합계표, 매입매출장 등)에, 하단부 분개 데이터는 재무회계자료(계정별원장, 재무제표 등)에 반영됩니다.

## ▣ 입력방법 — 상단부(신고자료)
- **❶ 일자**: 거래일자(거래증빙 표시 일자)
- **❷ 거래처명**: 입력방법은 일반전표와 동일. 단 매입매출전표는 **거래처코드 반드시 입력**(미입력 시 세금계산서합계표 등 미반영). 코드·명 입력 시 '사업자등록번호' 또는 '주민등록번호' 표시
  - 조회 4가지: ①거래처명 공란 더블클릭 ②[C]아이콘 ③F2 코드도움 ④자동완성 / 신규거래처는 Tab키로 등록

- **❿ 수입납부유예입력**
- **⓫ 일마감**: 일마감 시 마감일자 이전 데이터 수정·삭제·입력 불가
- **⓬ 전표복사**: 선택 전표를 지정 일자로 복사
- **⓭ 전표이동**: 선택 전표를 지정 일자로 이동
- **⓮ 자동분개설정**: 거래처명 항목 전표 자동완성 수행
- **⓯ 빠른 부가세 입력설정**: 과세유형선택 항목에 대해 별도 입력화면 표시
- **⓰ 차액분개보기**: 대차 차액 발생 전표 조회
- **⓱ 분개계정과목설정**: 환경설정 없이 기본계정·추가계정 설정, 전자세금계산서 거래처코드 변경 여부 설정
- **⓲ 데이터정렬**: 입력순·날짜순·거래처코드순·사업자번호순
- **⓳ 세금계산서 인쇄**: 인쇄조건 선택 후 인쇄
- **⓴ 국세청 신고 여부**: 전자세금계산서(계산서) 발급분 중 과세기간 종료일 다음달 11일 이내 국세청 신고 "여부" 선택. "부" 선택 시 합계표의 전자외 발행분에 반영

# 3. 정기청구입력 — 원문 전체

**메뉴위치**: 회계 전표관리 ▶ 정기청구입력

## ▣ 개요
매월 정기적으로 청구되는 수입에 대해 조회·등록하고 자동전표를 발행하는 메뉴. 기본사항등록과 정기청구 입력으로 구성.

## ▣ 입력방법 — 기본사항등록 탭
매월 청구 거래처 중 정기청구 대상 거래처를 조회·등록. (거래처등록 메뉴 입력이 선행되어야 함)
- **❶ 청구기간**: 청구 기간 입력. 청구 전송시 필수, 자금계획입력 메뉴에서 기입
- **❷ 청구일**: 청구일·청구주기월 입력. 31일 입력 시 해당 월 말일 반영. 전표전송시 필수
- **❸ 수금예정일**: 청구월 이후 수금예정월일. 청구일보다 이전 날짜 입력 불가. 31일 입력 시 말일 반영
- **❹ 자금항목**: 자금계획입력 정기청구 반영 자금항목 선택. 자금계획입력 전송시 필수
- **❺ 청구품목(적요)**: 전표발생시 반영할 청구품목·적요 입력
- **❻ 전표구분**: 전표 종류·반영 유형 선택
  - **1.일반전표**: 일반전표입력 반영, 추가 선택 없음
  - **2.매입매출**: 매입매출자료입력 반영, "1.과세"/"2.면세" 추가선택 — 1.과세 → 매출유형 11.과세 자동발생 / 2.면세 → 매출유형 13.면세 자동발생
- **❼ 분개금액**: 직접 입력 불가. 하단(청구분개(대변)) 공급가액(VAT별도) 자동 반영. 전표 전송시 필수
- **❽ 청구과목**: 청구 계정과목(대변) 설정. 조회: 더블클릭/[C]/F2/계정과목명 두 글자
- **❾ 공급가액, 부가세**: 청구 공급가액·부가가치세 입력. (전표구분 "1.일반전표" 또는 "2.매입매출"의 "2.면세" 선택 시 부가세 입력창 비활성)
- **❿ 수금과목**: 수금 상대 계정과목(차변) 설정
- **⓫ 금액**: 수금 금액(VAT포함) 입력
- **⓬ 사원, 부서, 현장, 프로젝트**: 코드 입력 또는 [C]아이콘 선택
- **⓭ 거래기간**: 거래처등록의 거래 시작(종료)일 자동 반영
- **⓮ 사용여부**: 정기청구 사용여부 반영. 미사용 거래처는 삭제하지 않고 "1.부"로 변경 관리

## ▣ 정기청구입력 탭
기본사항등록 탭 내용 반영. 청구일 선택·조회 후 기능모음의 전표추가 또는 우측 하단 전표추가 클릭 시 일반전표입력/매입매출전표입력(자금계획입력 포함)에 반영.

## ▣ 편리한 기능(기능모음)
- **❶ 전표추가**: 선택 데이터를 일반전표입력·매입매출전표입력·자금계획입력에 반영
- **❷ 주민번호 별표사용 설정**: 사원 주민등록번호 뒷자리 별표처리/숫자표시 설정

# ▣ BK 설계 매핑·시사점 (전표관리)

1. **전표 이원화** — 부가세 무관=일반전표(JV-01), 부가세 대상=매입매출전표(JV-04/05). BK 동일 채택.
2. **전표 구분 6종**(1출금·2입금·3차변·4대변·5결차·6결대)이 BK `JournalLine.entryCode`와 1:1. 결차/결대 오용 시 재무제표 미반영 경고 필요.
3. **전표이동(Ctrl+M)·전표복사(F4)** 원문 확인 — 화면설계서 v2.1 전표이동(F6)이 Smart A 동작과 정합. 단 자금항목/업무용승용차/카드계좌/자동전표는 이동·복사에서 제외되는 제한사항을 BK 이동 검증에 반영해야 함.
4. **관리항목 노출 로직** — 환경설정(관리코드 사용여부) + 계정과목별 관리항목 [O] 설정으로 결정 → BK **SA-BAS-03/07** 및 v2.1 우측 패널 근거.
5. **분개유형(매입매출)** 0~7(현금/외상/혼합/카드/외상추가1~3): '2.외상'은 외상매출/매입금 자동분개·수정불가, 수정하려면 '3.혼합'. → BK 자동분개 규칙 반영.
6. **세목 5자리**(81100) vs 3자리(811) — 환경설정 세목사용 → BK 계정과목 세목 5자리(C7) 정합.
7. **엑셀올리기 변환옵션**(계정/거래처/관리항목 코드기준 vs 신규등록) → BK 대량 업로드 매핑 정책 참고.
8. **예정누락·수정세금계산서** → BK 부가세 신고 정정 워크플로 근거.
