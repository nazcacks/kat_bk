## 1) 총계정관리원장

> 메뉴위치 : 회계 보조장부Ⅰ ▶ 총계정관리원장

### ▣ 개요
회계기간 선택한 계정과목의 거래처별, 현장별, 프로젝트별, 부서/사원별 월별 증가, 감소, 잔액을 조회하는 메뉴입니다.

### ▣ 조회방법
- **❶ 계정과목** : F2 코드도움으로 조회하고자 하는 계정과목의 코드를 입력합니다. 공란으로 조회를 하는 경우 전체 계정과목이 조회됩니다.
- **❷ 구분** : 거래처, 현장, 프로젝트, 부서/사원 중에 해당되는 항목을 선택하면 구분 값 선택에 따라 우측에 조회조건이 추가되며, 조회 조건에 해당하는 데이터가 조회됩니다.
- **❸ 거래처, 현장, 프로젝트, 부서/사원** : F2 코드도움으로 조회하고자 하는 거래처, 현장, 프로젝트, 부서/사원의 코드를 입력합니다. 공란으로 조회를 하는 경우 전체 데이터가 조회됩니다.

### ▣ 편리한 기능(기능모음)
- **❶ 원장조회** : 선택한 데이터의 원장 내용을 조회하며, 전표수정을 통해 수정이 가능합니다.

### ▣ 조회방법
- **❶ 기간** : 조회하고자 하는 기간의 월을 입력하여 조회가 가능합니다.
- **❷ 유형** : 세금계산서와 계산서 중에 선택합니다.
- **❸ 구분** : 매출과 매입 중에 선택합니다.

- **❷ 주민등록번호 별표 사용설정** : 개인정보인 주민등록번호의 뒷자리 별표 표시 여, 부를 설정합니다.
- **❸ 데이터정렬** : 거래처코드순, 거래처명순, 사업자(주민)번호순으로 데이터를 정렬합니다.

- **❸ 카드사** : F2 코드도움으로 조회하고자 하는 카드사명을 입력하거나, 전체 카드사 검색하고자 하는 경우 공란으로 조회합니다.

### ▣ 편리한 기능(기능모음)

## 8) 운행기록부(업무용승용차)

> 메뉴위치 : 회계 보조장부Ⅰ ▶ 운행기록부(업무용승용차)

### ▣ 개요
차량별 운행일지 작성 및 운행내역과 누적거리, 유류비 내역을 조회하는 메뉴입니다.

### ▣ 입력방법

**운행기록 탭에는 차량별로 입력하며, 구분 선택 후 주행거리를 입력하면 업무사용비율이 자동 계산됩니다.**
- **❶ 기간** : 입력(조회)월을 선택하면 누적으로 조회됩니다.
- **❷ 유형** : 기본선택은 9.업무용승용차(0~2)이며 회사차, 렌트, 리스 차량이 모두 선택되어 조회됩니다.
  - 9.업무용승용차(0~2), 0.회사차, 1.렌트, 2.리스, 3.직원명의차량, 4.직원소유타인명의, 5.회사차(비영업용), 6.리스(비영업용), 7.렌트(비영업용), 8.전체
- **❸ Code/차량번호** : F2 코드도움을 이용하여 업무용승용차등록 메뉴에 입력되어 있는 차량코드를 선택합니다.
- **❹ 일자** : 월일을 입력하면 해당 요일은 자동으로 표기됩니다.
- **❺ 사원/부서** : 업무용승용차등록 메뉴 운행자정보에 관리사원이 입력되어 있으면 자동 반영되며, 직접 입력하는 경우 F2 코드도움으로 사원을 선택하면 부서가 자동 반영됩니다.
- **❻ 구분** : 1.출근, 2.퇴근, 3.업무, 4.비업무, 5.출퇴근, 6.업무왕복, 7.개인왕복 중 차량의 사용 구분을 선택합니다.
  - [참고사항 : 구분]
    - 출근용 선택 시 : 출발지[자택] → 도착지[회사] 주소가 자동 반영됩니다.
    - 퇴근용 선택 시 : 출발지[회사] → 도착지[자택] 주소가 자동 반영됩니다.
    - 업무용 선택 시 : 직전 라인의 도착지가 출발지의 주소로 자동으로 반영됩니다.[거래처 코드도움 : F2]
    - 비업무용(개인) 선택 시 : 출발지와 도착지의 주소를 직접 입력합니다.
- **❼ 주행 Km** : 주행거리를 입력합니다.
- **❽ 주행 후 Km** : 주행 Km 입력시 자동 반영됩니다.
- **❾ 비고** : 업무용으로 차량을 사용하는 경우 세부적인 내용(0.직접입력, 1.거래처방문, 2.제조시설등 사업장방문, 3.회의참석, 4.판촉활동, 5.교육 등 기타업무활동)을 추가로 입력할 수 있습니다.

**누적거리확인 탭은 계기판의 주행거리를 확인하여 오차거리를 안분하는 경우 사용합니다.**
- **❶ 누적거리 점검방법** : 실 주행거리와 차량의 실제 계기판의 총 누적거리가 다른 경우에 누적거리점검을 이용하여 점검합니다.
  - 누적거리점검을 하고자 하는 데이터를 선택합니다.
  - 누적거리점검을 클릭합니다.
  - 점검 후 누적거리를 입력합니다. (차량 실제 계기판의 총 누적거리를 입력합니다.)
  - 오차허용범위를 입력합니다.(기본설정 : 10)
  - 확인(Tab) 버튼을 클릭합니다.
  - ※ 선택한 점검 전 주행거리에 오차거리가 안분되어 점검 후 주행km 반영됩니다.
- **❷ 운행기록부 반영** : 누적거리 점검 후 주행 거리를 운행기록부에 반영합니다. 기능모음의 운행기록반영을 클릭하면 점검 후 주행거리를 주행거리에 반영됩니다. 증감거리는 삭제됩니다.

**유류비 확인 탭은 주행거리에 따른 유류비 사용금액을 확인하여 유류비 예측 등 업무에 사용합니다.**
- **❶ 조회** : 유류비 계정설정에 계정과목 설정 후 조회를 클릭하면 일자별 운행내역, 유류비 금액이 반영됩니다.
- **❷ 유류비 금액에 단가 및 표준연비 적용** : 유류비 금액에 단가 및 표준연비를 적용하여 유류비 km를 확인할 수 있습니다.
  - 기능모음의 표준연비일괄적용을 클릭합니다.
  - 유류비 단가를 입력합니다.
  - 표준연비를 입력합니다.
  - 확인[Tab]을 클릭합니다.

### ▣ 편리한 기능(기능모음)
- **❶ 복사** : 운행기록이 동일한 경우 복사기간을 선택하고, 구분값(0.주말/공유일 제외, 1.복사기간전체)을 선택해 복사합니다.
- **❷ 주소입력열기** : 출발지와 도착지 주소를 입력합니다.
- **❸ 입력자료유형 보기/숨기기** : 입력된 자료 유형의 화면 표시하거나 숨깁니다.
- **❹ 운행기록 새로불러오기** : 운행기록 탭에 입력된 운행기록을 새로 불러와 누적거리 확인 탭에 반영합니다.
- **❺ 운행기록부 엑셀 업로드** : 지정된 엑셀 서식을 다운받아 데이터를 입력하여 운행기록부를 엑셀 업로드 합니다.
- **❻ 운행기록 반영** : 점검 후 주행거래를 주행거리에 반영합니다. 반영된 이후 증감거리는 삭제됩니다.
- **❼ 누적거리 점검** : 점검 후 누적거리를 입력해 점검 전 누적거리와 오차를 확인합니다. 선택된 최종 누적거리의 오차를 주행거리 비례기준으로 조정합니다.
- **❽ 표준연비일괄적용** : 선택된 차량의 유류비 단가 및 표준연비를 일괄적용합니다.
- **❾ 유류비계정설정** : 일반전표입력, 매입매출전표입력 메뉴에 입력된 계정과목, 적요명으로 불러오기 조건을 설정합니다.

## 9) 차량비용현황(업무용승용차)

> 메뉴위치 : 회계 보조장부Ⅰ ▶ 차량비용현황(업무용승용차)

### ▣ 개요
차량별, 부서/사원별 차량 관련 비용현황을 조회하는 메뉴입니다.

### ▣ 조회방법
- **❶ 기간** : 조회월을 선택하면 데이터는 누적으로 조회됩니다.
- **❷ 업무용 승용차** : F2 코드도움으로 조회하고자 하는 업무용 승용차(부서/사원)를 입력하거나, 전체 업무용 승용차(부서/사원)를 검색하고자 하는 경우 공란으로 조회합니다.
- **❸ 유형** : 기본선택은 9.업무용승용차(0~2)이며 회사차, 렌트, 리스 차량이 모두 선택되어 조회됩니다. 9.업무용승용차(0~2), 0.회사차, 1.렌트, 2.리스, 3.직원명의차량, 4.직원소유타인명의, 5.회사차(비영업용), 6.리스(비영업용), 7.렌트(비영업용), 8.전체
- **❹ 집계** : 하단에 해당 차량(사원), 전체 차량(사원)별로 각 비용현황에 금액이 반영됩니다. (미등록차량(사원)의 비용 금액은 전체차량(사원) 금액에 제외됩니다.)

### ▣ 편리한 기능(기능모음)
- **❶ 전표수정** : 선택한 데이터의 일반전표/매입매출전표의 입력된 데이터를 수정합니다.
- **❷ 원장조회** : 선택한 데이터의 원장 내용을 조회하며, 전표수정을 통해 수정이 가능합니다.
- **❸ 불러오기** : 전표 불러오기의 설정 구분 값(적요)명으로 해당 비용 전표들의 내용이 차량별(사원별)로 조회됩니다. 계정과목 설정보기를 클릭하면 비용 항목별 세팅된 계정과목을 확인할 수 있으며, 추가설정도 가능합니다.
- **❹ 데이터정렬** : 날짜순, 비용구분순, 계정과목순, 사원명순, 금액순, 적요순, 전표유형순으로 데이터를 정렬합니다.

## ★ BK 매핑·시사점

1. **관리항목별 보조원장** : 보조장부I은 관리항목(거래처·적요·차량·현장·외주)별 보조원장 모음 → 전표 라인의 관리차원/관리항목 입력값이 원천. BK가 라인 단위 차원/관리항목을 충실히 저장하면 추가 입력 없이 파생 가능.

2. **총계정관리원장** : 계정과목 × (거래처/현장/프로젝트/부서·사원) 교차 집계 + 원장조회·전표수정 드릴다운. BK 6 관리차원 피벗 원장으로 구현.

3. **건설업 현장 차원(공사대장·외주대장·전도금원장)** : 현장(SITE) 차원의 계약금액(현장등록 자동 반영)·기성액(계정과목 설정 기반 집계)·외주처/하도급 관리·전도금(현장코드 기반 가지급·정산·잔액). BK 6 관리차원 중 현장(SITE)과 거래처 유형(외주처)으로 직결. 기성액·외주비 집계 대상 계정은 사용자 설정 기반.

4. **업무용승용차 세무(운행기록부·차량비용현황)** : 운행기록부는 차량별 운행일지·업무사용비율 자동계산·누적거리 오차 안분·유류비 표준연비 적용까지 제공. 차량비용현황은 차량/사원별 비용 집계로 손금 한도 검토. BK는 차량을 관리항목/자산으로 모델링하고, 차량 유형(회사차/렌트/리스/직원명의 등 0~9 코드)과 업무사용비율을 세무 손금 계산 입력으로 채택.

5. **증빙 현황·부가세 대사** : 세금/계산서 발급·수취현황, 신용카드매출전표 발행현황(과세/영세율/면세/총괄 탭)은 부가세 신고 대사·증빙현황 보조부 → 08 부가세 주요신고서류(세금계산서합계표·카드매출 발행집계표)와 직접 연계.

6. **적요별원장** : 적요코드/적요명 기준 거래 집계 + 분개유형(현금/대체/결산) 필터. BK 적요코드 마스터 기반 분석 리포트 후보.

7. **공통 드릴·수정** : 거의 모든 보조장부가 원장조회·전표수정(F7) 드릴다운을 제공 → BK 보조장부 컴포넌트도 행→원전표 인라인 편집(권한·기간잠금 검증) 표준 적용.
