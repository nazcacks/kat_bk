## 자동전표 파이프라인 (★ BK 증빙 자동수집/자동분개 설계 근거)

```
수집정보등록(인증·연동 등록)
  → [전자세금계산서/전자계산서/신용카드/현금영수증/통장] 자료 수집
  → 자동분개(전표 생성)
  → 전표전송현황(전송/대사)
  → 국세청자료검증 및 사이트비교(정합성 검증)
```

## 수집정보등록 상세 (허브)

### 수집 대상 및 인증 방식
| # | 대상 | 인증/연동 |
|---|------|-----------|
| ❶ | 전자(세금)계산서 | 국세청 홈택스 공인인증서 등록 |
| ❷ | 현금영수증·신용카드(매입) | 홈택스 계정(ID/PW) |
| ❸ | 신용카드(매출) | 여신금융협회 계정 |
| ❹ | 통장 | 은행 선택 → 계좌구분·조회구분·계좌번호·ID/PW |
| ❺ | 신용카드(매입) | 카드사 선택 → 카드구분·카드번호·홈페이지 로그인 |

### 주요 기능
- 등록 정보 기준 「자료수집 및 자동분개」에서 자료 수집
- **매일 자동수집 + 실시간 수집**
- 수집정보 수정/삭제 가능(단, 삭제분 복원 불가)

### BK 시사점 (보안·설계)
- ⚠️ **자격증명 직접 보관·입력은 BK 금지정책 대상**. 홈택스/은행/카드사 ID·PW·인증서는 BK가 평문 저장하지 않고, **외부 스크래핑/오픈뱅킹·홈택스 API 위임 또는 password manager 위임** 방식으로 처리해야 함.
- 5단계 파이프라인은 BK 「증빙 자동수집 → 자동분개 → 검증」 모듈의 표준 흐름으로 채택.
- 통장 수집은 예금 보조부/현금흐름, 카드 수집은 카드 보조부와 연계.
- 「국세청자료검증·사이트비교」는 부가세 신고 전 **수집분-신고분 대사**의 핵심 → BK 세무 MRI/검증과 연계.
