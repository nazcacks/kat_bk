## 1) 수집정보등록

> 메뉴위치 : 회계 ▶ 자동전표처리 ▶ 수집정보등록

### ▣ 개요

자동전표처리에 필요한 전자세금계산서(계산서), 현금영수증, 신용카드 등 주요 거래증빙의 수집 정보를 등록하는 메뉴입니다. 발생한 거래내역을 수기로 입력하지 않고, 전표 메뉴에 자동 생성하는 자동전표 처리 기능을 사용하기 위한 수집정보를 등록하며, 등록된 수집정보를 통해 매일 자동수집 및 Scrapping Board에서 실시간으로 자료수집을 진행할 수 있습니다.

### ▣ 입력방법

자료수집을 위해 전자세금계산서(계산서), 현금영수증/신용카드(매입), 신용카드(매출), 통장(계좌), 신용카드(매입) 수집정보를 등록합니다. 입력된 정보를 기준으로 "자료수집 및 자동분개" 메뉴에서 각 메뉴별 자료를 수집할 수 있습니다.

#### ❶ 전자세금계산서/전자계산서 인증서 등록
전자(세금)계산서 자료 수집을 위해 국세청 홈택스 공인인증서를 등록합니다. 인증서가 정상 등록되면 국세청 홈택스에서 전자세금계산서와 전자계산서 거래 자료 수집이 가능합니다. (등록된 인증서의 이름과 서명정보는 암호화 되어 안전하게 관리됩니다.)

#### ❷ 현금영수증/신용카드(매입) 국세청 홈택스 계정(아이디, 비밀번호) 등록
국세청 홈택스 사이트에 등록된 아이디와 비밀번호를 입력하여 등록합니다. 국세청 홈택스 사이트에 가입하지 않았거나, 아이디와 비밀번호를 모르는 경우 "국세청 홈택스 계정정보"를 클릭하면 국세청 홈택스 사이트로 이동합니다.

#### ❸ 신용카드(매출) 여신금융협회 계정(아이디, 비밀번호) 등록
여신금융협회에 등록된 아이디와 비밀번호를 입력하여 등록합니다. 여신금융협회에 가입하지 않았거나, 아이디와 비밀번호를 모르는 경우 "여신금융협회 계정정보"를 클릭하면 여신금융협회 사이트로 이동되어 아이디와 비밀번호 찾기가 가능합니다.

#### ❹ 통장(계좌)
신규등록에 등록할 은행을 선택한 후 계좌구분(기업, 개인), 조회구분, 계좌번호, 계좌비밀번호, 아이디와 비밀번호를 입력하여 통장 계좌를 등록합니다. 등록된 통장 계좌는 수정 및 삭제할 수 있습니다. 단, 통장 계좌를 삭제하는 경우 이전에 수집된 거래 데이터는 유지되나 삭제된 수집정보는 복원할 수 없으니 삭제에 유의하시기 바랍니다.

- 계좌구분은 은행사이트에서 빠른조회, 간편조회 등 신청 시 등록한 기업/개인 구분을 선택합니다.
  - 기업 : 간편조회 등록 시 사업자등록번호로 신청한 경우 선택합니다.
  - 개인 : 간편조회 등록 시 주민등록번호로 신청한 경우 선택합니다.
- 조회구분은 은행사이트마다 통장내역 조회를 할 수 있는 서비스가 따로 있으므로 가입한 서비스를 정확히 선택합니다.

#### ❺ 신용카드(매입)
신규등록에 카드사를 선택한 후 카드구분(법인카드, 개인카드), 카드번호(숫자만 입력), 카드사 홈페이지 아이디와 비밀번호를 입력하여 등록합니다. 등록된 신용카드(매입) 정보는 수정하거나 변경, 삭제할 수 있습니다. 단, 등록된 카드 계정정보를 삭제하는 경우 이전에 수집된 거래 데이터는 유지되나, 삭제된 수집정보는 복원할 수 없으니 삭제에 유의하시기 바랍니다.

### ※ 참조 : 신용카드(매입) 등록 시 유의사항
- 동일한 카드사에 여러 카드번호를 입력한 경우 인증이 실패하면 더 이상 인증이 진행되지 않습니다.
- 농협카드사는 공인인증서를 통해야만 로그인이 가능하기 때문에 신용카드사로부터 데이터 자료수집을 할 수 없습니다.
- 자료수집이 가능한 카드사 : 국민, 신한, 비씨, 외환, 롯데, 현대, 삼성, 농협, 하나, 우리, 광주, 전북카드

## 2) 전자세금계산서

> 메뉴위치 : 회계 ▶ 자동전표처리 ▶ 전자세금계산서

### ▣ 개요
국세청 홈택스 사이트로부터 전자세금계산서(매출, 매입)를 자동으로 수집하여 분개를 추천하고, 추천된 분개를 확인 및 변경(수정)하여 매입매출전표입력 메뉴에 전송하는 메뉴입니다.

### ▣ 입력방법
**일자, 구분(1.매출, 2.매입), 거래처, 전표상태의 4가지 조회조건이 모두 입력되어야 자료 조회가 가능합니다.**

#### ❶ 일자
전자세금계산서 작성(거래)일자를 입력합니다.

#### ❷ 거래처명
- 조회된 사업자 정보(사업자번호)와 동일한 사업자 정보가 거래처등록 메뉴에 등록되어 있는 경우 거래처등록 메뉴의 거래처코드가 표시됩니다.
- 조회된 사업자번호가 거래처등록에 존재하지 않는 사업자번호인 경우 거래처코드를 자동으로 채번하여 표시되며, 거래처등록 메뉴에 자동으로 등록됩니다.
- 거래처등록 메뉴에 동일한 사업자번호가 2개 이상 존재하는 경우에는 첫 번째 입력된 거래처등록으로 표시되고, 해당 란은 주황색으로 표시됩니다. 이 때, 동일사업자번호 내에서 거래처코드 변경이 가능하며 거래처등록 메뉴의 사용여부를 체크하여 거래처 코드가 반영됩니다.

#### ❸ 유형
**매출유형**은 국세청 홈택스에서 수집한 유형을 보여주며, 비활성화되어 수정이 불가능합니다.
- 과세 : 공급가액과 세액이 존재하면 과세로 표시됩니다.
- 영세 : 세액이 '0'원이며, 홈택스에서 구분 값이 영세로 수집된 데이터들은 영세로 표시됩니다.

**매입유형**은 국세청 홈택스에서 수집한 유형을 보여주며 수정이 가능합니다. 과세에서 불공으로 수정 시 하단 분개의 부가세대급금 계정은 사라지고 금액은 큰 금액의 계정에 합산됩니다. 상대계정의 금액이 동일한 경우 첫 번째 순서의 계정에서 차감하여 생성됩니다. (영세와 수입은 수정 불가능, 과세에서 불공 수정가능, 불공에서 과세 수정가능)
- 과세 : 공급가액과 세액이 존재하면 과세로 간주하여 과세로 표시됩니다.
- 영세 : 세액이 '0'원이며, 홈택스에서 구분 값이 영세로 수집된 데이터들은 영세로 표시됩니다.
- 불공 : 거래처 품명 분석을 통해 인식이 가능한 데이터는 불공으로 표시됩니다. 과세에서 불공으로 유형 변경 시 불공제 사유선택 창이 활성화되며, 해당되는 사유를 선택합니다.
- 수입 : 홈택스에서 수입으로 구분되어 수집된 데이터는 수입으로 표시됩니다.

※ 불공유형에서 과세로 수정 시 하단 분개에 부가세대급금이 자동 생성됩니다.
※ 과세유형 12.영세, 52.영세, 55.수입 유형은 변경되지 않습니다.

#### ❹ 품명, 공급가액과 부가세
일자별, 거래 건별로 품명과 공급가액, 부가세가 표시됩니다.

#### ❺ 차변계정, 대변계정
거래처와 품명(적요)의 내용을 분석하여 50%를 초과하는 계정과목 가운데 확률이 가장 높은 계정과목을 추천합니다.
- 차변/대변 계정의 음영이 없고(흰색이고), 추천 계정이 있는 경우 : 계정과목 추천율이 100%인 경우
- 차변/대변 계정의 음영이 없고(흰색이고), 미추천인 경우 : 거래내역이 없거나 부정확하여 추천할 계정과목이 없는 경우
- 차변/대변 계정의 음영이 주황색이고, 추천 계정이 있는 경우 : 계정과목 추천율이 51% 이상인 경우
- 차변/대변 계정의 음영이 주황색이고, 미추천인 경우 : 계정과목으로 추천 되었으나 추천율이 50% 이하인 경우

#### ❻ 전표상태
- **확정가능** : 전표전송이 가능한 상태입니다.
- **전표확정** : 전표전송이 완료되어 확정된 상태로 전표의 내용이 변경되지 않으며, 매입매출전표입력 메뉴에서 해당 전표를 삭제하면 삭제전표로 변경됩니다.
- **확정제외** : 사용자가 임의로 전송을 제외한 상태의 전표이거나 승인번호 일치로 인해 전송 제외된 전표입니다. 전송이 제외된 전표는 반드시 확인한 후 전표전송 여부를 선택해야 합니다. 전표를 전송하지 않고 업무를 마무리 할 경우도 전송가능을 전송제외로 변경이 가능합니다.
- **중복전표** : 전표전송을 하지 않았으나 매입매출전표입력 메뉴에 전표가 이미 존재하는 것으로 의심되는 전표입니다. 중복전표는 전표전송이 되지 않습니다. (중복전표 의심기준: 일자, 사업자번호, 금액, 과세유형)
- **미추천** : 추천계정이 없는 경우 전표전송 상태는 미추천으로 표시되며, 미추천인 경우에도 분개없음으로 전송이 가능합니다. (전표전송 시 옵션에서 선택) 미추천인 상태에서는 바로 전표전송이 되지 않으며, 계정과목을 직접 입력하면 전표상태 값이 전송가능으로 변경되고 전표전송이 가능합니다.
- **삭제** : 전송된 전표를 매입매출전표입력 메뉴에서 삭제시 자동으로 변경되고, 전표가 임의적으로 삭제 또는 손상되어도 조회 시에 자동으로 삭제전표로 표시됩니다. 삭제전표는 전송가능 또는 전송제외로 전표상태 변경이 가능합니다.

#### ❼ 하단분개내역
전자세금계산서 거래에 대한 분개를 표시하며, 수정이 가능합니다.

## 3) 전자계산서

> 메뉴위치 : 회계 ▶ 자동전표처리 ▶ 전자계산서

### ▣ 개요
국세청 홈택스 사이트로부터 전자계산서(매출, 매입)를 자동으로 수집하여 분개를 추천하고, 추천된 분개를 매입매출전표입력 메뉴에 전송하는 메뉴입니다.

### ▣ 입력방법
**전자세금계산서와 동일하게 4가지 조회조건**(일자, 구분, 거래처, 전표상태)**을 모두 입력합니다. 상단부는 분개내역에 대한 하나의 전표로 작성일자, 거래처명, 품명, 공급가액 등의 확인이 가능하며, 하단부는 거래에 대한 분개 확인이 가능합니다.**

#### ❶ 일자
전자계산서 작성(거래)일자를 입력합니다.

#### ❷ 거래처명
- 조회된 사업자 정보(사업자번호)와 동일한 사업자 정보가 거래처등록 메뉴에 등록되어 있는 경우 거래처등록 메뉴의 거래처코드가 표시됩니다.
- 조회된 사업자번호가 거래처등록에 존재하지 않는 사업자번호인 경우 거래처코드를 자동으로 채번하여 표시되며, 거래처등록 메뉴에 자동으로 등록됩니다.
- 거래처등록 메뉴에 동일한 사업자번호가 2개 이상 존재하는 경우에는 첫 번째 입력된 거래처등록으로 표시되고, 해당 란은 주황색으로 표시됩니다. 이 때, 동일사업자번호 내에서 거래처코드 변경이 가능하며 거래처등록 메뉴의 사용여부를 체크하여 거래처 코드가 반영됩니다.

#### ❸ 품명, 공급가액
국세청 홈택스 사이트에서 불러온 품명을 보여주며, 품명과 하단분개 내역의 적요가 동일한 경우 품명 수정 시 하단 적요도 동일하게 수정됩니다. 부가가치세가 면제(면세)되므로 전자세금계산서와 달리 부가세 항목이 없습니다.

#### ❹ 차변계정, 대변계정
거래처와 품명(적요)의 내용을 분석하여 50%를 초과하는 계정과목 가운데 확률이 가장 높은 계정과목을 추천합니다.
- 차변/대변 계정의 음영이 없고(흰색이고), 추천 계정이 있는 경우 : 계정과목 추천율이 100%인 경우
- 차변/대변 계정의 음영이 없고(흰색이고), 미추천인 경우 : 거래내역이 없거나 부정확하여 추천할 계정과목이 없는 경우
- 차변/대변 계정의 음영이 주황색이고, 추천 계정이 있는 경우 : 계정과목 추천율이 51% 이상인 경우
- 차변/대변 계정의 음영이 주황색이고, 미추천인 경우 : 계정과목으로 추천되었으나 추천율이 50% 이하인 경우

#### ❺ 전표상태
- **확정가능** : 전표전송이 가능한 상태입니다.
- **전표확정** : 전표전송이 완료되어 확정된 상태로 전표의 내용이 변경되지 않으며, 매입매출전표입력 메뉴에서 해당 전표를 삭제하면 삭제전표로 변경됩니다.
- **확정제외** : 사용자가 임의로 전송을 제외한 상태의 전표이거나 승인번호 일치로 인해 전송 제외된 전표입니다. 전송이 제외된 전표는 반드시 확인한 후 전표전송 여부를 선택해야 합니다. 전표를 생성하지 않고 업무를 마무리할 경우도 전송가능을 전송제외로 변경이 가능합니다.
- **중복전표** : 전표전송을 하지 않았으나 매입매출전표입력 메뉴에 전표가 이미 존재하는 것으로 의심되는 전표입니다. 중복전표는 전표전송이 되지 않습니다. (중복전표 의심기준: 일자, 사업자번호, 금액, 과세유형)
- **미추천** : 추천계정이 없는 경우 전표전송 상태는 미추천으로 표시되며, 미추천인 경우에도 분개없음으로 전송이 가능합니다. 미추천인 상태에서는 바로 전표전송이 되지 않으며, 계정과목을 직접 입력하면 전표상태 값이 전송가능으로 변경되고 전표전송이 가능합니다.
- **삭제** : 전송된 전표를 매입매출전표입력 메뉴에서 삭제시 자동으로 변경되고, 전표가 임의적으로 삭제 또는 손상되어도 조회 시에 자동으로 삭제전표로 표시됩니다. 삭제전표는 전송가능 또는 전송제외로 전표상태 변경이 가능합니다.

#### ❻ 하단분개내역
전자계산서 거래에 대한 분개를 표시하며, 수정이 가능합니다.
- **코드/계정과목** : F2 코드도움 또는 C 아이콘으로 계정과목을 검색하여 입력합니다.

> 메뉴위치 : 회계 ▶ 자동전표처리 ▶ 신용카드

### ▣ 개요
매출, 매입 신용카드 자료를 여신금융협회(매출), 국세청과 각 카드사 홈페이지(매입)로부터 수집하여 분개를 추천하고, 추천된 분개를 확인 및 변경 후 전표로 전송하는 메뉴입니다.

### ▣ 입력방법

**❶ 일자** : 매출인 경우 승인일자(매출 신용카드 승인처리일자, 매출발생일), 매입인 경우 카드사가 VAN사로부터 승인된 카드 내역을 매입한 일자(사이트에서 수수료 및 입금예정액 확인 가능)의 거래를 조회합니다.

**❷ 자료** : 매출인 경우 여신금융협회, 엑셀을 선택하고, 매입인 경우 국세청, 카드사, 화물복지, 엑셀, 엑셀(국세청), 엑셀(화물복지) 중에서 선택합니다. (카드사는 수집정보등록 메뉴의 신용카드 매입에 등록하여 인증 성공한 자료인 경우 선택합니다.)

**❸ 체크박스** : 데이터를 전표 메뉴에 전송하거나 일괄 변경하는 경우에만 사용됩니다.

**❹ 카드사** : 매출 카드사와 유사한 카드사명이 거래처등록 메뉴 카드 탭에 입력되어 있는 경우에는 카드사 거래처코드를 자동으로 표시합니다. 유사한 카드사명이 없는 경우, 거래처등록 매입카드에 입력되어 있는 경우 카드사 코드로 자동 표시됩니다. (매출/매입 카드사 모두 없는 경우에는 빈 공란으로 표시합니다.)

**❺ 거래처** : 국세청 홈택스 사이트(매입)에서 불러온 거래처명을 표시합니다.

**❻ 품명/적요** : 품명과 하단분개 내역의 적요가 동일한 경우 품명 수정 시 하단 적요도 동일하게 수정됩니다. (단, 상단의 품명과 하단 적요가 동일한 경우에만 적용합니다.)

**❼ 구분(자료구분을 2.매입으로 하는 경우)** : 국세청 홈택스 사이트에서 제공하는 거래처 구분으로 변경하거나 입력할 수 없습니다. 법인, 일반, 면세, 간이로 표시됩니다.

**❽ 국내외(자료를 2.카드사로 하는 경우)** : 자료구분이 카드사인 경우에만 제공되며, 국내외 카드 사용내역에 관한 정보를 제공합니다.

**❾ 공급가액, 세액**
- 매출인 경우 합계금액만 수집이 가능하며, 기능모음의 '매출수수료분개 및 계정설정'에서 설정한 과세유형으로 자동 계산된 공급가액과 세액으로 표시합니다. 이때 합계금액을 1.1로 나누어 공급가액과 세액을 반영합니다.
- 매입인 경우, 수집된 공급가액과 세액을 표시합니다.

**❿ 국세청(매입)** : 국세청 홈택스 사이트에서 제공하는 공제/불공제 여부를 보여주며, 수정이 불가능합니다.

**⓫ 수수료, 입금액(매출)** : 여신금융협회 사이트에서 제공되는 금액으로 매출시점에서 해당 수수료 분개를 직접 반영할 경우, 매출수수료 분개 및 계정설정에서 수수료분개를 '1.매출시 수수료인식'으로 설정해야 수수료 분개 생성이 가능합니다. '2.입금시 수수료인식'으로 변경하면 수수료 분개가 하단분개에서 사라집니다.

**⓬ 전표유형(매입)** : 공제여부에 따라 일반전표입력과 매입매출전표입력 메뉴에 반영을 체크합니다.
- 일반전표 : 일반으로 전표가 발생합니다.
- 매출전표
  - 17.카과 : 공급가액과 세액이 존재합니다.
  - 18.과면/19.카영 : 세액이 '0'원이고, 면세와 영세로 수집된 데이터들입니다.
- 매입전표
  - 57.카과 : 공급가액과 세액이 존재합니다.
  - 58.카면/59.카영 : 세액이 '0'원이고, 면세와 영세로 수집된 데이터들입니다.
  - 63.화물 : 국세청에서 화물복지카드로 수집된 데이터들입니다.

**⓭ 상대계정** : 공급가액의 계정과목을 의미합니다. 거래처와 품명의 분석을 통해 50%를 초과하는 확률이 가장 높은 계정과목을 추천합니다.
- 흰색(계정이 있는 경우) : 추천율이 100%인 계정과목 / (미추천인 경우): 거래내역이 존재하지 않아 추천할 계정과목이 없는 경우
- 주황색(계정이 있는 경우) : 추천된 계정과목 중 51%이상인 계정과목 / (미추천인 경우): 계정추천은 되었으나 추천된 계정들의 추천율이 50% 이하인 계정과목

**⓮ 전표상태**
- 확정가능 : 전표전송이 가능한 상태
- 전표확정 : 전표전송이 완료된 상태로 수정할 수 없고 상태 값이나 전표의 내용이 변경되지 않으며, 일반 전표/매입매출전표입력 메뉴에서 해당 전표를 삭제하면 삭제전표로 변경됩니다.
- 확정제외 : 전송 제외된 전표는 반드시 확인 후 전표전송 여부를 선택해야 합니다. 전표를 전송하지 않고 업무를 마무리할 경우 전송가능을 전송제외로 변경이 가능합니다.
- 중복전표 : 전표전송은 하지 않았으나 일반전표입력이나 매입매출전표입력 메뉴에 전표가 있는 상태이며 중복전표는 전표전송되지 않습니다. (중복전표 의심 기준 - 일자, 카드번호, 금액, 과세유형/매입은 사업자등록번호, 카드거래처코드 포함)
- 미추천 : 추천계정이 없을 때 전표전송 상태는 미추천으로 표시되며, 분개없음으로 전송이 가능합니다. 미추천인 상태에서는 바로 전표전송이 되지 않으며, 계정과목을 직접 입력하면 전표상태 값이 전송가능으로 변경되고 전표전송이 가능합니다.
- 삭제전표 : 전송된 전표를 전표 메뉴에서 삭제 시 표시되며, 삭제전표는 전송가능 또는 전송제외로 전표 상태 변경이 가능합니다.

**⓯ 하단 분개 내역** : 신용카드 거래에 대한 분개를 표시하며, 수정이 가능합니다.
- 코드/계정과목 : F2 코드도움 또는 C 아이콘으로 계정과목을 검색하여 입력합니다.
- 거래처 : F2 코드도움 또는 C 아이콘으로 거래처를 검색하여 입력하거나, 코드를 직접 입력합니다.
- 적요 : 기본적으로 상단의 품명이 동일하게 적요에 적용되며, 계정과목별로 적요 수정이 가능합니다. 상단의 품명과 하단의 적요가 동일한 경우 상단의 품명을 수정하면 하단 적요도 수정 반영됩니다.

### ▣ 편리한 기능(기능모음)

**❶ 전표전송** : 전송할 데이터를 체크한 후 전표전송을 누르면 전표입력 메뉴에 데이터를 전송할 수 있습니다.
- 전표상태가 '전송가능'인 전표 → 매입매출전표 : '분개포함/분개없음'을 선택하여 전송할 수 있습니다. '분개없음'을 선택하여 전표전송시 상단 내역만 매입매출전표입력 메뉴에 전송됩니다.
- 전표상태가 '미추천'인 전표 → 체크박스를 체크 후 전송하면 매입매출전표입력 메뉴에 하단분개가 없는 '분개없음'으로 전송됩니다.
- ※ 체크박스를 체크를 해야 전표전송이 이루어지며 매입매출전표입력 메뉴에 전송이 가능합니다. 전송하려는 전표에 대차차액이 발생하는 경우 전표전송 버튼 옆에 [차액분개보기]버튼이 표시됩니다.

**❷ 매출수수료분개 및 계정설정** : 신용카드 매출자료를 처음 수집한 후 메뉴 실행 시 자동 실행되는 기능입니다. 값을 설정하여 추후 변경 사항이 있기 전까지는 설정한 값을 유지하고, 이후에 특별한 사유가 있어 설정을 변경하는 경우 변경된 내용이 수집된 모든 자료에 반영되므로 주의해야 합니다.
- 과세 유형 : 과세(과세사업자→카과, 공급가액·세액 각각 표시) / 면세(면세사업자→카면, 공급가액만 표시) / 간이(간이과세자→공급대가로 표시)
- 수수료 분개 : 매출시 수수료 인식(카드사 수수료를 매출하는 시점에 반영) / 입금시 수수료 인식(카드사 수수료를 입금하는 시점에 반영)
- 수수료 계정 : 원가경비별 신용카드 매출 수수료 계정과목을 설정합니다.(ex: 531, 631, 731, 831)

**❸ 채권계정카드사설정** : 신용카드 매출 중 '채권계정카드사 설정'에 등록된 계정과목들의 거래처는 하단분개에 거래처가 카드사로 적용됩니다. 상단 거래처를 변경해도 하단 거래처는 변경되지 않습니다. 신용카드 매출 거래시 거래처 코드를 입력하면, 하단 분개에도 반영됩니다.

**❹ 전표전송 설정**
- 일반전표입력 메뉴로 전송 시 거래처등록메뉴에 미등록 사업자번호 존재 시 거래처 생성 설정 : '여'로 설정하면 사업자번호가 등록되어 있지 않은 전표는 거래처 등록에 등록됩니다.
- 매입매출 전표 전송 시 분개 유형은 환경설정 > 매입매출 전표입력 자동설정관리 반영 : '여'로 설정하면 환경설정>회계>4.매입매출 전표입력 자동설정관리에 설정된 계정과목과 비교하여 분개 유형을 전송합니다.
  - ※ 설정된 계정과목과 동일한 경우 [외상]
  - ※ 설정된 계정과목과 동일하지 않은 경우 : (1) 상대계정이 현금인 경우 [현금] (2) 상대계정이 현금이 아닐 경우 [혼합]
- 일반전표입력 메뉴 전송 시 차변 또는 대변이 101현금일 경우 입금 또는 출금으로 전송 : '여'로 설정하면 일반전표입력 메뉴에 전표전송 시 차변 또는 대변 계정이 101현금일 경우 입금 또는 출금으로 전송됩니다.

**❺ 매출증빙병합** : 여러 건의 전표를 1장의 전표로 전송할 경우 사용하는 기능으로, 전표상태가 전송가능/미추천인 자료인 경우에만 병합이 가능합니다. 체크박스에 체크한 후 기능을 실행해야 하며, 전표가 합쳐지면 새로운 전표가 생성되고, 일자에는 파란색으로 표시됩니다. 병합된 전표를 다시 원상태로 하려면 병합된 전표를 선택한 뒤 기능모음에 '매출증빙분리'를 실행하면 됩니다.

**❻ 매입신용카드거래처등록** : 메뉴를 최초 실행하는 경우 새로 수집된 카드사가 있으면 자동으로 기능이 실행됩니다. 카드사 코드와 카드명은 자료조회를 위한 필수입력 사항이며, 수집된 카드사와 거래처 등록에 등록된 카드 거래처를 매칭 수집된 카드거래 내역을 확인할 수 있습니다. 수집한 카드사의 구분을 설정하여 구분이 신용 외에는 결제 계정과목과 결제 거래처를 입력해 주어야 합니다.

**❼ 수집 카드사 정보** : 여신금융협회의 가맹점 매출거래정보 통합조회 시스템이 조회할 수 있는 카드사(제휴은행 포함)가 한정되어 있으므로, 광주, 전북, 제주, 씨티, 수협 은행에서 자체 관리하는 가맹점 정보는 조회 대상에서 제외됩니다. (참고 : 여신금융협회 홈 > FAQ http://www.cardsales.or.kr/faq.do?method=list)

**❽ 자동분개시 과세유형 설정** : 차량유지비, 접대비 계정과목의 경우 전표를 "일반전표입력" 메뉴에 생성 또는 이미 전송된 매입 전표를 기준으로 추천된 과세유형으로 생성할 수 있습니다.

**❾ 세액변경** : 세액란을 수정가능한 상태로 변경합니다.(세액이 공란일 경우 수정이 가능합니다.)

**❿ 공제유형별 집계** : 부가가치세 과세기간에 따라 카드사, 엑셀(사용자설정)을 제외한 자료의 공제유형별 집계를 보여줍니다.

**⓫ 부가세신고 요약** : 부가가치세 과세기간에 따라 전자(세금)계산서, 신용카드, 현금영수증의 매출, 매입 자료를 요약해서 확인할 수 있습니다.

**⓬ 수집한 국세청자료집계** : 선택한 부가가치세 과세기간에 따라 국세청에서 수집된 자료를 카드번호별로 확인할 수 있습니다.

**⓭ 휴폐업조회** : 거래처 사업자번호를 국세청기준으로 조회하여, 거래처의 휴업 또는 폐업 여부를 조회합니다.

**⓮ 오류검증** : 현재 조회기간을 기준으로 검사항목을 체크하여 입력된 자동전표 데이터를 검사합니다.

**⓯ 매출엑셀서식 내려받기/불러오기** : 업로드할 수 있는 엑셀서식을 내려받고, 해당 서식에 매출데이터를 입력하여 불러오기합니다.

**⓰ 매입엑셀서식 내려받기/불러오기** : 업로드할 수 있는 엑셀서식을 내려받고, 해당 서식에 매입데이터를 입력하여 불러오기합니다.

**⓱ 매입엑셀서식(국세청) 불러오기** : 국세청 홈택스 사이트 > 조회발급 > 사업용신용카드 조회 > 매입세액 공제 확인/변경 > 엑셀내려받기 또는 국세청 홈택스 사이트 > 화물운전자복지카드 조회 > 매입세액 공제 확인/변경 > 엑셀내려받기를 통해서 내려받은 엑셀파일을 불러오기합니다.

**⓲ 조회한 데이터를 엑셀로 변환** : 현재 조회한 데이터를 엑셀로 변환하여 다운로드합니다.

### ▣ 참고사항(색상정보)
- ✜ 면세확인 : 공급가액의 10%와 세액을 비교하여 그 차이가 2원을 초과하고 공급가액과 세액 합산금액의 원단위가 '0'원이 아닌 데이터들을 표시합니다.
- ✜ 공급자(면세) : 매입거래처 중에 면세사업자를 메뉴화면에 표시해줍니다.
- ✜ 세액 확인 : 공급가액의 10%와 세액을 비교하여 차이가 2원을 초과하고, 공급가액과 세액 합산금액의 원단위가 '0'이 아닌 데이터들을 메뉴화면에 표시해줍니다.
- ✜ 고정자산 확인 : 수선비, 차량유지비, 소모품비, 사무용품비 등 자산비용계정 중 3,000,000원 이상인 데이터들을 메뉴화면에 표시해줍니다.
- ✜ 전자중복 : 신용카드에 동일한 사업자번호로 수집된 자료가 있는 경우 메뉴화면에 표시해줍니다.
- ✜ 공급자 확인 : 면세사업자가 아닌 불공인 사업자(여객, 철도공사) 등을 메뉴화면에 표시해줍니다.
- ✜ 체크카드 : 통장거래내역의 내용을 분석하여 체크카드를 사용했을 경우 메뉴화면에 표시해줍니다.

## 5) 현금영수증

> 메뉴위치 : 회계 ▶ 자동전표처리 ▶ 현금영수증

### ▣ 개요
국세청 홈택스 홈페이지에서 현금영수증 매출, 매입내역을 수집하여 분개를 추천하고, 추천된 분개를 확인 및 변경한 후 전표 메뉴에 전송하는 메뉴입니다.

### ▣ 입력방법

**❶ 조회방법** : 작성일자, 구분, 거래처, 전표상태 값을 입력한 후 조회를 클릭합니다.

**❷ 체크박스** : 체크된 데이터를 전표 메뉴에 전송하거나, 데이터를 일괄 변경하는 경우에만 사용이 가능합니다.

**❸ 일자** : 수집된 데이터의 현금영수증 승인일자가 조회됩니다.

**❹ 거래처**
- 매출: 매출 거래처의 등록된 거래처코드이며, 최초 수집 시 사이트 정보로 확인이 불가능하기 때문에 해당 코드는 모두 공란으로 표시됩니다.
- 매입: 거래처등록 메뉴의 사업자등록번호와 매칭하여 거래처코드 및 거래처 명을 보여줍니다. 동일한 사업자등록번호가 존재하는 경우에는 가장 최근에 입력한 사업자등록번호의 거래처를 보여주고, 거래처 코드에 음영(주황색)으로 표시됩니다.

**❺ 구분** : 법인, 일반, 면세, 간이 중 자동 표시됩니다. 국세청 홈택스 사이트에서 제공하는 거래처 구분 값으로 변경하거나 입력할 수 없습니다.

**❻ 품명/적요** : 기존 거래자료 분석을 통해 추천되며, 품명은 직접 입력이 가능합니다. 하단 분개의 적요가 동일한 경우 품명을 수정하면 하단 적요도 동일하게 수정됩니다.

**❼ 국세청(매입)** : 국세청 홈택스 홈페이지에서 제공하는 공제/불공제 여부가 반영되며, 수정이 불가능합니다.

**❽ 유형(전표유형)**
- 일반: 일반전표입력 메뉴에 전표를 반영합니다.
- 매출: 매입매출전표입력 메뉴에 아래의 선택된 항목으로 전표를 반영합니다.
  - 22.현과: 공급가액과 세액이 존재합니다.
  - 23.현면/24.현영: 세액이 '0'원이며 면세, 영세로 수집된 데이터들입니다.
- 매입
  - 61.현과: 공급가액과 세액이 존재합니다.
  - 62.현면: 면세로 수집된 데이터이며, 세액이 '0'원입니다.

**❾ 차변계정/대변계정** : 공급가액의 계정과목을 의미하며, 거래처와 품명의 분석을 통해 50%를 초과하는 확률이 가장 높은 계정과목을 추천합니다.
- 흰색 : 계정과목이 있는 경우(추천율 100%) / 미추천(거래내역 없어 추천할 계정과목 없음)
- 주황색 : 계정과목이 있는 경우(51% 이상) / 미추천(추천은 되었으나 추천율 50% 이하)

**❿ 전표상태**
- 확정가능: 전표전송이 가능한 상태
- 전표확정: 전표전송이 완료된 상태로, 상태 값이나 전표의 내용수정이 불가능합니다. 일반전표/매입매출전표입력 메뉴에서 해당 전표를 삭제하면 삭제전표로 변경됩니다.
- 확정제외: 전송 제외된 전표는 반드시 확인한 후 전표전송 여부를 선택해야 합니다.
- 중복전표: 전표전송은 하지 않았으나 일반전표입력이나 매입매출전표입력 메뉴에 전표가 있는 상태이며, 전표전송이 불가능합니다. (중복전표 의심기준-일자, 금액, 구분의 매입은 사업자번호, 카드 거래처코드 포함)
- 미추천: 추천할 계정과목이 없을 경우 미추천으로 표시되며, 미추천도 분개없음으로 전송이 가능합니다. 미추천인 상태에서 계정과목을 직접 입력하면 전표상태값이 전송가능으로 변경되고 전표전송이 가능합니다.
- 삭제전표: 전송된 전표를 일반/매입매출전표입력 메뉴에서 삭제하는 경우 표시되며, 삭제전표는 전송가능 또는 전송제외로 전표상태 변경이 가능합니다.

**⓫ 계정과목/거래처** : 전표상태가 미추천인 경우 F2 코드도움 또는 C 아이콘으로 계정과목과 거래처를 검색하여 입력합니다.

**⓬ 적요** : 기본적으로 상단의 품명이 동일하게 적요에 적용되며, 계정과목별로 적요 수정이 가능합니다. 상단의 품명과 하단의 적요가 동일한 경우 상단의 품명을 수정하면 하단 적요도 수정 반영됩니다.

### ▣ 편리한 기능(기능모음)

**❶ 전표전송** : 전송할 데이터를 체크한 후 전표전송을 누르면 전표입력 메뉴에 데이터를 전송할 수 있습니다.
- 전표상태가 '전송가능'인 전표 → 매입매출전표 : '분개포함/분개없음'을 선택하여 전송할 수 있습니다. '분개없음'을 선택하여 전표전송시 상단 내역만 매입매출전표입력 메뉴에 전송됩니다.
- 전표상태가 '미추천'인 전표 → 체크박스를 체크 후 전송하면 매입매출전표입력 메뉴에 하단분개가 없는 '분개없음'으로 전송됩니다.
- ※ 체크박스를 체크를 해야 전표전송이 이루어지며 매입매출전표입력 메뉴에 전송이 가능합니다. 전송하려는 전표에 대차차액이 발생하는 경우 전표전송 버튼 옆에 [차액분개보기]버튼이 표시됩니다.

**❷ 자동분개시 과세유형 설정** : 차량유지비, 접대비 계정과목의 경우 전표를 "일반전표입력" 메뉴에 생성할 수 있습니다.

**❸ 전표전송 설정**
- 일반전표입력/매입매출전표입력 메뉴로 전송 시 거래처등록메뉴에 미등록 사업자번호 존재 시 거래처 생성설정: '여'로 설정하면 사업자번호가 등록되어 있지 않은 전표는 거래처 등록에 등록됩니다.
- 매입매출 전표 전송 시 분개 유형은 매입매출전표입력 > 기능모음의 분개계정과목설정 반영: '여'로 설정하면 환경설정>회계>매입매출전표입력 자동설정관리에 설정된 계정과목과 비교하여 분개유형을 전송합니다.
  - ※ 설정된 계정과목과 동일한 경우 [현금]
  - ※ 설정된 계정과목과 동일하지 않은 경우 : (1) 상대계정이 현금인 경우 [현금] (2) 상대계정이 현금이 아닐 경우 [혼합]
- 일반전표입력 메뉴 전송 시 차변 또는 대변이 101현금일 경우 입금 또는 출금으로 전송: '여'로 설정하면 일반전표입력 메뉴에 전표전송 시 차변 또는 대변 계정이 101현금일 경우 입금 또는 출금으로 전송됩니다.

**❹ 매출증빙병합** : 여러 건의 전표를 1장의 전표로 전송할 경우 사용하는 기능으로, 전표상태가 전송가능/미추천인 자료인 경우에만 병합이 가능합니다. 체크박스에 체크한 후 기능을 실행해야 하며, 전표가 합쳐지면 새로운 전표가 생성되고, 일자에는 파란색으로 표시됩니다. 병합된 전표를 다시 원상태로 하려면 병합된 전표를 선택한 뒤 기능모음에 '매출증빙분리'를 실행하면 됩니다.

**❺ 엑셀자료 불러오기(매출엑셀)** : 국세청 홈택스 사이트 > 조회발급 > 현금영수증 조회 > 매출내역조회 > 엑셀 내려받기를 통해서 내려받은 엑셀파일을 매출의 유형(과세. 간이/명세)을 선택하고 불러옵니다.

**❻ 엑셀자료 불러오기(매입엑셀)** : 국세청 홈택스 사이트 > 현금영수증 수정 > 세액 공제 확인/변경 > 엑셀 내려받기를 통해서 내려받은 엑셀파일을 가맹점명 반영 옵션을 선택하고 불러옵니다.

**❼ 조회한 데이터를 엑셀로 변환** : 현재 조회한 데이터를 엑셀로 변환하여 다운로드합니다.

**❽ 오류검증** : 현재 조회기간을 기준으로 검사항목을 체크하여 입력된 자동전표 데이터를 검사합니다.

## 6) 통장

> 메뉴위치 : 회계 ▶ 자동전표처리 ▶ 통장

### ▣ 개요
각 금융기관 홈페이지에서 통장거래내역을 자동으로 수집하여 거래처코드를 찾아 분개를 추천하며, 추천된 분개를 확인 및 변경한 후 일반전표입력 메뉴에 전송하는 메뉴입니다.

### ▣ 입력방법

#### ❶ 조회조건
거래일자, 구분, 금융거래처, 전표상태 값을 입력하고 데이터를 조회합니다.

#### ❷ 통장이미지영역
각 은행 사이트에서 불러온 데이터를 통장 이미지로 나타내며, 이미지 화면은 수정이 불가능합니다. 거래 내역을 더블 클릭하면 통장>통장대조>상세로 이동되며, 원장과 비교해서 볼 수 있습니다.
※ 이미지와 우측 상단의 통장 전표전송의 거래내역을 선택하면 자동으로 같은 거래내역으로 매칭해주며, 하단 분개도 선택된 거래의 분개가 보여집니다.

**통장전표전송에서는 거래처, 전표적요, 상대계정, 전표상태 값을 입력할 수 있습니다.**

- **❶ 체크박스** : 통장내용 등을 전표적요로 일괄변경, 통장계정을 일괄변경, 송금수수료 일괄변경, 전표전송, 인쇄시에 사용가능합니다. (* 체크카드 : 통장거래내역의 [내용]을 분석하여 체크카드를 사용했을 경우 표시합니다.)
- **❷ 거래처명** : 통장의 적요를 분석하여 거래처를 자동으로 찾아서 입력해주며, 추천 거래처가 2개 이상인 경우 주황색으로 표시됩니다. 색상 표시가 되어 있는 코드나 비어 있는 거래처명은 F2 코드도움 또는 C 아이콘으로 거래처를 검색하여 입력할 수 있습니다. 거래처는 상대계정 거래처로 자동 부여되며, 하단분개에 자동으로 반영됩니다. 단, 하단 상대계정 거래처를 수정하여 상단의 거래처도 수정된 경우 이후 상단 거래처를 수정해도 하단 거래처에 반영되지 않습니다.
- **❸ 전표적요** : 통장의 적요를 그대로 복사해주며 하단분개의 적요에도 동일하게 적용됩니다. 단, 하단 상대계정 적요를 수정하여 상단의 전표 적요와 다를 경우 상단 전표 적요를 수정해도 하단 적요에 반영되지 않습니다.
- **❹ 상대계정** : 공급가액의 계정과목을 의미합니다. 거래처와 품명의 분석을 통해 50%를 초과하는 확률이 가장 높은 계정과목을 추천해줍니다.
  - 흰색(계정이 있는 경우): 추천율 100% / (미추천): 거래내역 없어 추천할 계정과목 없음
  - 오렌지색(계정이 있는 경우): 51% 이상 / (미추천): 추천은 되었으나 추천율 50% 이하
- **❺ 전표상태**
  - 확정가능 : 전표전송이 가능한 상태
  - 전표확정 : 전표전송이 완료된 상태로 수정할 수 없고 상태 값이나 전표의 내용이 변경되지 않으며, 일반전표입력 메뉴에서 해당 전표를 삭제하면 삭제전표로 변경됩니다.
  - 확정제외 : 체크카드의 경우는 자동으로 전송제외 상태로 적용되며 전송제외된 전표는 반드시 확인 후 전표전송 여부를 선택해야 합니다.
  - 중복전표 : 전표전송은 하지 않았으나 일반전표입력 메뉴에 전표가 있는 상태이며, 중복전표는 전송되지 않습니다.(중복전표 의심기준-동일 통장거래처코드의 일자, 금액/매입은 사업자번호, 카드 거래처코드 포함)
  - 미추천 : 추천계정이 없을 때 전표전송 상태는 미추천으로 표시합니다.(미추천도 분개없음으로 전송 가능합니다.) 미추천인 상태에서는 바로 전표전송이 되지 않으며, 계정과목을 직접 입력하면 전표 상태 값이 전송가능으로 변경되고 전표전송이 가능합니다.
  - 삭제전표 : 전송된 전표를 일반전표입력 메뉴에서 삭제하는 경우 표시되며, 삭제전표는 전송가능 또는 전송제외로 전표상태 변경이 가능합니다.

**하단부는 분개를 확인 및 수정합니다.**
- ❶ 구분 : 전표의 구분 3.차변, 4.대변 중 선택해서 입력이 가능합니다.
- ❷ 계정과목 : F2 코드도움 또는 C 아이콘으로 계정과목을 조회 후 입력합니다.
- ❸ 차변/대변 : 구분이 출금이나 차변일 경우 차변 란에 입력 및 수정이 가능하며, 구분이 입금이나 대변일 경우 대변 란에 입력 및 수정이 가능합니다.
- ❹ 거래처명 : F2 코드도움 또는 C 아이콘으로 거래처를 선택하거나, 거래처명을 직접 입력할 수 있습니다.
- ❺ 적요 : 상단의 전표 적요가 동일하게 적용되며, F2 코드도움 또는 C 아이콘으로 적요를 선택하거나 직접 입력 가능합니다.

### ▣ 편리한 기능(기능모음)

**❶ 매입매출** : 매입매출을 클릭하면 상단의 내역을 하단의 매입매출장과 거래처원장으로 나누어 데이터를 보여줍니다.
- 매입매출장(좌측하단)
  - 거래처가 공란일 경우 금액구간 설정 또는 동일금액(합계금액)으로 매입매출장을 조회합니다.
  - 입금 거래를 조회할 경우 매출(-매입)에 대한 유형이 조회되며, 출금 거래를 조회할 경우 매입(-매출)에 대한 유형이 조회됩니다.
  - 금액구간 : 설정 구간에 따라 매입매출 금액을 조회합니다.
  - 매입매출장의 해당 거래처를 더블 클릭할 경우 상단의 거래처에 반영됩니다.
- 거래처원장(우측하단)
  - 매입매출장에 선택된 거래처의 거래처원장이 조회됩니다.
  - 상단에 거래처가 있는 경우에는 상단 거래처의 거래처원장이 조회됩니다.

**❷ 통장대조** : 프로그램에 입력된 통장내역과 거래처원장을 비교하여 중복되는 데이터와 맞지 않는 데이터를 대조할 수 있습니다.
- 좌측 상단의 통장이미지는 전표전송 탭의 이미지와 같고, 전표상태 값을 변경 및 전표전송이 가능합니다. 통장 전표전송 탭의 전표상태 값과 동일하며, 상태 값이 전송가능인 것은 전표전송이 가능합니다.
- 우측 상단의 통장원장은 프로그램의 통장의 거래처원장이 조회됩니다.
- 좌측 하단의 분개내역은 전표전송 탭의 하단 분개와 동일하며, 거래내역에 대한 분개를 할 수 있습니다.

**❸ 전표전송설정**
1. 일반전표입력메뉴 전송 시 차변 또는 대변이 101현금일 경우 입금 또는 출금으로 전송 : '여'로 설정하면 일반전표입력 메뉴로 전표전송시 차변 또는 대변 계정이 101현금일 경우 입금 또는 출금으로 전송됩니다.
2. 통장 전표 전송 시 경비성 계정은 카드계좌 '여'로 적용되고, '부'로 선택 시 체크카드 사용분만 카드계좌에 '여'로 전송됩니다 : '1.여'로 설정하여 전표전송 시에 체크카드 사용분과 경비성 계정 사용분은 카드계좌에 '여'로 전송됩니다.

**❹ 전표전송** : 데이터 체크 후 전표전송을 누르면 체크한 데이터 중 전표상태 값이 가능인 건만 집계를 표시해 주는 창이 활성화됩니다. 전표상태가 가능인 것만 전표전송이 되며, 그 외의 상태 값은 전표전송이 되지 않습니다. 전표전송 시 전송하려는 일자에 잔액이 맞지 않을 경우 확인 메시지가 띄워집니다.

**❺ 송금수수료일괄분개** : 송금수수료 일괄변경은 거래내역이 출금이고 미전송인 건에 대해 적용되며, 통장 전표전송 화면에서 일괄분개할 내역을 체크한 뒤 송금수수료 일괄분개를 선택하고 송금수수료 금액과 계정과목을 입력하면 일괄 분개가 됩니다.
- 차변 계정이 하나일 경우, 차변의 계정 금액에서 수수료를 차감하고 차감된 금액을 송금수수료 계정으로 분개 처리됩니다.
- 차변 계정이 2계정 이상일 경우, 차변의 금액이 가장 큰 계정에서 차감되어 분개처리 됩니다. (가장 큰 계정의 금액이 같을 경우 금액이 같은 계정 중에 분개 순서가 가장 위의 것으로 차감되어 분개 처리됩니다.)
- ※ 입금 거래내역, 차변 계정이 없는 경우, 차변 계정이 송금수수료 금액보다 적거나 같은 경우는 수수료 분개 처리가 되지 않습니다.

**❻ 거래처명을 전표적요로 일괄변경** : 통장에 전표적요를 거래처명으로 일괄변경합니다.

**❼ 통장 내용을 전표 적요로 일괄 반영** : 이미지의 내용/취급점/비고란의 내용을 전표적요로 복사하는 기능입니다. 기능 선택 적용시 기존에 입력되어있는 전표의 적요는 삭제되고, 선택된 내용/취급점/비고의 내용이 복사됩니다.

**❽ 전표적요를 거래처명으로 일괄변경** : 통장 거래처명을 전표적요로 일괄변경합니다. 일괄반영시 거래처코드는 삭제되고 거래처명이 변경됩니다.

**❾ 일괄 변경** : 거래처명, 적요, 계정과목 등의 데이터를 일괄 변경합니다.

**❿ 적요코드 일괄변경** : 선택한 계정과목에 대하여 적요코드 및 적요명을 일괄로 변경합니다.

**⓫ 통장계정을 일괄 변경** : 통장의 계정과목을 다른 계정과목으로 일괄 변경합니다.

**⓬ 엑셀서식 내려받기** : 엑셀 업로드할 수 있는 기본양식을 내려받는 기능입니다.

**⓭ 엑셀서식 불러오기** : 내려받은 엑셀서식에 데이터를 입력하여 설정에 따라 업로드하는 기능입니다.

**⓮ 카드매출 입금 예정액 조회** : 여신금융협회에서 수집된 카드 매출분에 대해 일자별 입금예정액을 확인합니다.

## 7) 자료수집현황

> 메뉴위치 : 회계 ▶ 자동전표처리 ▶ 자료수집현황

### ▣ 개요
자료수집 및 자동분개 메뉴에서 수집한 자료를 수집일자 별로 조회할 수 있는 메뉴입니다. 각 수집일자를 체크하면 하단에 수집기간에 따른 자료수집 실패여부와 실패사유를 조회할 수 있습니다.

### ▣ 조회방법
- ❶ 최근 자료수집 요청일시 순으로 상단에 정렬됩니다.
- ❷ 각 자료수집 요청일시를 선택하면 하단에 수집기간에 따른 자료수집 결과와 실패사유를 확인할 수 있습니다.
- ❸ 각 일자별 수집자료는 홈페이지에서 수집된 전체 자료가 아닌 기존자료와 중복된 부분을 제외하고, 새롭게 수집되는 거래자료의 자동전표 수를 표시합니다.

- ❷ 최종 거래자료의 수집일자와 최종 거래일자를 쉽게 조회할 수 있습니다.
- ❸ 최종 거래일에는 각 메뉴별로 수집된 거래자료의 마지막 거래일이 표시됩니다.

## 9) 국세청자료검증 및 사이트비교

### ▣ 개요
매입매출전표입력 메뉴에 입력된 데이터와 국세청 홈택스에 신고되어 있는 데이터를 비교하여 오류를 검증하는 메뉴입니다. 국세청에 신고된 데이터와 WEHAGO SmartA CE 매입매출전표입력 메뉴에 입력된 데이터를 비교하여 누락된 세금계산서는 추가로 입력하고, 전자세금계산서가 아닌 데이터는 전자구분 값을 제외하여 정확한 신고를 돕는 검증 기능을 수행합니다.

- **❶ 불러오기** : 각 거래증빙이 수집되는 홈페이지에서 데이터를 불러옵니다.

## ★ BK 매핑·시사점

1. **5단계 파이프라인 채택** : 수집정보등록 → (전자세금계산서/전자계산서/신용카드/현금영수증/통장) 자료수집·자동분개 → 전표전송 → 전표전송현황 → 국세청자료검증·사이트비교. BK 「증빙 자동수집 → 자동분개 → 검증」 모듈의 표준 흐름으로 채택.

2. **⚠️ 자격증명 보안(BK 금지정책 정합)** : 홈택스 공인인증서·홈택스 계정(ID/PW)·여신금융협회 계정·은행 계좌 ID/PW·카드사 로그인은 SmartA가 직접 등록·암호화 보관. BK는 평문 자격증명 저장·입력을 금지하므로 **오픈뱅킹/홈택스 Open API 위임, 외부 스크래핑 서비스 위임, 또는 password manager 위임**으로 구현. 수집정보 삭제 시 거래 데이터는 유지되나 수집정보는 복원 불가 → BK도 커넥터 삭제와 수집 데이터 보존을 분리.

3. **계정 추천 신뢰도 모델** : 거래처+품명 분석 기반 추천율(100%=흰색, 51~99%=주황+추천, ≤50%=주황+미추천, 0=흰색+미추천). BK 자동분개 AI의 신뢰도 UI(색상 음영)와 미추천→수기입력→전송가능 상태전이를 직접 차용.

4. **전표상태 머신** : 확정가능 / 전표확정 / 확정제외 / 중복전표 / 미추천 / 삭제(전표). 중복전표 의심기준(일자·사업자번호·금액·과세유형, 카드/통장은 카드번호·통장거래처코드 추가)을 BK 자동전표 큐의 dedup 규칙으로 채택. 전표확정 후 전표 메뉴에서 삭제 시 삭제전표로 역전이.

5. **과세유형 자동판정 + 수정 제약** : 과세↔불공 상호 수정 가능(불공 전환 시 부가세대급금 제거·큰 금액 계정 합산, 과세 전환 시 부가세대급금 자동생성), 영세·수입은 수정 불가, 12.영세/52.영세/55.수입 고정. BK taxType 전이 규칙으로 반영.

6. **자동분개 설정 규칙** : 환경설정 자동설정관리 계정과 비교 → 동일(외상)/상대계정 현금(현금)/그 외(혼합). 차변·대변 101현금이면 입금/출금 전환. 미등록 사업자번호 자동 거래처 생성 옵션. 수량·단가 전송 옵션. BK 자동분개 정책 파라미터로 모델링.

7. **증빙별 특화 기능** : 가산세 계산(전자세금계산서), 매출수수료분개·채권계정카드사·매출증빙병합·휴폐업조회·공제유형별 집계(신용카드), 송금수수료 일괄분개·통장대조·매입매출 대사(통장). BK 증빙 모듈별 부가기능 후보.

8. **검증·대사** : 국세청자료검증·사이트비교는 신고 전 수집분-신고분 대사의 핵심 → BK 세무 MRI/검증과 연계. 오류검증(조회기간 검사항목), 부가세신고 요약(전자세금계산서·카드·현금영수증 매출/매입 요약)도 검증 대시보드 후보.
