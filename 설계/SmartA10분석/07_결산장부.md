## 1) 영수증수취명세서

> 메뉴위치 : 회계 결산장부 ▶ 영수증수취명세서

### ▣ 개요
법인세법, 소득세법에 의한 거래금액이 3만원(접대비 1만원) 초과분에 대해 적격증빙서류를 미수취한 경우 작성하는 메뉴.

### ▣ 조회방법
일반전표입력 메뉴에서 카드계좌 사용여부에 "**부**"로 선택하고, "**1.명세서 제출대상거래**", "**2.명세서 제출 제외대상내역**"으로 되어 있는 데이터는 지출증빙서류의 미수취로 보아 영수증수취명세서**(2)** 서식에 데이터가 집계됩니다.

- **❶ 거래금액** — 영수증수취 대상금액을 입력. 3만원 초과금액부터 입력가능하며, 접대비계정의 경우는 만원초과금액을 입력합니다.
- **❷ 구분코드** — 세금계산서·계산서·신용카드 등을 사용하지 아니한 3만원 초과 거래분 중 명세서제출제외 대상에 해당하는 구분코드를 사유별로 구분하여 입력. 명세서 제출대상에 해당될 때에는 구분코드를 입력하지 않음. 명세서 제출대상과 제출제외대상에 대한 합계를 메뉴에서 확인할 수 있으며 출력시에도 반영됩니다.
- **❸ 영수증수취명세서(2) 불러오기** — 일반전표입력 메뉴에서 카드계좌 사용여부에 "부"로 선택하고, "1.명세서 제출대상거래", "2.명세서 제출 제외 대상내역"으로 되어 있는 데이터를 지출증빙서류의 미수취로 보아 영수증수취명세서(2) 반영. 영수증수취명세서(2)에서 작성된 내역 중에서 영수증 제출대상/미제출대상에 의해 영수증 수취명세서(1) 화면으로 내역이 집계됩니다. 명세서(2) 불러오기 버튼을 이용해 내역을 새로 반영할 수 있으며 3만원초과 거래분 명세서제출제외대상내역에 사용자가 직접 편집도 가능합니다. 일반전표입력에서 카드계좌 '부'의 0.해당사항 없음으로 되어 있는 데이터를 불러옵니다.

### ▣ 편리한 기능(기능모음)
- **❶ 타 세무대리인 등록** — 타 세무대리인이 작성한 경우 성명과 관리번호를 입력. 부가가치세 전자신고 메뉴의 제출자등록이 "세무 대리인"으로 설정되어 있다면 성명과 관리번호를 자동으로 반영. 출력물 인쇄시 세무대리인내역에 반영.
- **❷ 원장조회** — 선택한 데이터의 원장 내용을 조회하며, 전표수정 버튼을 통해 수정 가능.
- **❸ 데이터정렬** — 거래일자순, 공급자상호순, 공급자명순, 구분코드순, 계정과목순으로 정렬.
- **❹ 전표수정** — 해당사항없음(기타) 탭에서 전표 수정 가능.
- **➎ 주민번호 별표 사용설정** — 주민번호 별표 사용여부를 선택.

## 2) 경비등의송금명세서

> 메뉴위치 : 회계 결산장부 ▶ 영수증수취명세서

### ▣ 개요
법인이 「법인세법 시행령」 제41조 제2항 제2호 또는 3만원을 초과하는 재화 또는 용역을 공급받고, 그 대가를 「법인세법 시행규칙」 제79조 제10호에 따라 금융기관을 통하여 지급하는 거래에 대하여 작성. 일반전표입력메뉴의 카드계좌 여부에서 "부"를 선택한 후 3.명세서제출제외대상내역에서 구분33 송금명세서제출분을 선택한 데이터를 경비등의 송금명세서 메뉴에 자동으로 반영.

### ▣ 입력방법
- **❶ 거래일자** — 송금한 거래일자 입력.
- **❷ 법인명(상호)** — 송금한 업체의 상호 입력.
- **❸ 성명** — 대표자 성명 입력.
- **❹ 사업자(주민)등록번호** — 사업자등록번호 또는 주민등록번호 입력.
- **❺ 거래내역/거래금액** — 송금한 내역 및 거래금액 입력.
- **❻ 송금일자** — 송금일자 입력.
- **❼ 은행명/계좌번호** — F2 코드도움 또는 C 아이콘으로 은행 코드 입력, 송금한 계좌번호 입력.

### ▣ 편리한 기능(기능모음)
- **❶ 불러오기** — 사용자가 직접 입력한 데이터를 제외한 전표 데이터를 다시 불러오거나, 추가 입력한 데이터를 포함한 모든 데이터를 삭제하고 전표 데이터를 불러옵니다.
- **❷ 작성요령** — 경비등 송금명세서의 작성요령 확인.

## 3) 지출증명서류합계표

> 메뉴위치 : 회계 결산장부 ▶ 지출증명서류합계표

### ▣ 개요
지출증빙서류합계표는 법인세법 제116조 제2항에 따른 지출증명서류를 수취해야 하는 거래(예: 유형자산, 무형자산, 재고자산, 비용)가 발생한 경우 작성하는 메뉴로, 신고용·관리용·현황 탭으로 구성.

### ▣ 입력방법

**표준대차대조표 계정과목별 지출증명서류 수취금액**
- **❶ 표준재무제표 계정과목** — 당해 사업년도에서 취득한 유형·무형자산에 대해 입력. 코드도움시 표준분류에 의한 표준계정과목코드를 반영하여 작성.
- **❷ 지출증빙서류 수취금액** — 당해 사업년도에서 취득한 유형·무형자산의 지출증명수취금액 입력. 각 자산별로 신용카드, 현금영수증, 세금계산서, 계산서 중 선택하여 금액 입력.
- **❸ 수취제외대상금액** — 당해 사업년도에서 취득한 유형·무형자산의 지출 중 수취제외대상금액 입력.

**표준손익계산서 계정과목별 지출증명서류 수취금액**
- **❶ 표준재무제표 계정과목** — 당해 사업년도에서 계상한 비용에 대해 입력. 코드도움시 표준분류에 의한 비용코드가 조회되며, 취득한 내역에 맞는 표준계정과목코드를 반영하여 작성.
- **❷ 지출증빙서류 수취금액** — 당해 사업년도에서 계상한 비용의 지출증명수취금액 입력. 각 비용별로 신용카드, 현금영수증, 세금계산서, 계산서 중 선택하여 금액 입력.
- **❸ 수취제외대상금액** — 당해 사업년도에서 계상한 비용 중 수취제외대상금액 입력.

**표준손익계산서 부속명세서 계정과목별 지출증명서류 수취금액**
- **❶ 표준재무제표 계정과목** — 당해 사업년도에서 계상한 비용의 지출증명수취금액 입력. 각 비용별로 신용카드, 현금영수증, 세금계산서, 계산서 중 선택하여 금액 입력. 구분 란에는 해당부속명세서[제조, 공사, 임대, 분양, 운송, 기타]에 따라 구분코드 입력.
- **❷ 지출증빙서류 수취금액** — 당해 사업년도에서 계상한 비용의 지출증명수취금액 입력. 각 비용별로 신용카드, 현금영수증, 세금계산서, 계산서 중 선택하여 금액 입력.
- **❸ 수취제외대상금액** — 당해 사업년도에서 계상한 비용 중 수취제외대상금액 입력.

법인유형은 일반법인 / 금융·보험·증권업법인으로 표기되며 표준재무제표의 법인유형에 저장된 값으로 반영됩니다. 일반법인과 금융법인 일 때 각 표준에 맞는 계정과목코드가 반영됩니다.

**관리용 탭** — 전표입력에서 새로불러오기로 불러온 데이터를 조회하는 화면. 해당사항없음 수취제외 포함으로 조회시 전표에서 카드사용여부가 '해당사항없음'으로 표기된 거래의 데이터를 수취제외 대상금액에 포함하여 조회.

**현황 탭** — 관리용 탭에 불러온 지출증명서류수취금액을 각 증명서류 구분별로 상세조회. 표준대차대조표, 표준손익계산서, 표준부속명세서를 선택하고 각 증명서류 구분을 선택하여 증명서류별로 상세조회.

### ▣ 편리한 기능(기능모음)
- **❶ 불러오기** — 전표에 입력된 계정과목과 표준대차대조표·표준손익계산서·표준부속명세서 표준코드와 연결되어 일반법인/금융·보험·증권업법인 유형에 따라 전표입력에서 해당 내역을 불러옵니다.
- **❷ 원장조회** — 화면에 조회된 자료 중 특정 계정과목의 세부내용을 확인하고자 하는 경우 사용.
- **❸ 일괄삭제** — 신고용 탭에 입력되어 있는 데이터를 일괄삭제.

## 5) 기간별손익계산서

> 메뉴위치 : 회계 결산장부 ▶ 기간별손익계산서

### ▣ 개요
각 기간별 매출, 매입, 노무비, 제조경비, 판매비와 관리비, 영업외 수익, 영업외 비용의 추이를 조회 및 출력하는 메뉴로 월별, 분기별, 반기별, 월별전년대비, 분기별전년대비, 반기별전년대비 탭으로 구성.

### ▣ 조회방법
- **❶ 일자** — 조회하고자 하는 월, 분기, 반기 기간 입력.
- **❷ 구분** — 손익계산서/포괄손익계산서를 조회할 때 사용. 포괄손익계산서 조회는 환경설정 - 중소기업회계기준 적용여부가 "0.사용안함"인 경우 조회 가능.
- **❸ 월별** — 각 월별 손익계산서를 1월~12월까지 비교식으로 조회·출력.
- **❹ 분기별** — 각 분기별 손익계산서를 비교식으로 조회·출력.
- **❺ 반기별** — 각 반기별 손익계산서를 비교식으로 조회·출력.
- **❻ 월별/전년대비** — 조회한 월별 손익계산서를 전기/당기로 구분하여 반영, 비교식으로 조회·출력.
- **❼ 분기/전년대비** — 조회한 분기별 손익계산서를 전기/당기로 구분하여 반영, 비교식으로 조회·출력.
- **❽ 반기/전년대비** — 상반기/하반기의 손익계산서를 비교식으로 조회·출력.

### ▣ 편리한 기능(기능모음)
- **❶ 유형설정** — 중단사업 사용여부 선택. 중단사업의 충족요건: ▪ 사업의 중단을 목표로 수립된 단일계획에 따라 기업의 일부를 일괄 매각/기업분할 방식으로 처분하거나 자산·부채를 분할하여 처분·상환·사업 포기. ▪ 주요사업별 또는 지역별 단위로 구분 가능, 재무보고 목적상 별도로 식별 가능.

## 6) 기간별원가명세서

> 메뉴위치 : 회계 결산장부 ▶ 기간별원가명세서

### ▣ 개요
각 기간별 원가명세서를 조회 및 출력하는 메뉴로 월별, 분기별, 반기별, 월별 전년대비, 분기 전년대비, 반기전년대비의 탭으로 구성. 원가명세서 메뉴에 입력된 계정과목에 준하여 월별 데이터를 조회.

### ▣ 조회방법
- **❶ 일자** — 조회하고자 하는 월, 분기, 반기 기간 입력.
- **❷ 구분** — 매출원가코드 및 계정과목에서 설정한 원가설정에 따라 구분 선택.
- **❸ 월별** — 각 월별 원가명세서를 1월~12월까지 비교식으로 조회·출력.
- **❹ 분기별** — 각 분기별 원가명세서를 비교식으로 조회·출력.
- **❺ 반기별** — 각 반기별 원가명세서를 비교식으로 조회·출력.
- **❻ 월별/전년대비** — 조회한 월별 원가명세서를 전기/당기로 구분하여 반영, 비교식 조회·출력.
- **❼ 분기/전년대비** — 조회한 분기별 원가명세서를 전기/당기로 구분하여 반영, 비교식 조회·출력.
- **❽ 반기/전년대비** — 상반기/하반기의 원가명세서를 비교식으로 조회·출력.

### ▣ 편리한 기능(기능모음)
- **❶ 원가설정** — 매출원가를 설정하고 원가경비코드를 연결하여 조회. 계정과목 및 적요등록에서 미리 설정해 놓은 원가설정으로 자동 조회되며, 기능모음에서 직접 편집 가능.
- **❷ 제목편집** — 대분류 명칭을 직접 편집. 편집된 대분류명은 화면 및 출력물에 반영.
- **❸ 원장조회** — 특정 계정과목 세부내용 확인. 더블 클릭 또는 [원장조회] 클릭 시 해당계정 원장메뉴 실행.
- **❹ 계정과목 코드보기** — 계정과목 코드를 메뉴화면에 표시.

## 7) 중단사업손익계산서

> 메뉴위치 : 회계 결산장부 ▶ 중단사업손익계산서

### ▣ 개요
중단사업에 대한 손익 항목을 조회 및 출력하는 메뉴로 중단된 사업의 손익을 계산하고자 하는 경우 사용. 환경설정 메뉴의 중소기업회계기준 적용이 "사용안함"으로 설정되어 있을 때 사용가능하며, 중단사업코드설정에서 부서 또는 프로젝트 중에 하나를 설정하면 관련 데이터만 반영.

### ▣ 입력방법
- **❶ 기간** — 중단 손익현황의 조회하고자 하는 월 선택.
- **❷ 중단부서/중단 프로젝트** — 환경설정 메뉴의 중단사업코드에서 설정한 부서 혹은 프로젝트 별로 손익항목 반영.

### ▣ 편리한 기능(기능모음)

- **❸ 법인세효과** — 법인세율 입력 후 법인세 효과 계산. 세율부분은 사용자 수정 가능, 입력한 세율에 따라 법인세 효과 계산.
- **❹ 계정과목 코드보기** — 계정과목 코드를 메뉴화면에 표시.
- **❺ 원장조회** — 특정 계정과목 세부내용 확인. 더블 클릭 또는 [원장조회] 클릭 시 해당계정 원장메뉴 실행.

## ★ BK 매핑·시사점

1. **영수증수취명세서 = 적격증빙 미수취 가산세 관리.** 핵심 트리거는 전표 라인의 **카드계좌 사용여부 '부' + 명세서구분(0.해당없음/1.제출대상/2.제출제외/3.송금명세서제출분)** 메타. BK는 비용·자산 전표 라인에 `legalEvidenceFlag`(적격증빙 수취 여부)와 `evidenceExclusionCode`(제출제외 사유코드)를 두어, 3만원(접대비 1만원) 초과 거래를 자동 분류하고 가산세 리스크를 집계한다.
2. **경비등의송금명세서**는 영수증수취명세서의 '구분33 송금명세서제출분'에서 파생 → BK는 동일 evidence 메타 위에 송금정보(거래일자·상호·사업자번호·송금일자·은행·계좌)를 부속 명세로 자동 생성. 은행연동 송금 내역과 매칭 가능.
3. **지출증명서류합계표 = 표준재무제표 계정과목 매핑 기반 집계.** 회사 계정과목 → 표준 BS/PL/부속명세서 코드 매핑 후, 증빙유형(신용카드/현금영수증/세금계산서/계산서)별 수취금액과 수취제외금액을 3개 구획(자산취득/비용/부속명세서)으로 집계. BK는 06의 통합계정매핑과 별도로 **표준계정과목 매핑 테이블**(일반법인/금융법인 분기)을 두고, 전표 증빙유형 메타를 자동 합산하는 파이프라인으로 구현. 법인세 신고 부속서류 자동 산출.
4. **기간별손익계산서·기간별원가명세서 = 동일 손익/원가 데이터의 시계열 피벗 뷰.** 월/분기/반기 × 당기/전년대비의 6탭 패턴 → BK 보고서 엔진은 단일 집계 데이터셋에 기간 그레인(월/분기/반기)과 비교 차원(당기/전기)을 파라미터로 부여하는 OLAP 스타일 피벗으로 구현. 포괄손익은 중소기업 회계기준 미적용 시에만 노출(06과 동일 환경설정 분기).
5. **중단사업 손익계산서·원가명세서 = 부서 또는 프로젝트 차원 필터 + 손익/원가 대체분개 전송.** 중단사업코드(부서 or 프로젝트)로 데이터를 필터링하고, 법인세효과(세율 입력)를 반영하며, 전표추가 시 **차분(이전 분개 금액 − 현재 조회 금액) 방식으로 멱등 전송**(조회일 말일자) → BK는 6 관리차원(부서/프로젝트 등) 필터를 보고서·자동분개에 공통 적용하고, 재전송 시 델타 분개로 중복을 방지하는 패턴을 채택. 환경설정 중단사업 차원(부서/프로젝트 택1) 토글 필요.
6. **전 메뉴 공통 '불러오기/원장조회/계정과목 코드보기' 패턴** → BK 결산장부 보고서들은 (a) 전표 데이터 재집계(불러오기), (b) 드릴다운 원장+전표수정(F8), (c) 코드 표기 토글을 공통 UX 컴포넌트로 제공.
