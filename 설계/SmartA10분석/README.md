## 4. BK 설계 반영 핵심 시사점 (총괄)

1. **전표 이원화** — Smart A는 *부가세 무관* 거래를 「일반전표입력」, *부가세 신고 대상* 거래를 「매입매출전표입력」으로 분리한다. BK도 동일 원칙(JV-01 vs JV-04/05)을 채택. (→ 01)
2. **전표 구분 6종(더존식)** — 1.출금 2.입금 3.차변 4.대변 5.결차 6.결대. BK `JournalLine.entryCode`(PAYMENT/RECEIPT/DR/CR/CLOSING_DR/CLOSING_CR)와 1:1. 결차/결대 오용 시 재무제표 미반영 경고 필요. (→ 01)
3. **자동전표 파이프라인** — 수집정보등록(인증)→자료수집→자동분개→전표전송→검증의 5단계. BK의 증빙 자동수집/자동분개 설계 근거. (→ 02)
4. **6 관리차원 = 부문 원가/손익의 토대** — 보조장부II 전체가 현장/프로젝트/부서 축의 원가·손익 안분·집계. BK DimensionConfig(사업장·코스트센터·부서·프로젝트·현장·사원)와 정합. (→ 05)
5. **결산자동분개** — 재고·감가상각·대손·퇴직급여·법인세를 금액 입력만으로 자동분개 생성. BK 결산 모듈 설계 근거. (→ 06)
6. **부가세 신고서식 체계** — 주요신고서류+업종별+공통첨부+영세율 4계층. 다업종 SaaS는 업종별 첨부서식 노출을 테넌트 업종으로 동적 결정해야 함. (→ 08~11)
