## 1) 공제받지못할매입세액명세서

메뉴위치 : 회계 [부가가치세] 첨부서류 ▶ 공제받지못할매입세액명세서

### ▣ 개요

"매입세액 불공제 대상 세금계산서의 내역을 작성하는 부가가치세 부속서류"로, 매입매출전표입력 메뉴에서 유형을 54.불공으로 입력하고, 불공제 사유를 선택한 데이터가 자동 반영되며, 직접 입력도 가능합니다.

### ▣ 입력방법

**공제받지 못할 매입세액 내역**

"공제받지 못할 매입세액 내역란에는 매입매출전표입력 메뉴에서 유형을 54.불공으로 선택하고, 불공제 사유별 입력데이터를 자동반영"합니다.

❶ **필요적 기재사항 누락**
필요적 기재사항(공급자의 사업자등록번호 누락 등)이 누락된 매입세금계산서의 매수, 공급가액, 세액을 입력합니다.

❷ **사업과 직접 관련 없는 지출**
업무와 관련 없는 자산을 취득, 관리함으로써 발생되는 유지비, 접대비 등의 지출 외 해당 매입세금계산서의 매수, 공급가액, 세액을 입력합니다.

❸ **비영업용 소형 승용 자동차구입 및 유지**
지프형 승용차, 소형자동차의 구입 등의 매입세금계산서 및 이와 관련된 유지비 등의 매입세금계산서 매수, 공급가액, 세액을 입력합니다.

❹ **접대비 및 이와 유사한 비용 관련**
접대비와 관련된 비용 등의 매수, 공급가액, 세액을 입력합니다.

❺ **면세사업 관련**
면세사업에 사용되는 재화, 용역을 공급 받은 경우, 토지취득과 관련된 해당매입세금계산서의 매수, 공급가액, 세액을 입력합니다.

❻ **토지의 자본적 지출관련**
토지 취득에 해당되는 자본적 지출비용을 입력합니다.

❼ **사업자등록 전 매입세액**
사업자등록 신청일 20일전에 수취한 매입세금계산서의 매수, 공급가액, 세액을 입력합니다.

❽ **금, 구리 스크랩 거래계좌 미사용 관련 매입세액**
금, 구리스크랩 거래계좌를 사용하지 않은 거래의 매수, 공급가액, 세액을 입력합니다.

**공통매입세액 안분계산**

"예정신고시 과세기간의 총공급가액에서 면세사업의 공급가액이 차지하는 비율에 공통매입세액을 곱한 금액이 면세사업관련 매입세액으로 불공제 처리"됩니다.

❶ 상단 기능모음의 불러오기를 클릭하거나 직접 계산식을 선택하고 입력합니다.

❷ 불러오기 시에는 매입매출전표입력 메뉴에서 유형을 54.불공으로 선택하고, 사유구분을 5.공통매입세액 안분계산서 분으로 예정신고기간에 입력한 데이터가 반영되며, 직접 입력도 가능합니다.

❸ 공통매입 공급가액, 공통매입세액, 면세공급가액, 총공급가액을 직접 입력하면 불공제매입세액이 자동 계산됩니다.

**공통매입세액 정산**

"확정신고시 예정신고기간에 공통매입세액을 안분계산한 경우 확정신고기간에 공급가액의 비율이 달라지므로, 과세기간 전체의 공급가액 비율로 공통매입세액을 재계산"합니다.

❶ 상단 기능모음의 불러오기를 클릭하거나 직접 계산식을 선택하고 입력합니다.

❷ 불러오기 시에는 매입매출전표입력 메뉴에서 유형을 54.불공으로 선택하고, 사유구분을 5.공통매입세액 안분 계산서 분으로 확정신고기간에 입력한 데이터가 반영되며, 사용자가 이 금액을 참고하여 입력합니다.

❸ 면세공급가액, 총공급가액, 기 불공제매입세액을 입력하면, 가산 또는 공제되는 매입세액이 자동 계산됩니다.

**납부 또는 환급세액 재계산**

"감가상각자산에 대한 공통매입세액 안분계산 후 면세비율이 증가, 감소되는 경우 납부세액환급액계산 규정에 따라 가산 또는 공제되는 매입세액을 계산"합니다.

✜ 면세비율이 5퍼센트 이상 차이가 나는 경우 납부세액 또는 환급세액을 다시 계산하여 확정신고시 신고, 납부해야 합니다.(면세비율의 차이가 5/100 이상인 경우에만 적용)

### ▣ 편리한 기능(기능모음)

❶ **불러오기**
"매입매출전표입력 메뉴에서 입력한 유형(54.불공) 데이터를 새로 불러옵니다."
- 공제받지 못할 매입세액 내역: 공제받지 못할 매입세액 탭에 데이터를 불러오기 유무를 체크합니다.
- 공통매입세액 안분계산식 설정: 공통매입세액 안분계산 탭에 데이터를 불러오기 유무를 체크하며 계산식 기준을 설정 가능합니다.
- 공통매입세액 정산계산식 설정: 공통매입세액 정산 탭에 데이터를 불러오기 유무를 체크하며 계산식 기준을 설정 가능합니다.
- 납부세액 또는 환급세액 재계산: 납부 또는 환급세액 재계산 탭에 데이터를 불러오기 유무를 체크합니다.

❷ **예정누락**
사용자가 직접 입력하여 예정신고기간의 누락분 데이터를 공제받지 못할 매입세액 내역 탭에 반영합니다.

❸ **신고일자(가이드 영역)**
해당 서식을 신고하기 위한 신고일자를 입력합니다. "조회기간의 마지막 월 기준으로 다음달 25일로 자동반영 되며 사용자가 수정 가능"합니다.

## 2) 건물등감가상각자산취득명세서

메뉴위치 : 회계 [부가가치세] 첨부서류 ▶ 건물등감가상각자산취득명세서

### ▣ 개요

"사업자가 사업설비를 신설, 취득, 확장 또는 증축하는 경우" 부가가치세 신고 시 조기환급을 받기 위해 작성하는 부가가치세 부속서류입니다. 매입매출전표입력 메뉴에서 유형을 51.과세, 52.영세, 54.불공, 55.수입, 57.카과로 입력하고, 분개 시 고정자산 계정과목으로 입력한 데이터가 반영됩니다.

### ▣ 입력방법

**❶ 기간**
조회하고자 하는 과세기간을 입력하거나 선택합니다.

**❷ 불러오기**
상단 기능모음의 불러오기를 클릭한 다음 보조 화면에서 항목별 고정자산 계정과목코드를 입력하면 매입매출전표입력 메뉴에서 설정된 계정과목코드의 전표 데이터를 불러옵니다.

**❸ 취득내역**
거래처별 감가상각자산 취득명세에서 직접 입력하거나, 전표데이터를 불러왔을 경우 자산구분에 따라 집계되어 보여지는 영역이며, 기능모음의 편집 기능으로 사용자가 직접 입력도 가능합니다.

**❹ 거래처별 감가상각자산 취득명세**
취득한 고정자산을 사용자가 직접 입력하는 영역이며, 상단 기능모음의 '불러오기' 실행 시 조회기간에 포함되는 전표 데이터가 반영됩니다.

### ▣ 편리한 기능(기능모음)

**❶ 예정누락**
"매입매출전표입력 메뉴에서 예정신고 누락분으로 설정한 전표 데이터를 조회 및 인쇄합니다."

**❷ 구분별 보기**
전체, 세금계산서, 신용카드 사용자가 조회하고자 하는 조건을 설정하여 데이터를 조회합니다.

**❸ 불러오기**
고정자산 구분별로 계정과목을 설정하고, 설정한 계정과목에 맞는 매입매출전표 데이터를 불러옵니다.

**❹ 편집**
상단 취득내역을 사용자가 직접 편집합니다.

**❺ 신고일자(가이드 영역)**
해당 서식을 신고하기 위한 신고일자를 입력합니다. 조회기간의 마지막 월 기준으로 다음달 25일로 자동 반영되며, 사용자가 수정 가능합니다.

## 3) 대손세액공제신고서

메뉴위치 : 회계 [부가가치세] 첨부서류 ▶ 대손세액공제신고서

### ▣ 개요

대손이나 대손 변제가 발생한 경우 작성하는 부가가치세 부속서류입니다. 사업자가 재화 또는 용역을 공급한 후 거래상대방의 부도, 파산 등으로 채권 및 그와 관련된 부가가치세의 전부 또는 일부를 회수할 수 없는 경우 대손이 확정되는 날이 속하는 과세기간의 매출세액에서 차감합니다.

### ▣ 입력방법 — (대손세액공제)

**❶ 기간**
대손이 발생한 기간을 입력하거나 선택 가능합니다.

**❷ 대손사유**
대손이 발생한 사유를 선택합니다.

**❸ 대손기준일**
대손 발생기준 년월일을 직접 입력합니다. 대손사유 5.부도(6월 되는 날)를 선택하면 6개월을 계산하여 반영합니다.

**❹ 대손확정일**
대손 확정년월일을 직접 입력합니다.

**❺ 대손금액**
공급가액과 세액을 포함한 대손확정금액을 입력합니다.

**❻ 대손세액**
입력된 대손금액에 공제율 10/110을 적용하여 자동 계산됩니다.

**❼ 상호명/사업자등록번호/주민등록번호/성명**
상호명에서 입력하고자 하는 상호명 두 글자를 입력하거나, F2코드도움 또는 C아이콘 클릭으로 상호명을 검색 후 입력합니다. 상호명을 알고 있는 경우 직접 입력할 수 있습니다.

**❽ 신고일자(가이드 영역)**
해당 서식을 신고하기 위한 신고일자를 입력합니다. 조회기간의 마지막 월 기준으로 다음달 25일로 자동 반영되며, 사용자가 수정 가능합니다.

### ▣ 입력방법 — (대손변제)

**❶ 기간**
대손 변제가 이루어진 기간을 입력하거나 선택 가능합니다.

**❷ 대손변제일**
대손 변제년월일을 직접 입력합니다.

**❸ 변제금액**
공급가액과 세액을 포함한 대손변제금액을 입력합니다.

**❹ 변제세액**
입력된 변제금액에 공제율 10/110을 적용하여 자동 계산됩니다.

**❺ 상호명/사업자등록번호/주민등록번호/성명**
상호명에서 입력하고자 하는 상호명 두 글자 이상을 입력하거나, F2코드도움 또는 C아이콘 클릭으로 상호명을 검색 후 입력합니다. 상호명을 알고 있는 경우 직접 입력할 수 있습니다.

**❻ 변제사유**
대손 변제의 사유를 직접 입력합니다.

**❼ 신고일자(가이드 영역)**
해당 서식을 신고하기 위한 신고일자를 입력합니다. 조회기간의 마지막 월 기준으로 다음달 25일로 자동 반영되며, 사용자가 수정 가능합니다.

## 4) 사업장별부가세납부환급신고명세

메뉴위치 : 회계 [부가가치세] 첨부서류 ▶ 사업장별부가세납부환급신고명세

### ▣ 개요

총괄납부 승인을 받은 사업자는 부가가치세신고시 "사업장별 부가가치세 과세표준 및 납부(환급)세액 신고서를 작성 후 첨부"해야 합니다. 총괄납부승인을 얻은 사업자는 주된 사업장의 신고내역과 종된 사업자의 신고내역을 통합해서 작성합니다.

### ▣ 입력방법

**❶ 기간**
조회기간을 입력하거나 선택합니다.

**❷ 총괄납부번호**
관할세무서에서 부여 받은 총괄납부 승인번호를 입력합니다.

**❸ 사업장명, 주소, 사업자등록번호**
F2코드도움 및 C아이콘 클릭으로 "회사등록 메뉴에 입력된 주된 사업장을 불러오며 사업장명과 사업장주소, 사업자등록번호가 반영"됩니다. 그 외의 종 된 사업장의 내역은 직접 입력합니다.

**❹ 과세표준 및 세율**
회사등록 메뉴에 입력된 주된 사업장의 내역은 부가가치세 신고서 메뉴에 저장된 내용을 자동 반영합니다. 종된 사업장의 매출 및 매입의 과세표준과 세율을 입력하면 세액이 자동으로 계산되며, 가산세 및 공제세액 및 내부거래는 직접 입력합니다.

### ▣ 편리한 기능(기능모음)

**❶ 불러오기**
조회한 기간의 부가가치세신고서 메뉴에 저장된 데이터를 새로 불러옵니다. "불러오기를 하면 기존에 입력된 데이터는 삭제"됩니다.

## 5) 사업자단위과세사업장별부가세

메뉴위치 : 회계 [부가가치세] 첨부서류 ▶ 사업자단위과세사업장별부가세

### ▣ 개요

사업자단위과세 승인을 받은 사업자가 작성하는 부가가치세 부속서류입니다. 회사등록 메뉴의 본지점 여부, 종사업장번호를 설정한 경우 데이터가 자동 반영됩니다.

### ▣ 입력방법

**❶ 기간**
신고기간 또는 조회기간을 입력하거나 ▼를 클릭하여 선택합니다.

**❷ 사업장명, 사업자등록번호**
F2코드도움이나 C아이콘 클릭으로 회사등록 메뉴에 사업자단위과세자로 설정된 사업장을 불러오거나 직접입력 또는 수정이 가능합니다.

**❸ 적용번호**
국세청에서 승인 받은 단위사업장적용번호를 입력하는 란으로, 회사등록에 메뉴에 입력된 종사업장번호를 자동 반영하며, 직접 입력 및 수정이 가능합니다. 단, 사업자단위과세등록번호가 없는 주 사업장인 경우 0000으로 자동 반영됩니다.

**❹ 매출세액 및 매입세액 신고내역**
회사를 선택하였을 경우 부가세신고서에 입력된 내용이 자동 반영 됩니다. 데이터를 자동 반영하려면 해당 메뉴를 작성하기 전에 해당 회사(사업장)코드의 부가가치세신고서를 작성하여 저장한 후 본점 회사코드에서 해당 메뉴를 작성해야 합니다. 매출 및 매입의 과세표준과 세율을 입력하면 세액이 자동으로 계산되며, 가산세 및 공제세액 및 내부거래를 입력합니다.

**❺ 공제세액**
예정신고 미환급세액과 예정고지세액, 기 납부세액은 공제세액란에서 조정합니다.

### ▣ 편리한 기능(기능모음)

**❶ 불러오기**
조회한 기간의 부가가치세신고서 메뉴에 저장된 데이터를 새로 불러옵니다. 불러오기를 하면 기존에 입력된 데이터는 삭제됩니다.

### ▣ 참고사항(사업자단위 과세제도)

✜ 2개 이상의 사업장이 있는 경우, 사업자단위로 본점 또는 주사무소에서 사업자등록을 하여 일괄 신고, 납부 등을 할 수 있도록 하는 제도입니다.

## 6) 과표수정 및 추가납부계산서(부가세)

메뉴위치 : 회계 [부가가치세] 첨부서류 ▶ 과표수정 및 추가납부계산서(부가세)

### ▣ 개요

부가가치세 수정신고시 반드시 작성해야 하는 부가가치세 부속서류로, 추가납부가 있는 경우 작성합니다.

### ▣ 입력방법

**❶ 기간**
부가가치세 수정신고 기간을 입력합니다.

**❷ 수정차수**
수정 신고한 부가가치세 신고서의 차수를 F2코드도움이나 C아이콘으로 선택합니다.

**❸ 신고인 정보**
회사등록 메뉴에 입력되어 있는 대표자성명, 법인&사업자등록번호, 상호명, 사업장주소, 사업장전화번호 자동으로 반영되는 란이며 직접입력 및 수정이 불가능 합니다.

**❹ 신고내용**
F2코드도움으로 불러온 수정신고서의 차수에 입력된 데이터가 자동 반영되며, 신고사유는 직접 입력합니다.

### ▣ 편리한 기능(기능모음)

**❶ 일괄삭제**
사용자가 조회한 기간에 작성된 데이터 전체를 일괄 삭제합니다.

**❷ 불러오기**
조회한 기간과 차수의 부가가치세 수정 신고서의 데이터를 새로 불러옵니다. 불러오기를 하면 기존에 입력된 데이터는 삭제됩니다.

**❸ 위임장 정보**
신고인의 위임을 받아 대리인이 과세표준수정신고 및 추가자진납부계산서를 신고하는 경우 입력합니다.
- 불러오기: 부가가치세 전자신고 메뉴 제출자 탭의 세무대리인에 대한 정보가 반영됩니다.
- 삭제: 입력 및 수정한 모든 데이터를 삭제 합니다.

### ▣ 참고사항

✜ 전자신고시 수정신고와 경정청구의 구분은 추가 차가감 납부세액으로 구분합니다.
(추가 차가감 납부세액 > 0 일 경우 수정신고, 추가 차가감 납부세액 < 0 일 경우 경정청구)

## ★ BK 매핑·시사점

1. **공제받지못할매입세액명세서**는 매입전표 54.불공 + 불공제사유코드(❶~❽)의 집계 + 공통매입세액 안분(예정)·정산(확정)·재계산(감가상각자산 면세비율 5% 변동) 4개 탭으로 구성. BK 전표 라인에 `불공제사유` enum과 `공통매입여부` 플래그가 필요하고, 신고 엔진은 과세기간 종류(예정/확정)에 따라 안분→정산 흐름을 연결. 면세비율 5/100 임계 룰을 재계산 트리거로 구현.

2. **공통매입세액 안분/정산 계산식**: 면세공급가액 ÷ 총공급가액 비율 기반. 안분(예정)→정산(확정, 과세기간 전체 비율 재계산)→감가상각자산 재계산의 3단 로직을 신고 엔진에 상태머신으로 모델링. 직접입력·불러오기(전표 자동집계) 모두 지원.

3. **건물등감가상각자산취득명세서**는 조기환급 목적. 매입전표 51/52/54/55/57 + 고정자산 계정과목 분개를 자산구분별로 집계. BK는 계정과목 마스터에 `고정자산구분`(건물/기계/차량 등) 태그를 두고, 서식별 "불러오기 계정구간 파라미터"로 자동 집계 → 조기환급 신고 플로우와 연동.

4. **대손세액공제/대손변제**: 공제율 10/110 자동계산. 대손사유(특히 5.부도→기준일+6개월 자동) 코드 룰과 대손확정일·변제일 관리가 핵심. BK는 채권(미수금/외상매출금) 마스터에 대손상태(발생/확정/변제) 라이프사이클을 두고 매출세액 차감/가산을 자동 반영. 거래처 식별(사업자/주민등록번호) 매핑 필수.

5. **총괄납부(사업장별명세) vs 사업자단위과세(사업장별부가세)**: 둘 다 멀티-사업장 통합신고지만 제도가 다름. BK 멀티테넌트에서 한 법인의 여러 사업장(회사코드)을 묶는 모델 필요 — `사업장 그룹`(주/종 사업장, 총괄납부번호 또는 단위과세 적용번호) 엔티티. 각 사업장 부가세신고서 저장 후 본점에서 합산 불러오기. 불러오기 시 기존 데이터 삭제(덮어쓰기) 정책 주의.

6. **수정신고(과표수정·추가납부) vs 경정청구(과표및세액경정청구)**: 동일 입력구조, 분기는 **추가 차가감 납부세액 부호**(>0 수정신고 / <0 경정청구). BK 신고 엔진은 원신고서 차수(수정차수)를 참조해 델타를 계산하고 부호로 서식 유형을 자동 결정. 신고인 정보는 회사등록 마스터에서 read-only 자동반영, 위임장(세무대리인) 정보는 전자신고 제출자 탭과 공유.

7. **신고일자 자동 기본값 공통 규칙**: 다수 서식이 "조회기간 마지막 월 + 다음달 25일"을 신고일자 기본값으로 자동 반영(수정 가능) → BK 신고 워크플로의 공통 유틸로 구현. 부가세 신고기한 캘린더와 연동.

8. **소규모개인사업자 신청서**는 가이드 미공개 → 카탈로그 엔트리·전자신고 매핑만 유지하고 UI는 베타/비노출 처리. 한시적·정책성 서식은 시행기간 메타로 관리.
