## 1) 부가가치세신고서

> 메뉴위치 : 회계 [부가가치세] 주요신고서류 ▶ 부가가치세신고서

### ▣ 개요
일반과세, 간이과세 탭으로 구성되어 있으며, 각 사업자(기장업체)별로 해당하는 탭을 선택하여 입력. 신고구분은 1.정기신고, 2.수정신고 중 선택할 수 있으며, 수정신고를 선택하기 위해서는 동일기간내의 정기신고 데이터가 저장 및 마감되어 있어야 합니다.

### ▣ 입력방법
- **❶ 세금계산서 발급분** — 매입매출전표입력에서 유형 11.과세로 입력한 매출금액이 자동 반영.
- **❷ 매입자발행세금계산서** — 유형 25.매세로 입력한 매출금액이 자동 반영. (과세유형 25.매세는 기초정보관리/데이터관리→환경설정 메뉴의 과세유형 추가 사용설정에서 설정.)
- **❸ 신용카드, 현금영수증** — 유형 17.카과, 21.전자, 22.현과로 입력한 매출금액이 자동반영.
- **❹ 기타** — 유형 14.건별, 15.종합으로 입력한 매출금액이 자동 반영. 15.종합으로 입력된 금액은 "공급대가"로 반영되므로, 일반과세자는 공급가액과 세액으로 나누어 직접 입력 후 불러와야 합니다.
- **❺ 세금계산서 발급분** — 유형 12.영세로 입력한 매출금액이 자동 반영.
- **❻ 기타** — 유형 16.수출, 19.카영, 24.현영으로 입력한 매출금액이 자동 반영.
- **❼ 예정신고 누락분** — 예정신고시 누락된 매출자료가 있을 경우 입력. 더블클릭하면 예정신고 누락분명세 화면이 나타나며, 매출 누락분을 입력한 후 닫기 또는 Esc키로 복귀. 매입매출전표입력에서 매출 유형으로 선택하고, 기능모음의 예정누락에 예정신고 누락분 신고 대상월을 입력한 데이터가 반영.
- **❽ 대손세액가감** — 대손발생액에 대한 가감액 입력. 대손발생시 −(음수), 대손세액으로 신고되었던 금액을 회수한 경우 +(양수)로 입력.
- **❾ 일반매입** — 유형 51.과세, 52.영세, 54.불공, 55.수입으로 입력한 매입금액 및 세액이 자동 반영. 단, 분개시 계정과목을 고정자산코드로 입력한 매입금액 및 세액은 반영되지 않음.
- **❿ 수출기업수입납부유예** — 재화의 수입에 대한 부가가치세 납부유예를 승인받아 납부 유예된 세액을 입력.
- **⓫ 고정자산매입** — 유형 51.과세, 52.영세, 54.불공, 55.수입으로 입력하고 분개시 계정과목을 고정자산코드로 입력한 매입금액과 세액이 자동 반영.
- **⓬ 예정신고누락분** — 예정신고시 누락된 매입자료가 있을 경우 입력. 더블클릭하면 예정신고 누락분명세 화면이 나타나며, 매입 누락분을 입력한 후 닫기 또는 Esc키로 복귀. 매입매출전표입력에서 매입 유형으로 선택하고, 기능모음의 예정누락에 예정신고 누락분 신고 대상월을 입력한 데이터가 반영.
- **⓭ 매입자발행세금계산서** — 유형 64.매세로 입력한 매출금액이 자동 반영. (과세유형 64.매세는 환경설정 메뉴의 과세유형 추가 사용설정에서 설정.)
- **⓮ 그 밖의 공제매입세액** — 기타공제 매입세액란을 더블클릭하면 그 밖의 공제매입세액 명세의 입력화면으로 진입.
  - 41.신용매출전표수취/일반: 유형 57.카과, 61.현과로 입력한 데이터 반영.
  - 42.신용매출전표수취/고정: 유형 57.카과, 61.현과로 입력하고 분개시 고정자산 계정과목으로 입력한 데이터 반영.
  - 43.의제매입세액/평창포함: 입력시 면세 유형에서 적요를 입력한 데이터 반영. 평창올림픽 사업의제매입공제신고서가 작성되어 있는 경우 그 밖의 공제매입세액의 의제매입세액/평창포함 란에 금액과 세액이 반영(의제매입세액이 있는 경우 해당란에 합산).
  - 44.재활용폐자원등 매입세액: 매입매출전표입력시 면세유형에서 적요7 재활용폐자원매입세액 입력한 데이터 반영.
  - 45.과세사업전환매입세액, 46.재고매입세액, 47.변제대손세액, 48.외국인관광객환급세액에 해당되는 사업자는 세액을 직접 입력.
- **⓯ 공제받지못할매입세액** — 더블클릭하면 공제받지 못할 매입세액명세서의 입력창이 나타남.
  - 50.공제받지못할 매입세액: 유형 54.불공으로 입력한 데이터 반영.
  - 51.공통매입세액 면세사업: 공제받지못할매입세액명세서 메뉴에서 안분 및 정산한 데이터 반영.
  - 52.대손처분받은 세액: 이미 공제받은 매입세액이나 신고할 매입세액에 대하여, 거래 상대방(공급자)이 대손확정으로 신고하여 대손세액 공제를 받는(받은) 경우, 그 처분 받은 대손세액 상당액을 입력.
- **⓰ 그 밖의 경감, 공제세액** — 전자신고세액공제, 전자세금계산서 발급세액공제, 택시운송사업자경감세액, 대리납부세액공제세액, 현금영수증 사업자세액공제액을 직접입력.
- **⓱ 신용카드매출전표등발행공제계** — 개인사업자로서 소매업자, 음식점업자, 숙박업자 등 부가가치세법시행령 제79조의2제1항 및 제2항에 따른 사업자가 신용카드 및 전자화폐에 의한 매출이 있는 경우에 입력. 유형 17.카과, 19.카영, 21.전자, 22.현과, 24.현영으로 입력한 공급대가를 반영.
- **⓲ 예정신고 미환급세액/예정고지세액/사업양수자의 대리납부 기납부세액/매입자 납부특례 기납부세액/신용카드업자가 대리납부한 세액**
  - 예정신고 미환급세액: 예정신고시 환급세액이 있었으나 환급신고를 하지 못한 세액이 있을 경우 입력.
  - 예정고지세액: 예정신고기간에 부가가치세신고는 하지 않았으나 고지서에 의해 세액을 납부한 경우 입력.
  - 사업양수자의 대리납부 기납부세액: 부가가치세법 시행령 제95조 제5항 및 제106조의13제4항에 따른 부가가치세 관리기관이 국고에 직접 입금한 부가가치세액 입력.
  - 매입자 납부특례 기납부세액: 조세특례제한법 시행령 제106조의9에 따른 부가가치세 관리기관이 국고에 직접 입금한 부가가치세액 입력.
  - 신용카드업자가 대리납부한 세액: 2019.1.1.부터 조세특례제한법 시행령 제106조의14제1항에 따른 특례사업자일 경우 신용카드업자가 대리납부한 부가가치세액 입력.
- **⓳ 가산세액계** — 가산세액계란을 더블클릭하면 가산세명세 입력화면이 나타나며, 해당사항을 입력한 후 닫기 클릭.
- **⓴ 총괄납부사업자 납부할세액(환급받을 세액)** — 부가가치세 총괄납부승인을 얻은 사업장 중 주된 사업장에서 종된 사업장의 부가가치세액을 합산하여 입력. 주된 사업장에서 사업장별부가세납부환급신고명세 메뉴를 작성시 자동 반영.

### ■ 과세표준명세 입력방법
- **❶ 과세표준명세** — 28~30란은 과세표준 합계액(9)의 업태·종목별로 세분화한 데이터가 반영, 32.합계액은 과세표준 합계액(9)의 과세표준 합계와 일치해야 함. 31.수입금액제외란은 고정자산매각, 직매장공급 등 수입금액에서 제외되는 금액을 입력.
- **❷ 면세수입금액** — 80~81란은 부가가치세가 면제되는 사업의 수입금액을 업태·종목별로 구분하여 반영. 82.수입금액 제외란은 면세수입금액중 고정자산매각, 직매장공급 등 수입금액에서 제외되는 금액 입력.
- **❸ 계산서 발급 수취** — 84.계산서 발급금액란에는 부가가치세가 과세되지 아니한 재화 또는 용역을 공급하고 교부한 계산서의 합계액 입력(유형 13.면세로 입력한 데이터 반영). 85.계산서수취금액란에는 거래상대방으로부터 교부받은 계산서의 합계액 입력(53.면세로 입력한 데이터 반영).

### 사업장명세
사업장명세는 확정신고시 음식·숙박업 또는 서비스업 중 해당업종 사업자에 한해 작성하며, 수정신고시에는 첨부서식이 아니므로 제출하지 않습니다.
- **❶ 기간** — 과세기간의 월·일 입력.
- **❷ 기본사항** — 자가와 타가 중 해당사항에 마우스를 위치하여 클릭.
- **❸ 기본경비** — 6, 12월 또는 폐업시 폐업 직전월을 기준으로 작성하고, 확정신고 최종 월의 임차보증금·임차료·경비를 입력.

### ▣ 편리한 기능(기능모음)
- **❶ 불러오기** — 데이터를 새로 불러옵니다.
- **❷ 저장** — 현재 시점까지 입력된 데이터 저장.
- **❸ 과표** — 기간을 조회한 상태에서 과표버튼 클릭시 과세표준명세 입력화면이 나타나며 TAB키 및 커서를 이동하여 편집·수정 작업.
- **❹ 마감** — 전자신고 제출을 위해 저장된 데이터를 마감.
- **❺ 작성조회** — 현재 시점까지 저장되어 있는 부가가치세신고서를 조회하며, 선택한 신고서를 불러올 수 있음.
- **❻ 종사업자** — 일반과세자 구분이 총괄납부사업자의 주사업자가 아닌 종사업자인 경우 선택.
- **❼ 인쇄옵션** — 수정신고시에만 활성화, 수정신고 출력물 인쇄시 상단 표기 방법(빨간색/△/굵게) 설정.
- **❽ 첨부서식 인쇄** — 작성한 부가가치세에 첨부된 부속서류를 체크박스 선택하여 인쇄.
- **❾ 납부서 인쇄** — 부가가치세 납부서 인쇄.
- **❿ 매출액보기** — 직전년도 부가가치세 과세표준과 면세수입금액 확인.
- **⓫ 국세청 부가율 적용여부** — 국세청 부가율 적용(공제받지못할매입세액 차감) "여,부" 선택.

### ▣ 참고사항
**✜ 수정신고서 작성방법** — 정기신고서를 작성하여 마감한 경우 신고구분을 2.수정신고로 선택한 후 수정차수를 입력. 정기신고서의 수정신고서를 작성 후 재수정신고를 작성하려면 수정차수 변경 후 재조회시 입력 가능.

**✜ 역추적** — 매입매출전표입력 및 일반전표입력 메뉴에 저장된 데이터와 부가가치세 부속서류에 저장된 데이터를 표시.
- 전표내역: 해당 라인별로 매입매출전표입력 및 일반전표입력 메뉴에서 입력된 데이터를 과세유형별로 건수·금액·세액을 표시. 전표내역 클릭시 하단 전표 상세내역에서 건별 전표 확인 가능.
- 작성서식: 해당 라인별 관련된 첨부서식의 저장된 데이터를 작성서식 별로 건수·금액·세액을 표시.

## ★ BK 매핑·시사점

1. **부가가치세신고서 = 매입매출전표 과세유형(taxType) 코드의 집계 종착점.** 본표의 각 란은 특정 과세유형 코드 집합의 자동 집계로 채워진다 — 매출: 11.과세→세금계산서, 12.영세→영세 세계산서, 14.건별/15.종합→기타, 16.수출/19.카영/24.현영→영세 기타, 17.카과/21.전자/22.현과→카드·현금; 매입: 51.과세/52.영세/54.불공/55.수입(고정자산코드 여부로 일반/고정 분기), 57.카과/61.현과→그밖의공제. BK는 전표 라인의 `taxType` 코드 → 신고서 란 매핑 테이블을 단일 소스로 두고, 고정자산 계정 여부 등 부가 조건을 룰로 평가하여 신고서를 자동 생성한다.
2. **마감(lock) → 전자신고 파일 제작 파이프라인.** 각 합계표·명세서·신고서는 '마감' 상태가 되어야 전자신고 파일에 포함되며, 마감 후 입력 제한·마감해제 전까지 동결. BK는 신고 산출물에 `status: draft→locked→filed` 상태머신과 마감 스냅샷을 두고, 마감된 데이터만 전자신고 XML/전산매체 생성에 투입.
3. **세무리스크자동검증(부가가치세MRI) = BK 세무 MRI/AI 검증의 직접 대응 기능.** (가이드 미공개) 신고 전 오류·누락 자동 탐지 → BK는 신고서 마감 게이트 직전에 룰엔진+AI 이상탐지(미전송 자동전표, 합계표-신고서 정합성, 휴폐업 거래처, 적격증빙 누락 등)를 돌려 리스크 점수·경고를 제시.
4. **합계표(세금계산서/계산서)의 전자 vs 전자외, 사업자번호 vs 주민번호 구분 집계 + 합산 기능** → BK는 전표 라인에 전자발행 플래그·상대방 식별유형(사업자/주민) 메타를 두고, 거래처 사업자등록번호 단위 합산(merge) 옵션을 제공. 휴폐업조회는 국세청 API 연동으로 거래처 상태 검증.
5. **신용카드 수령명세서/발행집계표 = 매입세액공제·발행세액공제의 양면.** 수령명세서(매입측, 57.카과/61.현과/63.복지 + 의제매입 차감 적요)와 발행집계표(매출측, 발행금액 1.3% 공제·연 500만원 한도) → BK는 카드·현금영수증 전표에 공제 적격성 메타와 한도 계산기를 두고 두 서식을 동시 산출.
6. **역추적(drill-down) = 신고서 본표 → 전표 + 부속서식 양방향 추적.** BK는 모든 신고서 라인에 원천 전표(taxType·건수·금액·세액)와 부속서식 링크를 보존하여 감사 추적성(audit trail)을 보장. 이는 02_자동전표 상태머신·03 장부 드릴다운과 동일한 추적 인프라 위에 구현.
7. **예정신고 누락분·수정신고 차수 관리** → BK는 신고 기간(예정/확정)과 수정차수를 1차 키로 갖는 신고 버전 관리 모델을 두고, 정기신고 마감본을 기준으로 수정신고 델타를 작성하는 구조를 채택.
