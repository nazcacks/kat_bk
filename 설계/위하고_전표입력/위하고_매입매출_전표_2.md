# 위하고 매입매출전표입력(2) 과세유형별 입력방법 영상 상세 정리

## 문서 성격

- 영상: [[WEHAGO l Smart A 10] 매입매출전표입력 (2) 과세유형별 입력방법](https://www.youtube.com/watch?v=B8herxnuehA)
- 채널: 위하고(WEHAGO) 공식 채널
- 업로드일: 2022-10-11
- 영상 길이: 11분 39초
- 확인 기준: 2026-06-25 15:23 KST
- 자막 상태: 공개 자막/자동 자막이 확인되지 않음
- 정리 방식: 영상의 원문 대사를 전재하지 않고, 공개 영상 메타데이터의 시간대 구성과 WEHAGO 고객센터 문서, 저장소 내 Smart A 10 분석 문서를 대조하여 실무자가 다시 따라 할 수 있도록 재구성했다.

이 문서는 1편의 기본 사용방법 다음 단계로, `매입매출전표입력`에서 과세유형을 잘못 선택하면 어떤 신고서식과 장부가 달라지는지를 중심으로 정리한다. 핵심은 상단의 `유형`이 부가가치세 신고자료의 라우팅 기준이고, 하단의 `분개`가 원장과 재무제표의 라우팅 기준이라는 점이다.

## 영상 핵심 요약

1. 매출 입력에서는 `11.과세`, `13.면세`, `17.카과`를 예시로 세금계산서 매출, 계산서 매출, 카드 과세매출이 각각 어느 신고서식에 반영되는지 확인한다.
2. 매입 입력에서는 `51.과세`, `54.불공`, `57.카과`, `61.현과`를 예시로 세금계산서 매입, 불공제 매입, 카드 과세매입, 현금영수증 과세매입을 구분한다.
3. `51.과세`와 `57.카과`는 하단 분개 계정이 고정자산인지 여부에 따라 부가가치세신고서의 일반매입/고정자산매입 반영 위치가 달라진다.
4. `54.불공`은 일반적인 매입세액공제가 아니라 불공제 사유와 함께 `공제받지못할매입세액명세서`로 이어지는 입력이다.
5. 의제매입세액공제와 재활용폐자원세액공제는 단순히 과세유형만 고르는 것이 아니라, 면세 매입 유형과 하단 계정과목/적요 코드 조합으로 신고서에 반영한다.

## 화면 구조 재확인

`매입매출전표입력`은 상단부와 하단부가 분리된 구조다.

| 영역 | 입력 목적 | 대표 필드 | 반영 대상 |
|---|---|---|---|
| 상단부 | 세무 증빙과 부가세 신고자료 입력 | 일자, 거래처, 유형, 품명, 수량, 단가, 공급가액, 부가세, 전자 여부 | 부가가치세신고서, 세금계산서합계표, 계산서합계표, 신용카드매출전표발행집계표, 신용카드매출전표등수령명세서, 매입매출장 |
| 하단부 | 회계 분개 입력 | 계정과목, 차변/대변, 거래처, 적요, 관리항목 | 계정별원장, 분개장, 거래처원장, 재무제표, 고정자산/재고/비용성 분류 |

실무상 오류는 대부분 다음 두 경우에서 발생한다.

- 상단 유형은 맞지만 하단 분개 계정이 틀려 신고서의 일반매입/고정자산매입, 원장 계정이 달라지는 경우
- 하단 분개는 맞지만 상단 유형이 틀려 부가세 신고서식에 누락되거나 다른 부속서류로 들어가는 경우

## 매출유형별 입력 방법

### 1. `11.과세` 세금계산서 과세 매출

`11.과세`는 과세 재화나 용역을 공급하고 세금계산서를 발급한 매출이다. 전자세금계산서든 종이 세금계산서든 과세 세금계산서 매출이라는 점이 핵심이다.

입력 흐름:

1. 상단 일자에 세금계산서 작성일자를 입력한다.
2. 거래처 코드를 선택한다. 세금계산서합계표 반영을 위해 사업자등록번호가 등록되어 있어야 한다.
3. 유형을 `11.과세`로 선택한다.
4. 품명, 공급가액, 부가세를 입력한다. 수량/단가를 쓰면 공급가액과 세액을 계산할 수 있고, 단수 차이는 직접 보정한다.
5. 전자세금계산서라면 `전자` 상태를 확인한다. 발급 절차를 생략하고 이미 발급된 전자분을 입력하는 경우에는 전자 입력 상태를 맞춘다.
6. 분개유형을 선택하고 하단에 매출채권, 매출, 부가세예수금 등 필요한 분개를 확인한다.

자동 반영:

| 서식/장부 | 반영 방식 |
|---|---|
| 부가가치세신고서 | 과세표준의 세금계산서 발급분으로 반영 |
| 세금계산서합계표 | 매출세금계산서 집계에 반영. 전자 여부에 따라 전자/전자외 구분 |
| 매입매출장 | 상단 세무자료 기준으로 조회 |
| 계정별원장/재무제표 | 하단 분개 기준으로 반영 |

대사 포인트:

- `11.과세`는 카드매출이나 현금영수증 매출과 구분해야 한다.
- 카드로 대금을 받았더라도 세금계산서를 발급한 거래라면 세금계산서 발급분과 카드 결제분 중복 집계 여부를 별도로 확인한다.
- 거래처 사업자등록번호가 없으면 합계표 마감이나 전자신고 단계에서 오류가 날 수 있다.

### 2. `13.면세` 계산서 면세 매출

`13.면세`는 계산서를 발급하는 면세 매출에 사용한다. 부가세가 없는 거래이므로 `11.과세`와 달리 매출세액이 발생하지 않는다.

입력 흐름:

1. 상단 일자에 계산서 작성일자를 입력한다.
2. 거래처 코드를 선택하고 사업자등록번호 또는 주민등록번호 구분을 확인한다.
3. 유형을 `13.면세`로 선택한다.
4. 공급가액을 입력한다. 부가세는 입력 대상이 아니다.
5. 전자계산서 발급/수취 상태가 있으면 전자 여부를 맞춘다.
6. 하단 분개에서 매출채권/면세매출 등 계정을 확인한다.

자동 반영:

| 서식/장부 | 반영 방식 |
|---|---|
| 계산서합계표 | 매출계산서 집계에 반영. 전자 여부에 따라 전자/전자외 구분 |
| 부가가치세신고서 | 면세수입금액 및 계산서 발급 관련 항목에 반영 |
| 매입매출장 | 면세 매출 자료로 조회 |
| 계정별원장/재무제표 | 하단 분개 기준으로 반영 |

대사 포인트:

- 과세 매출을 `13.면세`로 입력하면 매출세액이 누락된다.
- 면세 매출은 부가세가 없는 대신 계산서합계표와 면세수입금액 대사가 중요하다.
- 전자계산서 여부가 계산서합계표의 전자/전자외 구분에 영향을 준다.

### 3. `17.카과` 신용카드 과세 매출

`17.카과`는 과세 재화나 용역을 공급하고 신용카드매출전표를 발급한 매출이다. 증빙이 세금계산서가 아니라 신용카드매출전표라는 점이 `11.과세`와 다르다.

입력 흐름:

1. 카드 승인일 또는 매출 발생일 기준으로 일자를 입력한다.
2. 거래처는 실제 고객을 관리할 수도 있고, 카드사/기타 거래처 관리 방식에 따라 하단 분개 거래처를 별도로 조정할 수도 있다.
3. 유형을 `17.카과`로 선택한다.
4. 공급대가에서 공급가액/부가세를 분리하거나, 화면 설정에 따라 공급가액 기준으로 입력한다.
5. 분개유형은 현금, 카드, 혼합 등 회사 처리 방식에 맞춘다.
6. 하단 분개에서 카드채권, 매출, 부가세예수금, 수수료 처리 여부를 확인한다.

자동 반영:

| 서식/장부 | 반영 방식 |
|---|---|
| 부가가치세신고서 | 신용카드/현금영수증 과세 매출로 반영 |
| 신용카드매출전표발행집계표 | 신용카드 매출전표 등 발행금액 현황에 반영 |
| 매입매출장 | 카드 과세 매출 자료로 조회 |
| 계정별원장/재무제표 | 하단 분개 기준으로 반영 |

대사 포인트:

- `17.카과`는 세금계산서합계표로 가지 않는다.
- 세금계산서를 발급하고 카드로 결제받은 거래는 `11.과세 + 분개유형 카드`처럼 중복 발급 내역으로 관리될 수 있으므로, `17.카과`와 혼동하면 안 된다.
- 봉사료나 카드수수료가 있는 업종은 공급가액/부가세/봉사료/수수료 분리가 필요하다.

## 매입유형별 입력 방법

### 1. `51.과세` 세금계산서 과세 매입

`51.과세`는 과세 세금계산서를 수취한 매입이다. 일반적인 매입세액 공제의 기본 유형이다.

입력 흐름:

1. 세금계산서 작성일자를 상단 일자로 입력한다.
2. 매입처 거래처 코드를 선택한다. 합계표 반영을 위해 사업자등록번호가 정확해야 한다.
3. 유형을 `51.과세`로 선택한다.
4. 품명, 공급가액, 부가세를 입력한다.
5. 전자세금계산서 수취분이면 전자 상태를 맞춘다.
6. 하단 분개에서 비용/재고/고정자산 계정과 부가세대급금, 외상매입금 등을 확인한다.

자동 반영:

| 하단 계정 성격 | 부가가치세신고서 반영 | 기타 반영 |
|---|---|---|
| 일반 비용/재고 계정 | 일반매입 | 세금계산서합계표 매입세금계산서, 매입매출장 |
| 고정자산 계정 | 고정자산매입 | 세금계산서합계표 매입세금계산서, 매입매출장 |

대사 포인트:

- 같은 `51.과세`라도 하단 분개 계정이 고정자산이면 신고서에서 일반매입이 아니라 고정자산매입으로 분류된다.
- 고정자산 취득인데 비용 계정으로 분개하면 부가세 신고서 위치뿐 아니라 자산대장/감가상각 흐름도 어긋난다.
- 전자 여부는 세금계산서합계표의 전자/전자외 구분에 영향을 준다.

### 2. `54.불공` 공제받지 못할 매입세액

`54.불공`은 세금계산서를 수취했지만 매입세액 공제 대상이 아닌 거래에 사용한다. 예를 들면 접대비, 비영업용 소형승용차 관련 지출, 사업과 직접 관련 없는 지출, 필요적 기재사항 누락 등 불공제 사유가 있는 경우다.

입력 흐름:

1. 세금계산서 작성일자와 거래처를 입력한다.
2. 유형을 `54.불공`으로 선택한다.
3. 공급가액과 부가세를 입력한다.
4. 불공제 사유를 선택하거나 확인한다.
5. 하단 분개에서 부가세를 회수 가능한 `부가세대급금`으로 둘지, 비용/자산 취득가액에 포함할지 회사의 회계처리 기준과 프로그램 자동분개 결과를 확인한다.

자동 반영:

| 서식/장부 | 반영 방식 |
|---|---|
| 세금계산서합계표 | 매입세금계산서 데이터로 집계 |
| 공제받지못할매입세액명세서 | 불공제 사유별 내역으로 반영 |
| 부가가치세신고서 | 공제받지 못할 매입세액 영역으로 연결 |
| 계정별원장/재무제표 | 하단 분개 기준으로 반영 |

대사 포인트:

- `54.불공`은 세금계산서가 없다는 뜻이 아니라, 세금계산서는 있으나 공제를 받지 못한다는 뜻이다.
- 불공제 사유가 빠지면 부속서류 대사에서 문제가 생긴다.
- 실무 검토에서는 국세청 자료가 공제/불공제로 표시된 내역과 사용자가 입력한 `54.불공` 전표가 일치하는지 확인해야 한다.

### 3. `57.카과` 신용카드 과세 매입

`57.카과`는 과세 신용카드매출전표를 수취하고, 매입세액 공제 요건을 충족하는 경우에 사용한다. 세금계산서 수취가 아니므로 `51.과세`와 반영 서식이 다르다.

입력 흐름:

1. 카드 승인일 기준으로 일자를 입력한다.
2. 공급자 거래처를 입력한다. 사업자등록번호와 업종, 과세/면세 여부를 확인한다.
3. 유형을 `57.카과`로 선택한다.
4. 공급가액과 부가세를 입력한다.
5. 하단 분개에서 비용/재고/고정자산 계정과 카드채무/미지급금 등을 확인한다.
6. 고정자산 매입이면 하단 계정을 고정자산 계정으로 분개한다.

자동 반영:

| 하단 계정 성격 | 부가가치세신고서 반영 | 기타 반영 |
|---|---|---|
| 일반 비용/재고 계정 | 신용카드매출전표수취 일반 | 신용카드매출전표등수령명세서 |
| 고정자산 계정 | 신용카드매출전표수취 고정 | 신용카드매출전표등수령명세서 |

대사 포인트:

- `57.카과`는 세금계산서합계표가 아니라 신용카드매출전표등수령명세서로 간다.
- 카드전표에 부가세가 별도로 구분되어 있고 공제 요건을 충족해야 한다.
- 세금계산서를 중복 수취한 카드 결제분은 이중 공제 위험이 있으므로 전자세금계산서 자료와 중복 검증이 필요하다.

### 4. `61.현과` 현금영수증 과세 매입

`61.현과`는 과세 현금영수증을 수취하고 매입세액 공제 요건을 충족하는 경우에 사용한다. 신고 흐름은 카드 과세매입과 유사하다.

입력 흐름:

1. 현금영수증 승인일 또는 거래일을 입력한다.
2. 공급자 거래처를 입력한다.
3. 유형을 `61.현과`로 선택한다.
4. 공급가액과 부가세를 입력한다.
5. 하단 분개에서 비용/재고/고정자산 계정과 지급수단 계정을 확인한다.

자동 반영:

| 하단 계정 성격 | 부가가치세신고서 반영 | 기타 반영 |
|---|---|---|
| 일반 비용/재고 계정 | 신용카드매출전표수취 일반과 같은 공제매입 영역 | 신용카드매출전표등수령명세서 |
| 고정자산 계정 | 신용카드매출전표수취 고정과 같은 공제매입 영역 | 신용카드매출전표등수령명세서 |

대사 포인트:

- 현금영수증도 카드 수취명세서 계열의 서식에 함께 반영된다.
- 국세청 현금영수증 자료의 공제/불공제 표시와 사용자 입력 유형을 대사해야 한다.
- 접대비, 비영업용 차량, 면세/간이 사업자 거래 등은 `61.현과`로 공제 처리하기 전에 공제 가능 여부를 검토해야 한다.

## 의제매입세액공제와 재활용폐자원 반영

영상 후반은 일반 과세 매입이 아니라 특수 공제 서식에 반영하는 입력 조합을 다룬다. 여기서 중요한 점은 `51.과세`, `57.카과`, `61.현과` 같은 과세 매입 유형으로 처리하는 것이 아니라, 면세 매입 유형과 하단 분개 정보를 함께 써야 한다는 점이다.

### 의제매입세액공제

대상 흐름:

- 면세 농산물, 축산물, 수산물, 임산물 등을 원재료로 매입해 과세 재화나 용역을 공급하는 업종에서 사용한다.
- 매입세금계산서의 실제 부가세가 있는 것이 아니라, 법정 공제율로 의제 매입세액을 계산한다.

입력 조합:

| 구성요소 | 입력 기준 |
|---|---|
| 상단 유형 | `53.면세`, `58.카면`, `60.면건`, `62.현면` 등 면세 매입 유형 |
| 하단 계정 | 상품, 원재료, 부재료 등 원재료성 재고/매입 계정 |
| 하단 적요 | 의제매입세액 원재료 차감 성격의 적요 코드 |
| 반영 서식 | 의제매입세액공제신고서, 부가가치세신고서의 그 밖의 공제매입세액 |

점검:

- 과세유형만 면세로 선택하고 적요를 넣지 않으면 의제매입세액공제신고서로 자동 분류되지 않을 수 있다.
- 원재료가 아닌 일반 소모품/비용 계정으로 분개하면 공제대상 집계에서 제외될 수 있다.
- 공제율과 한도는 업종, 사업자 유형, 신고기간에 따라 달라질 수 있으므로 신고서 작성 단계에서 재확인한다.

### 재활용폐자원세액공제

대상 흐름:

- 재활용폐자원 또는 중고품을 과세사업자가 아닌 자 등으로부터 취득해 제조, 가공, 공급하는 경우 사용한다.
- 매입세액공제를 받기 위한 별도 부속서류에 반영한다.

입력 조합:

| 구성요소 | 입력 기준 |
|---|---|
| 상단 유형 | `53.면세`, `58.카면`, `60.면건`, `62.현면` 등 면세 매입 유형 |
| 하단 계정 | 상품, 원재료, 부재료 등 재활용폐자원 매입 성격의 계정 |
| 하단 적요 | 재활용폐자원 매입세액 성격의 적요 코드 |
| 반영 서식 | 재활용폐자원세액공제신고서, 부가가치세신고서의 그 밖의 공제매입세액 |

점검:

- 단순 면세 매입과 재활용폐자원 공제 대상 매입을 적요 코드로 구분한다.
- 중고차 등 품목별 공제율과 추가 필수정보가 있는 경우 신고서에서 별도 확인한다.
- 재활용폐자원세액공제신고서에서 불러오기 후 관리용/신고용 탭이 맞는지 대사한다.

## 과세유형별 신고서 반영 빠른 표

| 유형 | 거래 성격 | 주요 증빙 | 부가세 | 핵심 반영 서식 |
|---|---|---|---:|---|
| `11.과세` | 과세 매출 | 세금계산서 | 10% | 부가가치세신고서 세금계산서 발급분, 세금계산서합계표 |
| `13.면세` | 면세 매출 | 계산서 | 없음 | 계산서합계표, 부가가치세신고서 면세수입/계산서 발급 |
| `17.카과` | 카드 과세 매출 | 신용카드매출전표 | 10% | 부가가치세신고서 신용카드/현금영수증 매출, 신용카드매출전표발행집계표 |
| `51.과세` | 과세 매입 | 세금계산서 | 10% | 부가가치세신고서 일반매입 또는 고정자산매입, 세금계산서합계표 |
| `54.불공` | 불공제 매입 | 세금계산서 | 10% | 공제받지못할매입세액명세서, 부가가치세신고서 불공제 매입세액, 세금계산서합계표 |
| `57.카과` | 카드 과세 매입 | 신용카드매출전표 | 10% | 신용카드매출전표등수령명세서, 부가가치세신고서 신용카드수취 일반/고정 |
| `61.현과` | 현금영수증 과세 매입 | 현금영수증 | 10% | 신용카드매출전표등수령명세서, 부가가치세신고서 신용카드수취 일반/고정 |
| `53/58/60/62` + 적요 | 의제/재활용 관련 면세 매입 | 계산서, 카드, 영수증, 현금영수증 | 없음 | 의제매입세액공제신고서, 재활용폐자원세액공제신고서 |

## 실무 입력 체크리스트

### 입력 전

- 거래가 매출인지 매입인지 먼저 구분한다.
- 증빙이 세금계산서, 계산서, 카드, 현금영수증, 영수증 중 무엇인지 확인한다.
- 과세, 영세, 면세, 불공제 여부를 확인한다.
- 거래처 코드와 사업자등록번호가 등록되어 있는지 확인한다.
- 고정자산 취득 여부를 확인한다. 고정자산이면 상단 유형뿐 아니라 하단 계정과목도 맞아야 한다.

### 입력 중

- 상단 유형 코드를 먼저 정확히 선택한다.
- 공급가액과 부가세가 증빙 금액과 맞는지 확인한다.
- 전자세금계산서/전자계산서라면 전자 여부가 맞는지 확인한다.
- `54.불공`은 불공제 사유를 빠뜨리지 않는다.
- `57.카과`, `61.현과`는 공제 가능한 사업용 지출인지 확인한다.
- 의제매입/재활용폐자원은 면세 유형, 계정, 적요 코드가 모두 맞는지 확인한다.

### 입력 후

- 부가가치세신고서에서 해당 금액이 원하는 칸에 반영됐는지 확인한다.
- 세금계산서합계표/계산서합계표/신용카드매출전표 관련 서식을 불러와 대사한다.
- 매입매출장에서 유형별 금액을 확인한다.
- 계정별원장과 분개장에서 하단 분개가 의도한 계정으로 반영됐는지 확인한다.
- 전자신고 전에는 거래처 사업자번호, 전자/전자외 구분, 불공제 사유, 중복 공제 가능성을 점검한다.

## 자주 발생하는 오류와 예방

| 오류 상황 | 영향 | 예방 방법 |
|---|---|---|
| 과세 매출을 `13.면세`로 입력 | 매출세액 누락, 계산서합계표로 잘못 반영 | 증빙이 세금계산서인지 계산서인지 먼저 확인 |
| 카드 매출을 `11.과세` 또는 `17.카과` 중 잘못 선택 | 세금계산서합계표 또는 카드 발행집계표 반영 오류 | 세금계산서 발급 여부와 카드 결제 여부를 분리해서 판단 |
| `51.과세` 고정자산 매입을 비용 계정으로 분개 | 부가세신고서 고정자산매입 누락, 자산관리 누락 | 하단 계정과목을 고정자산 계정으로 지정 |
| `54.불공` 입력 후 사유 미관리 | 공제받지못할매입세액명세서 대사 오류 | 불공제 사유를 필수 메타데이터로 관리 |
| 카드/현금영수증 매입을 `51.과세`로 입력 | 세금계산서합계표와 수령명세서 모두 왜곡 | 증빙이 세금계산서인지 카드/현금영수증인지 확인 |
| 면세 원재료 매입을 의제매입 적요 없이 입력 | 의제매입세액공제신고서 누락 | 면세 유형 + 원재료 계정 + 적요 코드를 세트로 입력 |
| 재활용폐자원 매입을 일반 면세매입으로만 입력 | 재활용폐자원세액공제신고서 누락 | 재활용폐자원 적요와 품목/거래처 정보를 함께 관리 |
| 거래처 사업자등록번호 누락 | 합계표 마감/전자신고 오류 | 거래처등록 후 전표 입력, 신규 거래처는 즉시 보완 |

## BK 설계 반영 포인트

BK 회계/세무 SaaS 관점에서는 이 영상의 내용이 단순 입력 가이드가 아니라 `taxType` 기반 신고 라우팅 규칙으로 정리되어야 한다.

1. `taxType` 코드는 신고서식 반영의 단일 진실원으로 관리한다.
2. 상단 세무 헤더와 하단 분개 라인을 분리 저장한다.
3. 상단 유형은 부가세 신고서/부속서류/매입매출장 반영 기준이 된다.
4. 하단 계정과목은 원장/재무제표 반영 기준이 되며, 고정자산매입 분류에도 영향을 준다.
5. `54.불공`은 단순 세율 코드가 아니라 불공제 사유, 공제불가 판단, 부속서류 연결을 포함한 별도 상태로 다뤄야 한다.
6. `57.카과`와 `61.현과`는 세금계산서 수취가 아니므로 세금계산서합계표와 분리해야 한다.
7. 의제매입세액공제와 재활용폐자원세액공제는 과세유형, 계정과목, 적요 코드가 결합된 파생 규칙으로 구현해야 한다.
8. 신고서 불러오기/마감/전자신고는 재실행 가능해야 하며, 같은 전표를 중복 집계하지 않도록 멱등키가 필요하다.
9. 세금계산서, 카드, 현금영수증, 국세청 수집자료와 직접 입력 전표 간 중복 검증이 필요하다.
10. 과세유형 변경, 불공제 사유 변경, 마감 후 해제/수정은 감사로그와 변경이력을 남겨야 한다.

## 구현 모델로 바꾸면 필요한 규칙

| 규칙 | 설명 |
|---|---|
| `taxType` 마스터 | 11~25 매출, 51~64 매입 코드를 표준 코드로 관리 |
| 증빙 유형 | 세금계산서, 계산서, 카드, 현금영수증, 영수증, 수입세금계산서 등 |
| 세율 | 10%, 0%, 면세/해당없음, 불공제 여부 |
| 신고서 라우팅 | 부가가치세신고서, 세금계산서합계표, 계산서합계표, 카드 발행/수령명세서, 불공제명세서 등 |
| 계정 기반 분기 | 고정자산 계정이면 일반매입이 아니라 고정자산매입으로 집계 |
| 적요 기반 분기 | 의제매입세액, 재활용폐자원, 구리스크랩 등 특수 공제 서식 연결 |
| 전자/전자외 판정 | 전자세금계산서/전자계산서 상태와 국세청 전송 여부로 합계표 구분 |
| 거래처 검증 | 사업자등록번호, 주민등록번호, 휴폐업 상태, 과세/면세 사업자 여부 |
| 중복 검증 | 전자세금계산서와 카드/현금영수증, 자동수집자료와 직접입력자료 중복 방지 |
| 마감 잠금 | 마감된 신고기간의 전표 수정 제한, 해제 절차와 감사로그 기록 |
