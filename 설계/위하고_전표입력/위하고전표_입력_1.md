## 핵심 요약

`일반전표입력`은 부가가치세 신고와 직접 연결되는 매입/매출 세금계산서성 거래를 제외한 일반 회계거래를 입력하는 메뉴다. 입력된 전표는 분개장, 총계정원장, 재무제표에 반영된다. 부가세 신고 서식에 반영되어야 하는 거래는 `매입매출전표입력`에서 처리해야 한다.

영상은 다음 흐름을 다룬다.

1. 일반전표입력 메뉴의 기본 입력 흐름
2. 거래일자 입력
3. 전표 선택/체크 방법
4. 전표번호와 전표묶기
5. 거래처 코드 입력 및 신규 거래처 등록
6. 전표 구분 코드
7. 적요 입력
8. 차변/대변 금액 입력과 `+` 키 활용
9. 대체전표 입력 및 차액 처리
10. 전표 삭제와 자동전표 일괄삭제
11. 전표 인쇄와 결재란 추가

## 기능별 상세 메모

### 거래처 입력

- 거래처는 `거래처코드`와 `거래처명`을 기준으로 관리한다.
- 조회는 더블클릭, `C` 아이콘, `F2`, 자동완성으로 가능하다.
- 빠른 신규 등록은 Code 칸에서 `+`를 누르고 거래처명을 입력한 뒤 거래처등록 팝업에서 상세 정보를 보완하는 흐름이다.
- 분개설정에서 거래처코드 복사 옵션을 켜면 엔터 입력 시 상단 거래처 정보를 하단 라인으로 복사할 수 있다.
- 거래처명이 같아도 사업자번호, 계좌, 담당자 정보가 다를 수 있으므로 신규 등록 시 상세정보 보완이 필요하다.

### 적요 입력

- 적요는 거래의 실무 의미를 남기는 짧은 설명이다.
- 동일 거래가 반복되면 적요코드를 등록해 재사용한다.
- 적요는 장부 조회, 전표 검색, 검토자 이해에 직접 영향을 준다.
- 일괄적용 기능으로 선택된 전표들의 적요명을 한 번에 반영할 수 있다.

### 대체전표 입력

- 현금 입금/출금이 아닌 계정 간 이동은 대체전표로 입력한다.
- 차변 라인은 `3.차변`, 대변 라인은 `4.대변`으로 입력한다.
- 같은 전표번호 안에서 차변 합계와 대변 합계가 맞아야 한다.
- 차액 발생 시 선택전표소계와 차액전표보기로 원인을 찾는다.
- 분개설정의 금액자동분개는 상대계정 입력 시 차액을 자동 분개하는 보조 기능이다.
- 차대변 변환 옵션은 대체전표 입력 중 차액이 발생할 때 차변/대변 방향을 자동 조정하는 보조 기능이다.

### 전표묶기/전표분리

- Smart A 10은 일반전표번호를 직접 수정하는 대신 전표묶기로 번호 구조를 조정한다.
- 전표묶기는 같은 일자의 전표끼리 사용한다.
- 서로 다른 전표번호를 하나로 묶을 때는 대상 전표를 모두 체크하고 `전표묶기(F7)` 후 `묶기(Tab)`를 실행한다.
- 한 전표번호 안의 일부 라인을 분리할 때는 분리할 전표를 체크하고 `새로운 전표 생성(F8)`을 선택한 뒤 묶기 처리를 한다.
- 같은 번호 안에서 표시 순서가 맞지 않으면 전표순서 올리기/내리기로 조정한다.

### 삭제

- 일반 삭제는 선택한 전표를 대상으로 수행한다.
- 삭제 전에는 전표번호, 일자, 거래처, 금액, 자동전표 여부를 확인한다.
- 자동전표로 생성된 자료는 일반 삭제보다 자동분개 일괄검색/삭제 메뉴에서 출처와 기간을 확인하고 처리하는 것이 안전하다.
- 일마감된 기간의 자료는 수정, 삭제, 입력이 제한될 수 있다.

### 인쇄와 결재란

- 출력할 전표를 체크한 뒤 인쇄를 실행한다.
- 인쇄 미리보기와 출력 방식은 사용자 환경설정의 인쇄 옵션을 따른다.
- 결재란은 회사의 내부 검토/승인 양식에 맞춰 출력 옵션에서 반영한다.
- 인쇄 전에는 선택 전표 범위, 전표일자, 전표번호, 결재란 포함 여부를 확인한다.

## 실무 체크리스트

### 입력 전

- 회사와 회계기간이 맞는지 확인한다.
- 일반전표로 입력할 거래인지, 매입매출전표로 입력해야 할 부가세 관련 거래인지 먼저 구분한다.
- 거래처, 계정과목, 부서/현장/프로젝트 등 관리항목 마스터가 준비되어 있는지 확인한다.
- 반복 거래는 적요코드와 분개설정을 먼저 정비하면 입력 속도가 빨라진다.

### 입력 중

- 거래일자부터 정확히 입력한다.
- 구분 코드를 먼저 잡고 계정과목과 금액을 입력한다.
- 거래처가 필요한 계정은 거래처 코드까지 연결한다.
- 관리항목이 필요한 계정은 가이드 영역에서 누락 없이 입력한다.
- 대체전표는 같은 전표번호 안에서 차변/대변 합계가 일치하는지 확인한다.
- 금액 입력 시 `+` 키를 활용하되, 0이 과다 입력되지 않았는지 확인한다.

### 입력 후

- 선택전표소계로 대차 일치를 확인한다.
- 전표번호가 의도대로 묶였는지 확인한다.
- 분리해야 하는 라인이 있으면 전표묶기에서 `새로운 전표 생성`으로 분리한다.
- 중복전표검색 또는 체크펜을 활용해 검토 대상을 관리한다.
- 필요한 전표만 체크한 뒤 삭제, 인쇄, 일괄적용을 실행한다.
- 일마감 이후 수정/삭제 제한이 생기므로 마감 전 검토를 끝낸다.

## BK 설계 반영 메모

이 문서는 위하고 사용법 정리이며 BK의 공식 스펙은 아니다. 다만 현재 `kat_bk` 설계와 연결할 때는 아래 포인트를 참고할 수 있다.

- BK의 `journals` 도메인은 일반전표, 승인 요청, 수정/취소, 첨부, 이력을 다룬다.
- 위하고식 구분 코드 `출금/입금/차변/대변/결차/결대`는 BK의 전표 라인 입력 UX와 매핑 후보가 된다.
- 전표번호는 일자와 함께 식별되어야 하며, 전표묶기/분리는 단순 번호 수정이 아니라 감사 가능한 전표 구조 변경으로 다뤄야 한다.
- 삭제는 상태에 따라 임시저장 전표 삭제, 자동전표 재처리 삭제, 승인/기표 후 취소 또는 역분개로 구분해야 한다.
- 일마감 또는 마감 잠금 이후의 전표 수정/삭제는 차단하거나 해제 절차를 거쳐야 한다.
- 일반전표와 매입매출전표의 경계는 부가세 신고자료 정확성에 직결되므로, BK에서도 부가세 관련 거래를 별도 흐름으로 유도해야 한다.
- 거래처, 계정과목, 관리항목, 적요코드 입력은 단순 텍스트가 아니라 마스터 데이터와 연결되는 구조로 설계해야 한다.
