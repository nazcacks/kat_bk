1. 문서 개요 · 페이즈 05 범위

목적페이즈 05 구현 착수를 위해 결산정리·결산대체·재무제표·결산부속명세서·세무 명세서 화면별 레이아웃(와이어프레임)·입력 필드·액션/검증·연계 API·권한·마감상태 가드를 구현 가능한 수준으로 정의한다. 전표(P03)·장부(P04)에서 누적된 잔액을 결산정리분개 → 자동결산분개 → 손익대체 → 보고패키지로 마감하는 전 과정을 다룬다.
페이즈 정의결산자료입력(감가상각·대손충당·퇴직급여·재고·미수미지급·선급선수)으로 CLOSING_DR/CLOSING_CR 결차/결대 라인을 자동 생성하고, 합계잔액시산표 대차 검증 → 재무상태표·손익계산서·제조원가명세서·이익잉여금처분계산서·자본변동표·현금흐름표 산출 → 결산부속명세서·세무 명세서·기간/중단사업 보고서로 마무리한다. 마감상태(TEMP_CLOSED/CLOSED/LOCKED)가 전표 변경 가드로 작동한다.
화면 구성결산처리·검토 5화면(결산자료입력·결산부속명세서·현금및예금잔액검토·카드/상품권 검토·AI 시산표) + 재무제표 본표 4화면(합계잔액시산표·재무상태표·손익계산서·제조원가명세서) + 자본·현금흐름 3화면 + 출력·분석 2화면 + 세무 명세서 3화면 + 기간·중단사업 4화면 = 총 21화면(SA-CLS-01~21).
표기 원칙화면 ID는 본 문서 전용으로 SA-CLS-xx를 사용하고, 설계서 bk_설계서_v2.1.md 25.9·25.10 절과 연결한다. 검증은 경고(저장가능)·차단(저장불가) 2단계로 표기하며, 정상은 정상으로 표기한다.
본 페이즈는 집계·산출·검증 중심이므로, 화면 대부분이 결산 작업대(좌측 단계 트리 + 중앙 조정 그리드 + 우측 검증/주석 패널) 또는 재무제표 편집(탭 + 비교식 본표 그리드 + 차액 알림 모달) 형태다. 입력 화면(결산자료입력)은 정리사항 금액만 입력하면 결차/결대 분개를 자동 생성한다.
페이즈 05 마감 가드 핵심 — 모든 결산 산출물은 결산 스냅샷으로 고정된다. CLOSED 확정 후에는 해당 기간 전표의 입력·수정·이동·코드변환이 차단되며, 변경이 필요하면 마감해제 승인 후에만 가능하다. 변경이력은 append-only, 물리삭제 금지(감사로그).

2. 공통 UI 표준 (요약)

2.1 결산 작업대 · 재무제표 편집 표준

  • 셸: 상단 GNB(회사명·기장 모드·회계기간·마감상태 배지·사용자) + 좌측 LNB(결산/재무제표) + 콘텐츠. 마감상태가 CLOSED면 입력·전표추가 버튼은 비활성, 조회/출력만 허용.
  • 결산 작업대: 좌측 결산 단계(자료수집→조정분개→재무제표→검토/마감) + 중앙 조정 그리드(조정전·결산분개·조정후·검증) + 우측 검증/주석 패널.
  • 재무제표 편집: 상단 탭(과목별·세목별·관리용·제출용·표준용) + 본표 그리드(비교식: 전기/당기) + 차액 알림 모달(대차차액·잉여금 일치).
  • 2단계 검증: Amber 경고(저장 가능) / Red 차단(저장 불가). 서버 검증 이중 유지. 정합 통과 시 정상.
  • 결산 분개: 분개구분 결차(CLOSING_DR)/결대(CLOSING_CR) 사용. 6 관리차원(사업장*·코스트센터·부서·프로젝트·현장·사원)으로 부문별 손익 롤업.
  • 마감 스냅샷: 확정 시점 시산표·재무제표를 스냅샷 고정. TEMP_CLOSED(잠정)→CLOSED(확정)→LOCKED(잠금) 단계 가드.
  • RLS 스코프: 모든 결산 조회·산출은 tenant_id RLS 필터를 통과. 부문별 조회는 차원 권한으로 추가 제한.
