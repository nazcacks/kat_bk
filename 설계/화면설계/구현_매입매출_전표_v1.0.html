BK 회계/세무 서비스 구현 화면설계서 v1.0 — 매입매출전표입력

정합 기준: bk_설계서_v3.0.md §25.4.2 매입매출전표입력 SA-JNL-02 · 화면 형태: 구현_화면_전표_v3.4.html(일반전표입력) 동일 구조 · 입력 사용흐름 출처: 매입매출 전표입력 1~4편

작성일: 2026-06-25 · 문서 버전: v1.0 · 범위: 매입매출전표를 거래(부가세 신고자료)를 입력하는 상단부와 분개(재무회계자료)를 입력하는 하단부의 2계층으로 설계한다. 상단부는 [부가가치세신고서]·[세금계산서합계표]·[계산서합계표]·[매입매출장]에, 하단부는 [계정별원장]·[총계정원장]·재무제표에 반영된다. 과세유형 코드(매출 11~25 / 매입 51~64)를 신고서식 라우팅의 단일 진실원으로 삼고, 유형별 입력방법·전자발행·자동분개·액션바/툴바·더보기 기능을 모두 화면에 반영한다.

기술 스택: 백엔드 NestJS / 프론트 React 18 / DB PostgreSQL 16(SHARED+RLS·파티셔닝). 모든 그리드는 가상 스크롤·키보드 주도·실시간 검증을 전제로 한다. 부가세 자동라인은 부가세 계정코드(과세유형) 입력 시에만 생성한다.

1. 문서 개요 · 출처 · 범위

목적부가가치세 신고와 관련한 매입매출 거래를 입력하는 화면을 BK 화면설계로 구성한다. 한 전표가 세무 신고자료(상단부)와 재무회계 분개(하단부)를 동시에 만들어 내는 2계층 구조를 그대로 반영하고, 과세유형 선택이 부가세신고서·부속서류 반영의 기준이 된다는 점을 핵심으로 둔다.
구성 범위① 상단부 세무헤더(일자·거래처·유형·품명·수량·단가·공급가액·부가세·전자) · ② 하단부 분개라인(구분·계정·차변/대변·거래처·적요·관리) · ③ 과세유형 코드와 신고서 라우팅 · ④ 유형별 입력방법·의제/재활용 공제 · ⑤ 분개유형·자동분개 · ⑥ 전자발행·전송 · ⑦ 조건검색·액션바·툴바·더보기 · ⑧ 관리항목·검증·체크리스트로 정리한다.
화면 ID 체계본 문서 전용 WPV-xx(BK 매입매출전표입력)를 사용하고, bk_설계서 SA-JNL-02(25.4.2)와 연결한다. 일반전표입력 WJV-01(SA-JNL-01)과는 채번 source 구분(매입매출 50001~99999)·메뉴 형제 관계로 매핑된다.
핵심 전제매입매출전표입력 화면은 ① 상단 세무헤더 그리드 + ② 하단 분개라인 그리드 + ③ 액션바·툴바 + ④ 더보기 기능모음 + ⑤ 가이드영역으로 구성된다. 과세유형(taxType)이 세율·부가세계정·신고서식 매핑을 결정하고, 부가세 자동라인은 과세유형 입력 시에만 생성된다. 상·하단 금액 불일치 시 차액은 [부가가치세신고서] '과표/수입금액제외'로 자동 반영된다.
표기반영 설계 직접 반영 / 보완 기능 확장 / 검토 위험·주의 / 차단 저장·진행 불가. 매출유형은 빨강, 매입유형은 파랑으로 구분. 단축키는 F2 형태로 표기.
주의(공통) — 본 문서는 매입매출전표 입력 사용 흐름을 BK 화면설계로 정리한 것이며 BK 공식 스펙 확정본이 아니다. 정합 기준은 bk_설계서_v3.0.md §25.4.2이고, 입력 동작 서술은 매입매출 전표입력 1~4편을 BK 도메인(journals·tax·integrations·관리차원·보조부)으로 연결하는 방향 제안이다.

2. 화면 골격 · 2계층 구조

매입매출전표는 한 번의 입력으로 세무 신고자료와 재무회계 분개를 동시에 만든다. 그래서 화면이 상단부(세무헤더)와 하단부(분개라인)의 2계층으로 나뉜다. 상단부에서 거래(부가세 신고자료)를 입력하면 부가세신고서·합계표·매입매출장에 반영되고, 하단부에서 분개(재무회계자료)를 입력하면 계정별원장·재무제표에 반영된다.

12.4 구분별 전표 생성 로직 (상세) ★

업로드 행 한 건마다 다음 순서로 판정해 상단 세무헤더 + 하단 분개라인(+부가세 자동라인)을 생성한다.

① 판정 순서 (행 단위)

  1. 매입/매출 구분 — 엑셀 자료 구분(공급자=자사면 매출, 공급받는자=자사면 매입) → 매출 11~25 / 매입 51~64 군 결정
  2. 증빙 종류 — 세금계산서 / 계산서 / 신용카드 / 현금영수증 → 코드 대역 결정
  3. 세액 유무(과세성격) — 세액>0 → 과세, 세액=0 & 세금계산서 → 영세, 계산서 → 면세
  4. taxType 확정 — 위 조합으로 코드 매핑(아래 표). 특수유형(54·55·의제/재활용)은 보류 후 수동 확정
  5. 분개유형 결정 — 세금계산서/계산서=2.외상(기본), 신용카드=4.카드, 현금영수증=1.현금(자동분개 설정으로 변경 가능)
  6. 부가세 자동라인 — 과세유형(세액 있는 과세)에서만 매출=부가세예수금(대변)·매입=부가세대급금(차변) 생성. 영세·면세는 미생성
  7. 고정자산 분기 — 매입 51·57·61 + 계정이 고정자산이면 신고서 일반매입→고정자산매입
