1. 문서 개요 · 페이즈 03 범위

목적페이즈 03 구현 착수를 위해 전표 입력·자동전표 처리 화면별 레이아웃(와이어프레임)·입력 필드·액션/검증·연계 API·권한·상태를 구현 가능한 수준으로 정의한다. 페이즈 01에서 확정한 부모 마스터(회사·환경설정·6 관리차원·거래처·계정과목)와 페이즈 02 전기이월 잔액 위에서, 모든 회계 트랜잭션의 원천 입력 채널(수기 전표·간편입력·매입매출 전표·외부자료 자동분개)을 정의한다.
페이즈 정의일반전표(입금/출금 통합)·현금 간편입력·매입/매출 세금계산서 전표·전표 도구(복사/반복/템플릿/역분개)·전표 검색·일괄 업로드 + BK 전표관리 4화면(일반/매입매출/정기청구/물류전표)·자동전표 파이프라인 10화면(수집정보·전자세금계산서·전자계산서·신용카드·현금영수증·통장·수집현황·전송현황·국세청검증·ScrappingBoard). 외부자료 → 정규화 → 매칭 → 검증 → 전표전송의 5단계 파이프라인을 단일 진실원으로 모델링한다.
화면 구성전표 코어 JV-01~JV-08 8화면 + BK 전표관리 SA-JNL-01~04 4화면 + 자동전표 SA-ATX-01~10 10화면 = 총 22화면. 구조 유형: 전표 헤더/라인(마스터-디테일), 현금 간편입력, 매입매출 2계층, 전표 검색, 일괄 업로드, 자동전표 파이프라인.
표기 원칙화면 ID는 본 문서 전용으로 JV-xx(전표 코어)·SA-JNL-xx(BK 전표)·SA-ATX-xx(자동전표)를 사용하고, 설계서 bk_설계서_v2.1.md 0.2·11·25.4·25.5 절과 연결한다. 검증은 경고(저장가능)·차단(저장불가) 2단계, 정상은 정상으로 표기한다.
설계 의도 · 전표 입력 화면은 채번·대차·보조부·마감 검증이 입력 중 실시간으로 보여야 하므로, 모든 입력형 화면을 좌측 전표 헤더 + 중앙 분개 라인 그리드 + 우측 검증/추가입력 패널의 3분할(wf-split + wf-panel pnl)로 통일했다. 자동전표 화면은 5단계 파이프라인 스트립 + 수집 그리드 + 추천 분개/오류 패널 구조로, 외부자료가 전표로 승격되는 흐름을 한 화면에서 추적할 수 있게 했다.
페이즈 03 핵심 규칙 — 전표번호는 전표일자 단위 매일 001 리셋(더존식), PK=(tenant_id, 전표일자, 전표번호)이며 수동수정 불가. 분개구분 6종(차변/대변/입금/출금/결차 CLOSING_DR/결대 CLOSING_CR)은 현금(101) 라인 위치로 입금/출금을 자동판별한다. 부가세 자동라인은 부가세 계정코드(과세유형) 입력 시에만 생성한다. 입력 우측 「선택 라인 추가입력」 패널의 노출·필수는 SA-BAS-03 환경설정 관리코드 사용여부와 SA-BAS-07 계정과목별 관리항목이 결정한다. 마감기간(TEMP_CLOSED/CLOSED/LOCKED)은 저장·이동을 차단하고, 모든 전표는 append-only·물리삭제 금지(역분개로 정정).

2. 공통 UI 표준 (요약)

2.1 전표 셸 · 입력 그리드 표준

  • 셸: 상단 GNB(회사명·기장 모드 배지·회계기간·마감상태·사용자) + 좌측 LNB(전표/장부·결산/자동전표) + 콘텐츠. 메뉴는 Role 메뉴권한(BF-12), 버튼은 행위권한(BF-07)으로 렌더링.
  • 전표 3분할: 좌측 전표 헤더(일자·번호·구분·사업장) + 중앙 분개 라인 그리드(계정·차변·대변·거래처·관리차원·적요) + 우측 검증/선택 라인 추가입력 패널. 코드도움(F2), 다음 행(Enter), 저장(Ctrl+S), 라인복사(F4).
  • 채번: 전표번호 = 전표일자 단위 00001부터 자동 채번(더존식)·수동수정 불가. PK=(tenant_id, 전표일자, 전표번호). 일자 변경 시 재채번.
  • 분개구분 6종: 차변·대변·입금·출금·결차(CLOSING_DR)·결대(CLOSING_CR). 현금(101) 라인 위치로 입금/출금 자동판별, 「구분」 컬럼에 표시.
  • 2단계 검증: Amber 경고(저장 가능) / Red 차단(저장 불가). 대차 일치·마감기간·부가세 계정 조건·보조부 필수값을 서버 이중 검증.
  • 마스킹(17장): 계좌·카드·주민번호 기본 마스킹, 👁 평문 조회는 Step-up 인증 + 사유 + 자동 재마스킹. 자동전표 수집 자격증명(인증서·ID/PW)은 tenant 단위 암호화 볼트 보관.
  • 불변성: 모든 전표 append-only, 물리삭제 금지. 기표·확정 후 정정은 역분개(CLOSING_DR/CR 또는 반대분개)로 처리하고 감사로그(data_change_log)에 기록.
  • 마감 가드: 마감상태 TEMP_CLOSED/CLOSED/LOCKED 기간은 저장·이동·삭제 차단. 자금(어음)·일마감·계좌관리·증빙·자동전표 데이터는 전표이동 불가.

6. 자동전표 파이프라인 (SA-ATX-01~10)

외부 증빙자료(전자세금계산서·계산서·신용카드·현금영수증·통장)를 수집→정규화→매칭→검증→전표전송하는 5단계 파이프라인이다. 모든 단계는 단일 진실원(collected_doc)을 거치며 상태머신(수집→매칭대기→매칭완료→전표생성→전송완료)으로 추적된다.

파이프라인 원칙 · ① 수집정보등록(SA-ATX-01)에서 채널·인증·계정매핑 규칙을 정의 → ② 채널별 수집(02~06) → ③ 자료수집현황(07)에서 정규화 자료 확인 → ④ 전표 자동생성·전송현황(08) → ⑤ 국세청 자료와 대사검증(09). 스크래핑 작업 모니터는 ScrappingBoard(10).
