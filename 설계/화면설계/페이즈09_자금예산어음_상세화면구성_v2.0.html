1. 문서 개요 · 페이즈 09 범위

목적페이즈 09 구현 착수를 위해 자금·예산·예적금/차입금·어음/당좌 화면별 레이아웃(와이어프레임)·입력 필드·액션/검증·연계 API·권한·상태를 구현 가능한 수준으로 정의한다. 모든 화면은 전표(P03)에서 발생한 자금관리 데이터를 보조원장 키(자금항목·차입금번호·어음번호·계좌)로 재집계하는 자금 보조부(AUX) 성격이며, 예산 화면은 6 관리차원(부서·사원·프로젝트 등) 기준 예산편성과 전표 실적의 BvA(예산대비실적) 분석을 담당한다.
페이즈 정의자금일보·자금계획·일일자금현황·자금일월보·입출금내역·자금계획서 + 경비/매출/고정자산/물품 다차원 예산편성(당기·전기·차기·추정) 및 월/분기/반기 실적 대비 + 예적금·차입금 보조부 + 받을어음/지급어음/어음집계(수불)/당좌수표 보조부. Smart A의 25.17~25.19 절(자금·예산·예적금·어음)을 BK fund_*·budget_*·note(어음)·loan·deposit_account 모델로 일반화한다.
화면 구성자금 SA-FND-01~14 14화면(자금일보·계획·현황·일월보·입출금·계획서 6 + 예산 8) + 예적금/차입금 SA-DEP-01~03 3화면 + 어음/당좌 SA-BIL-01~04 4화면 = 총 21화면.
표기 원칙화면 ID는 본 문서 전용으로 SA-FND-xx(자금·예산)·SA-DEP-xx(예적금/차입금)·SA-BIL-xx(어음/당좌)를 사용하고, 설계서 bk_설계서_v2.1.md 25.17~25.19 절과 연결한다. 검증은 경고(저장가능)·차단(저장불가) 2단계로 표기한다.
관련 페이즈전표 자금관리 입력(P03 JV)·계정과목 관리항목(P01 SA-BAS-07)·거래처 금융탭/초기이월(P01·P02)·할인료 자동분개(BF/JV)·결산 마감상태(P05). 본 페이즈는 읽기·집계 중심이되 자금계획·예산·차입스케줄·어음책 채번은 입력 마스터다.
본 페이즈 화면은 크게 세 가지 패턴이다. ① 자금 대시보드/리포트(상단 KPI 스트립 + 좌측 일자·항목 그리드 + 우측 조건/집계 패널) — 일일자금명세·현황·일월보·입출금. ② 다차원 예산 그리드(좌측 조건 + 중앙 계정×월 매트릭스 + 우측 차원 분석) — 예산입력·실적대비. ③ 어음 보조원장(목록 그리드 + 우측 어음 상세/만기/할인 패널 + 추적 타임라인 모달) — 받을/지급어음·수불·당좌. 입력 마스터는 v2.1 전표 화면의 핵심 그리드 + 우측 패널 패턴을 응용했다.
페이즈 09 연계 핵심 — 자금·예적금·차입·어음 화면의 데이터는 스스로 입력하지 않는다. 원천은 [일반전표]·[매입매출전표]의 자금관리(거래처·어음번호·차입금번호·만기·계좌) 라인이며, 본 화면은 보조원장 키로 재집계·드릴다운한다. 따라서 ① 계정과목 관리항목(SA-BAS-07)에서 어음/차입/당좌 사용 "O", ② 어음은 어음책 채번(SA-BIL-03 어음등록) 선행, ③ 자금항목 코드매핑(SA-FND-02 항목코드등록)이 켜져 있어야 화면에 값이 나타난다.

2. 공통 UI 표준 (요약)

2.1 화면 셸 · 자금/예산/어음 조회 표준

  • 셸: 상단 GNB(회사명·기장 모드 배지·회계기간·마감상태·사용자) + 좌측 LNB(자금/예산/예적금/어음) + 콘텐츠. 메뉴는 Role 메뉴권한(BF-12), 버튼은 행위권한(BF-07)으로 렌더링.
  • 자금 대시보드: 상단 KPI 스트립(전일잔액·입금예정·지급예정·가용자금) + 일자·자금항목 그리드 + 우측 조건/계정설정 패널. 자금항목 코드는 계정 그룹핑·보조원장 키로 통일.
  • 예산 매트릭스: 좌측 조건(기수·차원·계정) + 중앙 계정×월 입력/실적 그리드 + 우측 차원(부서/사원/프로젝트) 분석. 차액=예산−실적, 실적비율=실적/예산×100(예산 0 시 미표시).
  • 어음 보조원장: 목록 그리드 + 우측 어음 상세(원금·보유·만기·은행) + 상태전이 타임라인(발행→배서→할인→만기→결제). 어음번호=note 엔티티 PK, 전표 라인↔보조부 양방향 연계.
  • 2단계 검증: Amber 경고(저장 가능) / Red 차단(저장 불가). 서버 검증 이중 유지.
  • 마스킹(17장): 계좌번호·차입처 계좌 기본 마스킹, 👁 평문 조회는 Step-up 인증 + 사유 + 자동 재마스킹.
  • 변경 이력: 자금계획·예산·차입스케줄·어음책은 append-only 변경이력, 물리 삭제 금지(12.3). 마감(TEMP_CLOSED/CLOSED/LOCKED) 기간은 저장/이동 차단.
  • 6 관리차원: 사업장(*기본필수)·코스트센터·부서·프로젝트·현장·사원. 예산은 차원별 분리 편성, DimensionConfig.required로 회사별 필수 조정.

예산은 6 관리차원(부서·사원·프로젝트·현장·코스트센터·사업장) 중 선택한 축으로 계정 × 월 매트릭스에 편성하고, 전표 실적과 BvA(예산대비실적) 분석한다. 편성 단위는 경비·매출·고정자산·물품 등 예산유형(BudgetType)별로 분리되며, 당기/전기/차기/추정 4개 시나리오를 보관한다. 예산입력은 입력 마스터, 실적대비는 읽기 집계다.
