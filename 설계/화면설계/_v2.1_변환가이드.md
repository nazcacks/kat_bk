# v2.1 화면설계서 변환 가이드 (페이즈01~10 공통)

이 문서는 `구현_화면_전표_v2.1.html`의 비주얼/구조 스타일로 각 페이즈 화면설계서를 재작성하기 위한 **표준 규격**이다. 모든 페이즈 산출물은 아래 CSS 블록과 골격을 **그대로** 사용해 일관성을 유지한다.

---

## 1. 산출물 규칙

- 입력(원본): `페이즈NN_..._v1.0.html` (구식 CSS/구조)
- 출력(신규): 같은 폴더에 `페이즈NN_..._v2.0.html` 로 **새 파일 생성**(원본 v1.0은 보존).
- 원본 v1.0의 **모든 화면 ID·화면명·핵심 필드·목 와이어프레임 내용을 보존**하고, v2.1 스타일로 재표현하면서 **세부 내용을 더 상세하게 보강**한다.
- 화면 누락 금지: 원본의 "페이즈 화면 인벤토리" 표에 있는 모든 화면 ID를 빠짐없이 상세 섹션으로 작성한다.

## 6. 배지·콜아웃 사용 규칙

- `.b-blue` 정보/주요, `.b-green` 정상/완료/기존, `.b-amber` 경고/검토, `.b-red` 차단/오류, `.b-gray` 보조/예정, `.b-teal` 입금/신설, `.b-purple` 특수, `.b-dark` 헤더.
- `.note`(앰버) 주의·차단 규칙 / `.tip`(블루) 설계 의도·레이아웃 설명.
- 2단계 검증 표기: `<span class="badge b-amber">경고</span>`(저장가능) vs `<span class="badge b-red">차단</span>`(저장불가). 서버 검증 이중.

## 7. BK 도메인 공통 개념 (반영)

- 멀티테넌트 SHARED 스키마 + RLS, PostgreSQL 16 파티셔닝.
- 전표번호 = 전표일자 단위 매일 001 리셋(더존식), PK=(tenant_id, 전표일자, 전표번호).
- 6 관리차원: 사업장(*기본필수)/코스트센터/부서/프로젝트/현장/사원. `DimensionConfig.required`로 회사별 필수 조정.
- 분개구분 6종: 차변/대변/입금/출금/결차(CLOSING_DR)/결대(CLOSING_CR).
- 마스킹(계좌·카드·주민번호) 기본, 평문조회 Step-up 인증+사유+재마스킹.
- 변경이력 append-only, 물리삭제 금지(감사로그).
- 마감 상태: TEMP_CLOSED/CLOSED/LOCKED → 해당 기간 저장/이동 차단.
