1. 문서 개요 · 페이즈 07 범위

목적페이즈 07 구현 착수를 위해 부가가치세 첨부서류 및 업종별 서식(영세율첨부·면세유류·월별판매액·농어업환급·전자화폐·전자신고공제·보세판매장·공제받지못할매입세액·건물등감가상각취득·대손세액공제·사업장별/사업자단위과세·과표수정/경정청구·수출실적·외화획득·관세환급·외국인관광객 환급) 20개 화면의 와이어프레임·입력 필드·액션/검증·연계 API·권한·마감 상태를 구현 가능한 수준으로 정의한다. 본 페이즈 서식은 모두 페이즈06 부가세 신고서 본표(SA-VAT-01)의 부속/첨부 명세이며, 본표 집계·전자신고(NTS) 제출 포맷·신고 확정분 잠금과 정합성을 유지한다.
페이즈 정의매입매출전표(매출유형 12/13/16/18/19/23/24, 매입유형 51/52/54/55/57)에서 업종별로 분기하여 자동 집계되는 부속명세와, 사용자가 직접 입력하는 신고·수정·경정 서식을 한 페이즈로 묶는다. 각 서식은 vat_attachment 계열 엔티티에 차수(수정차수)·사업장 차원으로 저장되고, 본표(vat_return)의 해당 라인으로 환류된다.
화면 구성업종·면세 첨부명세 6(SA-VAT-16~21) + 매입세액·자산 명세 3(SA-VAT-22~24) + 사업장·수정신고 4(SA-VAT-25~28) + 영세율 첨부서류 5(SA-VAT-29~33) + 외국인관광객 환급 2(SA-VAT-34~35) = 총 20화면(전부 SA 기능 화면).
표기 원칙화면 ID는 본 문서 전용으로 SA-VAT-16~SA-VAT-35를 사용하고, 설계서 bk_설계서_v2.1.md 25.12~25.14 절과 연결한다. 검증은 경고(저장가능)·차단(저장불가) 2단계로 표기하며, 마감 상태(TEMP_CLOSED/CLOSED/LOCKED)는 첨부 명세 편집·재집계의 가드로 작용한다.
본 페이즈 화면은 좌측 신고 조건/불러오기 패널 + 중앙 서식 본문(탭) + 우측 자동 집계·검증·전자신고 패널의 3분할 「신고서 작업대」 패턴을 공통 적용한다. 동일 업무군 안에서도 첨부 명세(전표 불러오기 그리드형)·합계표(탭 분류형)·신고서(발생/변제 부호 입력형)·납부/환급(본·지점 합산형)·전자신고 파이프라인(단계 상태형)으로 본문 구조를 달리한다.
페이즈 07 연계 핵심 — 모든 첨부 명세는 「부가가치세 불러오기」로 페이즈06 본표(SA-VAT-01) 라인과 동기화된다. 본표가 신고 확정(CLOSED/LOCKED)되면 부속 명세는 조회 전용으로 잠기고, 수정은 수정차수(SA-VAT-27/28)를 통해서만 가능하다. 면세공급가액=0인데 총공급가액≠0(공제받지못할매입세액), 원화금액≠외화×환율(수출실적), 경정청구이유 중복(경정청구서) 등은 저장/마감 차단 규칙이다.

2. 공통 UI 표준 (요약)

2.1 화면 셸 · 신고서 작업대 표준

  • 셸: 상단 GNB(회사명·과세유형 배지·신고기간·사용자) + 좌측 LNB(부가가치세 › 부속·첨부서류) + 콘텐츠. 메뉴는 Role 메뉴권한(BF-12), 버튼은 행위권한(BF-07)으로 렌더링.
  • 신고서 작업대 3분할: 좌측 신고 조건/계정·거래처 설정 후 불러오기 + 중앙 서식 본문 탭(기본사항/명세/집계/검증/전자신고) + 우측 자동 집계·검증·전자신고 반영 패널. 코드도움(F2), 행 추가(Enter), 저장(Ctrl+S).
  • 2단계 검증: Amber 경고(저장 가능) / Red 차단(저장 불가). 클라이언트 + 서버 검증 이중 유지.
  • 마스킹(17장): 사업자등록번호·주민번호·계좌(국세환급금계좌)는 기본 마스킹, 👁 평문 조회는 Step-up 인증 + 사유 + 자동 재마스킹.
  • 변경 이력: 모든 첨부 명세는 append-only 변경이력, 물리 삭제 금지. 「전체 삭제/불러온데이터만 삭제」도 논리 삭제·재집계 로그로 남긴다.
  • 본표 연계·잠금: 「부가가치세 불러오기」로 본표(SA-VAT-01) 라인 동기화. 본표 신고 확정 시 부속 명세 조회 전용 잠금, 수정차수로만 갱신.
  • 6 관리차원: 사업장(*기본필수)·코스트센터·부서·프로젝트·현장·사원. 본·지점/종사업장 신고(SA-VAT-25/26)는 사업장 차원 + RLS로 격리·합산.
