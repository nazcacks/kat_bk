1. 문서 개요 · 페이즈 02 범위

목적페이즈 02 구현 착수를 위해 전기이월·초기 데이터 이관 화면별 레이아웃(와이어프레임)·입력 필드·액션/검증·연계 API·권한·상태를 구현 가능한 수준으로 정의한다. 전기분 재무제표(재무상태표·손익계산서·제조원가명세서·이익잉여금처분계산서·자본변동표·현금흐름표) 입력으로 비교식 재무제표 전기값과 당해연도 기초잔액(opening balance)을 생성하고, 거래처·부서/사원·프로젝트·현장·외주처 등 6 관리차원별 초기이월 명세를 적재하여 후행 전표(P03)·장부(P04)·결산(P05) 화면의 출발점을 확정한다.
페이즈 정의최초 작업년도(이관 첫 해)에만 입력하는 전기 스냅샷 + 차원별 opening balance 화면군. 차기 이후는 SA-BAS-14 마감후이월이 직전 기수의 마감 결과를 fiscal_period 경계로 자동 이월(직접 수정 가능)한다. BK의 "전기분 단일파일" 모델을 tenant + fiscal_period + 계정×차원 기초잔액 엔티티로 일반화한다.
화면 구성전기분 재무제표 입력 SA-OPN-01~06 6화면 + 관리항목별 초기이월 SA-OPN-07·08·13 3화면 + 전기 월별/프로젝트 배분 SA-OPN-09~12 4화면 + 중단사업·중도시산 SA-OPN-14~16 3화면 + 건설·현장 초기이월 SA-OPN-17~19 3화면 = 총 19화면(전부 SA-OPN-* 기능 화면).
표기 원칙화면 ID는 본 문서 전용으로 SA-OPN-xx를 사용하고, 설계서 bk_설계서_v2.1.md 25.2·25.3 절과 연결한다. 검증은 경고(저장가능)·차단(저장불가) 2단계로 표기한다. 금액 우측칸 +키는 천원단위 "000" 자동입력, 계정/거래처명 2글자 또는 F2 코드도움.
본 페이즈는 입력·검증·정합성 중심이므로 화면 대부분이 좌측 계정/항목 그리드 + 우측 정합성·차액 검증 패널의 마스터-디테일이거나, 월별·연도별 배분 매트릭스 그리드다. 전기분 재무제표 4종은 상호 정합성(대차균형·당기제품제조원가 일치·당기순손익 일치·이월이익잉여금 일치)을 서버 측 제약으로 강제하고, 초기이월 보조부는 좌측 재무상태표 준거금액 vs 우측 차원별 합계의 차액 0을 검증한다.
페이즈 02 연계 핵심 — 전기이월은 SA-BAS-14 마감후이월 및 fiscal_period와 연계된다. SA-OPN-01 전기분재무상태표가 모든 초기이월의 준거 금액(차원 합계가 일치해야 하는 기준)이며, SA-OPN-02/03/04는 당기제품제조원가·당기순손익·이월이익잉여금으로 서로 묶인다. 입력 결과는 계정×차원 opening_balance로 적재되어 P03 전표의 기초잔액·P04 보조부 전기이월·P05 비교식 재무제표 전기열에 공급된다.

2. 공통 UI 표준 (요약)

2.1 화면 셸 · 전기이월 입력 표준

  • 셸: 상단 GNB(회사명·기장 모드 배지·회계기간·사용자) + 좌측 LNB(전기이월 그룹) + 콘텐츠. 메뉴는 Role 메뉴권한(BF-12), 버튼은 행위권한(BF-07)으로 렌더링.
  • 최초년도 가드: 전기분 입력은 이관 첫 해에만 활성. 차기부터 SA-BAS-14 자동반영 행은 자동이월 배지로 표시하고 직접 수정 시 변경이력 기록.
  • 입력 단축키: 계정/거래처명 2글자 자동완성 또는 F2 코드도움(세목 사용 시 5자리), 금액칸 + 천원단위 "000", 다음 행 Enter, 저장 Ctrl+S.
  • 2단계 검증: Amber 경고(저장 가능) / Red 차단(저장 불가). 대차차액·차원합계 불일치는 서버 측 제약으로 이중 검증.
  • 마감 상태 가드: 대상 fiscal_period가 TEMP_CLOSED/CLOSED/LOCKED면 해당 기간 전기이월 저장/이동 차단(조회만).
  • append-only: 모든 초기이월·전기 스냅샷은 변경이력 append-only, 물리 삭제 금지. 초기화는 논리 삭제 + 감사로그.
  • 6 관리차원: 사업장(*기본필수)·코스트센터·부서·프로젝트·현장·사원. 초기이월 입력 가능 차원은 DimensionConfig.required 및 SA-BAS-03 관리코드 사용여부로 결정.
