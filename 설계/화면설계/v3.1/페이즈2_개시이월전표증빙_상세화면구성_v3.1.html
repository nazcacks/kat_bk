1. 문서 개요 · 페이즈 2 범위

목적페이즈 2는 페이즈 1에서 확정된 기초정보·마스터 위에 개시(전기·초기)이월 → 전표 입력 → 증빙 수집·자동전표의 데이터 입력 핵심 동선을 구현한다. 전기분 재무제표·관리항목별 초기이월로 기초잔액의 준거를 확정하고, 일반/매입매출 전표 코어와 BK 전표관리, 그리고 증빙 수집→정규화→매칭→전표전송 자동전표 파이프라인을 화면 단위로 정의한다.
페이즈 정의최초년도 전기분 6서식(BS·PL·원가·처분·자본변동·현금흐름)과 관리항목별·월별·프로젝트·중단사업·중도시산·건설/현장 초기이월(OPN-01~19) + 전표 코어(JV-01~08)·BK 전표관리(SA-JNL-01~04) + 자동전표 파이프라인(SA-ATX-01~10). 모든 입력은 tenant_id RLS 스코프와 마감기간(fiscal_period) 제약을 통과한다.
화면 구성전기·초기이월 OPN-01~19 19화면 + 전표 입력 JV-01~08·SA-JNL-01~04 12화면 + 증빙·자동전표 SA-ATX-01~10 10화면 = 총 41화면. 신규 화면은 없으며, 전 페이즈 v3.0(P02 전기이월·P03 전표/자동전표)의 상세 와이어프레임을 v3.1 5-페이즈 구조로 재편 수록한다.
메뉴 매핑채널 ② 회사 작업공간(TN) 기준: M-WS-01 회사 설정·기초정보 ▸ 전기·초기이월(OPN-01~19) · M-WS-03 전표·장부 ▸ 전표 입력(JV-01~08·SA-JNL-01~04) · M-WS-04 증빙·전자문서(자동전표)(SA-ATX-01~10).
표기 원칙화면 ID는 본 문서 전용으로 OPN-xx(전기·초기이월, 소스 SA-OPN-xx)·JV-xx/SA-JNL-xx(전표)·SA-ATX-xx(자동전표)를 사용한다. 검증은 경고(저장가능)·차단(저장불가) 2단계로 표기한다.
개시이월(OPN)은 좌측 계정/준거 그리드 + 우측 정합성 검증 패널의 정합성 강제형 입력이, 전표(JV/SA-JNL)는 헤더 + 분개 라인 그리드 + 우측 추가입력·검증 패널의 3분할이, 자동전표(SA-ATX)는 채널 수집 → 정규화 → 매칭 → 전송의 상태머신 모니터형이 핵심 패턴이다.
페이즈 2 연계 핵심 — OPN-01 전기분재무상태표의 계정금액이 OPN-07·08·13·17~19 차원별 초기이월의 준거금액이며, OPN-16 중도시산표는 개시전표 배치로 전환된다. 전표 입력 우측 「선택 라인 추가입력」 패널의 노출/필수는 페이즈 1의 SA-BAS-03 환경설정·SA-BAS-07 계정 관리항목 설정이 결정한다. SA-ATX 자동전표는 매칭완료 자료를 journal로 커밋하며 source_doc_id로 양방향 추적·롤백된다.
