1. 문서 개요 · 페이즈 4 범위

목적페이즈 4는 회계 데이터의 기간 관리·마감 통제 → 결산정리 → 재무제표 작성 → 외부 공시(XBRL) → 다중 회계기준·외화 처리를 하나의 페이즈로 묶어, 결산 사이클의 통제(거버넌스)·산출물(재무제표/공시)·고급 회계(병행원장/외화환산)를 구현 가능한 수준으로 정의한다. 회계기간 상태머신(OPEN→SOFT_CLOSE→HARD_CLOSE→LOCKED)이 전 페이즈의 전표·결산·신고 가드레일의 단일 기준이 된다.
페이즈 정의두 채널 메뉴 카탈로그 중 TN(세무·결산 워크스페이스) 측의 결산 영역을 담당한다. M-WS-05 회계기간·마감(기간/검증/마감/해제) · M-WS-07 결산(결산정리·재무제표 본표·자본/현금흐름·출력/분석·세무명세서·기간/중단사업) · M-WS-09 공시(주석편집·XBRL 생성/검증·제출) · M-WS-10 다중기준·외화(병행원장·이중통화·외부수신·기준변환·외화환산·환율마스터).
화면 구성이관 21화면(SA-CLS-01~21, 페이즈05 v3.0) + 신규 13화면(PRD-01~04 회계기간·마감 4 / DSC-01~03 공시 3 / FX-01~06 다중기준·외화 6) = 총 34화면.
표기 원칙화면 ID는 메뉴 채널 기반으로 PRD-xx(기간·마감)·SA-CLS-xx(결산·재무제표)·DSC-xx(공시)·FX-xx(다중기준·외화)를 사용한다. 검증은 경고(저장가능)·차단(저장불가) 2단계로 표기한다. 마감 상태머신은 본 페이즈 전역 가드의 기준이다.
본 페이즈에서 가장 많은 신규 설계가 발생하는 영역은 회계기간·마감(PRD)과 다중기준·외화(FX)다. PRD는 BK의 "기수 단일파일 마감후이월" 모델을 fiscal_period 상태머신 + 마감 체크리스트 + 마감 해제 승인 워크플로우로 일반화하고, FX는 K-IFRS/일반기업회계 병행원장과 기능통화/표시통화 이중통화를 신규 도입한다.
페이즈 4 연계 핵심 — PRD-03 본마감(HARD_CLOSE)이 전 페이즈 전표 입력·수정의 차단 기준이며, 본마감 스냅샷(closing_snapshot)이 SA-CLS-13 재무제표 일괄출력·DSC-02 XBRL 생성의 입력이 된다. FX-05 외화환산 결과는 결산정리분개(SA-CLS-01)로, FX-04 기준변환 차이는 병행원장(K-IFRS) 재무제표로 반영된다.

2. 업무 흐름 (마감 통제 → 결산 → 재무제표 → 외화/병행 → 공시)

결산 사이클은 마감 전 검증으로 입력 정합성을 확보한 뒤, 가마감으로 데이터를 동결하고 결산정리분개·외화환산을 반영하여 재무제표를 산출하고, 본마감으로 스냅샷을 고정한 후 공시(XBRL)로 외부 제출한다.
