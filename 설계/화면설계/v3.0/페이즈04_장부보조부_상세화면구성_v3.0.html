1. 문서 개요 · 페이즈 04 범위

목적페이즈 04 구현 착수를 위해 주요 장부·보조 장부 화면별 조회조건·결과 그리드·합계/차액·원전표 드릴다운·대사·추적 타임라인을 구현 가능한 수준으로 정의한다. 전표(P03)에서 기표된 POSTED 분개를 계정·거래처·기간·6 관리차원으로 재집계·열람하는 읽기형 리포트 군이며, 보조부 일부는 전표 라인 입력 중 F4로 호출되는 입력 팝업을 함께 갖는다.
페이즈 정의전표 보조부 AUX-01~09(거래처/어음/예금/카드/가지급금·가수금/부가세/현금/재고/고정자산)와 BK 장부 SA-LDG-01~29(전표출력·분개장·일월계표·총계정원장·현금출납장·계정별/거래처/총괄/매입매출장·관리원장·계산서 현황·적요별·전도금·공사·외주·운행·차량·원가/손익 안분·현황·집계·프로젝트 누적)를 하나의 페이즈로 묶는다. 모든 장부는 append-only 분개 데이터의 뷰이므로 장부 화면에서는 데이터를 생성하지 않고(보조부 F4 입력은 전표 라인에 귀속) 집계·드릴다운만 수행한다.
화면 구성전표 보조부 AUX-01~09 9화면 + BK 장부 SA-LDG-01~29 29화면 = 총 38화면. 화면 구조는 ① 원장 리포트(조회조건 툴바 + 목록 그리드 + 합계 풋터), ② 대사형(예금), ③ 추적 타임라인(어음), ④ 차원 GROUP BY 원장, ⑤ 현금출납장, ⑥ 카드/부가세 집계표 6유형으로 분류된다.
표기 원칙화면 ID는 본 문서 전용으로 AUX-xx(전표 보조부)·SA-LDG-xx(BK 장부)를 사용하고, 상위 화면설계서 bk_설계서_v2.1.md 25.6~25.8 절·11.4·11.11과 연결한다. 검증은 경고(저장가능/조회허용)·차단(저장불가/조회차단) 2단계로 표기한다.
본 페이즈는 조회·집계 리포트 중심이므로 화면 대부분이 v2.1 전표 화면의 조회조건 툴바 + 결과 그리드 + 합계/검증 풋터 패턴을 사용한다. 장부는 좌측에 무거운 마스터를 두지 않고 상단 조회조건 → 본문 대용량 그리드 → 풋터 합계의 1열 흐름이며, 보조부는 여기에 좌(F4 입력 팝업) / 우(대사·추적·집계 패널)를 더한다. 모든 그리드 행은 더블클릭 시 원전표(P03 JV-07)로 드릴다운한다.
페이즈 04 연계 핵심 — 보조부 호출은 관리항목이 설정된 계정에서만 발생한다. P01 SA-BAS-07 계정과목 관리항목(거래처필수/어음/카드/예금/수량/세무)이 켜진 계정 라인에 진입할 때 P03 전표에서 F4 보조부 팝업이 자동/수동 호출되고, 저장 시 journal_line_aux(전표 라인 1:N 보조부)가 생성된다. 본 페이즈 장부·보조원장은 그 journal_line_aux·journal_line을 POSTED 전표만 집계한다(임시저장·결재대기 미반영). 마감기간(TEMP_CLOSED/CLOSED/LOCKED) 신규 기표는 차단되나 조회는 허용된다.

2. 공통 UI 표준 (요약)

2.1 장부·보조부 조회 표준

  • 셸: 상단 GNB(회사명·기장 모드 배지·알림·사용자) + 좌측 LNB(장부·결산 군) + 콘텐츠. 메뉴는 Role 메뉴권한(BF-12), 버튼은 행위권한(BF-07)으로 렌더링. 장부는 대부분 tenant:ledger:read 권한.
  • 조회조건 툴바: 기간(회계기간 컨텍스트 내)·계정과목(F2/C 코드도움)·거래처(F2/C, 공란=전체)·6 관리차원 필터(사업장*필수 + 코스트센터·부서·프로젝트·현장·사원) + 구분(세목 사용 시 0.과목/1.세목). 조회 실행 후 그리드 렌더.
  • 결과 그리드 + 합계 풋터: 일자→전표번호(복합키) 정렬, 좌측 텍스트·금액 우측정렬(.r), 전기이월·기간합계·누계·차기이월 풋터. Enter/더블클릭으로 원전표 드릴다운.
  • 2단계 검증: Amber 경고(필수누락이나 조회 가능) / Red 차단(권한·차대불일치·기간초과 조회 차단). 서버 검증 이중 유지.
  • 마스킹(17장): 예금 계좌번호·카드번호 기본 마스킹, 👁 평문 조회는 Step-up 인증 + 사유 입력 + 자동 재마스킹. 마스킹 로그 append-only.
  • append-only·물리삭제 금지: 장부는 분개 데이터의 뷰이므로 장부에서 수정/삭제 불가 — 정정은 원전표 역분개(JV-06). 보조부 상태변경(어음 배서/할인/결제)은 이력으로 누적.
  • 6 관리차원: 사업장(*기본필수)·코스트센터·부서·프로젝트·현장·사원. DimensionConfig.required로 회사별 필수/선택 조정. 차원 조건은 라인 단위 매칭(차원별 원장 11.2).

4. 전표 보조부 화면 (상세)

4.1보조부 개념 · 전표 연동 구조

전표 보조부(보조원장)는 특정 계정의 거래 내역을 관리항목(거래처·어음·카드·예금·수량·세무 등) 단위로 분해·집계하는 종속 장부다. 보조부는 별도 입력 화면이 아니라 ① 전표 라인 입력 중 관리항목 계정에 진입할 때 호출되는 입력 팝업(F4)과 ② 본 페이즈의 독립 조회용 보조원장 뷰 두 형태로 제공된다. 입력은 항상 전표에서 발생(단일 입력원)하며, 보조부는 그 결과를 journal_line_aux 관리항목 키로 누적한다.

구조설명 (근거)
입력원모든 보조부 기표는 전표 라인에서 발생. 라인 저장 시 journal_line_aux(1:N)에 보조부 유형·참조키·금액·수량·일자 기록(11.1).
관리항목 트리거계정과목의 관리항목 설정(거래처필수/어음/카드/예금/수량/세무, SA-BAS-07)에 따라 라인 진입 시 해당 보조부 팝업이 자동/F4 호출. 필수 미입력 시 Red 차단.
집계 시점보조원장 잔액·증감은 POSTED(기표) 전표만 집계(11.2). 임시저장·결재대기 전표는 미반영(별도 표시).
독립 뷰(본 페이즈)거래처/어음/예금/카드/가지급금/부가세/현금/재고/고정자산 보조원장은 장부 조회(TN-07)에서 조건 검색·원전표 드릴다운.
차원 직교성보조부 1차 키는 관리항목, 차원(사업장·코스트센터·프로젝트·현장)은 전표 라인 속성이라 F4 팝업에는 차원을 두지 않는다(라인에서 입력). 독립 뷰는 보조부 성격에 맞게 차원을 필터/컬럼으로 선별 제공.
마감 연동마감기간 보조부 신규 기표는 전표와 동일하게 CLOSING_LOCKED로 차단(16.1). 조회는 허용.
