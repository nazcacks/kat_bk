1. 문서 개요 · 페이즈 08 범위

목적페이즈 08 구현 착수를 위해 부가세 영세율·기타 신고서류(SA-VAT-36~50)와 고정자산(SA-FA-01~07) 화면별 레이아웃(와이어프레임)·입력 필드·액션/검증·연계 API·권한·상태를 구현 가능한 수준으로 정의한다. 영세율 매출 부속서식과 고정자산 등록·감가상각·처분·월별 상각비 계상을 하나의 페이즈로 묶어, 부가세신고서(P06) 영세율 금액 정합과 결산 감가상각 자동전표(P05) 연계를 확정한다.
페이즈 정의영세율 첨부서류(수출실적명세서·영세율매출명세서·외화획득명세서·내국신용장/구매확인서 전자발급명세서·영세율 첨부서류 제출명세서) + 외국인관광객·외교관·외항선박·항공 영세율 부속서식 + 과세유형 전환(재고품/감가상각자산) + 사업용계좌·납부서·사업양도 + 고정자산 마스터(취득가·내용연수·상각방법 정액/정률)·감가상각비 계산(월할/연할)·감가상각명세서·자산 양도/폐기/이동·세무상 상각범위액(K-IFRS vs 법인세법)·업무용승용차 한도. BK "기수 단일파일" 모델을 tenant+fiscal_period로 일반화한다.
화면 구성영세율·기타 신고서류 SA-VAT-36~50 15화면(숙박/즉시환급/외교관면세/공급기록/공급가액확정/선박/외항선박/영세율매출명세/과세전환 재고품·감가상각자산/매입자발행/과세유흥/사업용계좌/납부서/사업양도) + 고정자산 SA-FA-01~07 7화면(등록·미상각/양도 상각계산·원가경비별 명세·관리대장·현장/부서 명세·월별 계상) = 총 22화면.
표기 원칙화면 ID는 본 문서 전용으로 SA-VAT-xx(부가세 영세율·기타)·SA-FA-xx(고정자산)를 사용하고, 설계서 bk_설계서_v2.1.md 25.14·25.15·25.16 절과 연결한다. 검증은 경고(저장가능)·차단(저장불가) 2단계로 표기한다.
본 페이즈는 신고 작업대 + 고정자산 마스터/리포트 중심이므로, SA-VAT 화면 대부분이 좌측 신고조건 + 중앙 서식 본문(탭) + 우측 검증/전자신고 패널의 3분할(신고서 작업대)이고, SA-FA 화면은 좌측 조회조건 + 중앙 자산 그리드 + 우측 라이프사이클/연계 패널이다. 영세율 첨부서류는 모두 매입매출전표(12.영세/16.수출 계열) 집계를 불러오기로 채운다.
페이즈 08 연계 핵심 — SA-VAT-43 영세율매출명세서의 (13)영세율 적용 공급실적 총합계가 부가세신고서(SA-VAT-01) 과세표준 영세율 금액과 정합해야 하며, SA-FA-07 월별 감가상각비 계상이 생성하는 결차/결대 자동전표가 결산(P05) 감가상각분개로 흘러간다. 고정자산대장(SA-FA-05) 금액은 총계정원장 잔액과 대사된다.

2. 공통 UI 표준 (요약)

2.1 화면 셸 · 신고 작업대/자산 마스터 표준

  • 셸: 상단 GNB(회사명·기장 모드 배지·알림·사용자) + 좌측 LNB(부가세 신고/고정자산/결산) + 콘텐츠. 메뉴는 Role 메뉴권한(BF-12), 버튼은 행위권한(BF-07)으로 렌더링.
  • 신고 작업대 3분할: 좌측 신고조건(신고기간·사업장·수정차수) + 중앙 서식 본문 탭(기본/명세/집계/검증/전자신고) + 우측 MRI 검증·전자신고 반영 패널. 코드도움(F2), 다음 행(Enter), 저장(Ctrl+S).
  • 2단계 검증: Amber 경고(저장 가능) / Red 차단(저장 불가). 서버 검증 이중 유지.
  • 마스킹(17장): 계좌(사업용계좌)·사업자번호·여권번호 기본 마스킹, 👁 평문 조회는 Step-up 인증 + 사유 + 자동 재마스킹.
  • 변경 이력: 모든 신고서·자산 마스터는 append-only 변경이력, 물리 삭제 금지. 양도/폐기는 라이프사이클 이벤트로 논리 처리.
  • 마감 가드: 마감 상태(TEMP_CLOSED/CLOSED/LOCKED)인 기간의 신고서 저장·감가상각 재계산·월별 계상은 차단 또는 경고.
  • 6 관리차원: 사업장(*기본필수)·코스트센터·부서·프로젝트·현장·사원. 고정자산 추가등록사항에 부서·프로젝트·현장·사원을 배부(SA-FA-06).
